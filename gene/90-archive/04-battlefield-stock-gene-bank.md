# 第四战场：个股基因库 (Stock Gene Bank) 初步方案

**状态**: 概念构想 (Concept Formulation)
**定位**: Gamma 观察台 (Observatory) 长期演进方向

---

## 1. 战场定位与核心问题

**系统定位**：作为选股与交易系统的**终极滤镜与仓位分配器**。彻底切除宏观与行业的自上而下视角（如原 MSS 市场温度、IRS 行业轮动），将关注点完全收敛至“微观个股的绝对特征与历史印记”。一切交由“演员个人的表演”来决定。

**核心解决的问题**：
1. **个股相对强弱（Stock Relative Strength）**：
   - *问题*：在 BOF（Breakout Factor 或其他触发条件）已经选出的候选中，哪只股票最强？
   - *方法*：比较**个股间**的相对强弱，以及与**该股自身历史巅峰期**的强弱比对，而不是行业间的相对强弱。
2. **个股风险特征（Stock Risk Profile）**：
   - *问题*：这只股票的波动率、最大回撤、夏普比率如何？它是一只有韧性的好股，还是爱“坑人”的妖股？
   - *方法*：通过沉淀的长周期数据定义其核心风险特征。

---

## 2. 核心组件：六大基因序列组合

基于“变废为宝”原则，从原先废弃的 MSS、IRS、PAS 组件中，提取仅针对**个股**有效的微观定量特征，重组为“个股基因字典”：

| 基因族 | 核心拆解指标（示例） | 评估目的（人物特写） |
| :--- | :--- | :--- |
| **波段爆发基因 (Uptrend)** | 历史主升浪时长、涨幅极值中位数、新高密集度 | 评估演员顺风局的爆发力上限（能涨多高、涨多久） |
| **波段调整基因 (Downtrend)** | 历史杀跌回撤极值、阴跌平均时长 | 评估退潮期的杀跌下限，测算安全垫厚度 |
| **量能弹性基因 (Liquidity)** | 最大换手率阈值（爆量见顶参数）、天量频次 | 识别资金对该股的偏好度及真实的流动性特征 |
| **极端性格基因 (Extreme)** | “核按钮”频次（一字跌停开盘）、A字杀频率 | 黑名单属性：排雷、规避“股性极差”的坑人标的 |
| **波动振幅基因 (Volatility)** | 日均上下影线振幅、启动日波动特点 | 捕捉日内震荡极限，定义绝对 Risk Profile |
| **时间周期基因 (Seasonality)** | 两波大行情间的沉寂跨度、活跃月份偏好 | 滤除尚未洗盘充分、时间周期未到的标底 |

---

## 3. 验证与落地机制

基因库建立后，将在整个执行流的尾端发挥以下两大决定性作用：

### 3.1 强弱验证：Normandy 方式的 Formal Readout 
*   **动作**：去伪存真的长窗审视。
*   **落地**：当某只股票触发底层买点后，并不立刻上报。系统调用基因库，对比它 *当前的走势形态与量价表现* 与其 *历史最优的波段爆发基因*。如果当前的表现连它自己历史上的“及格线”都没达到，或者撞上了“时间周期基因”中的休眠期，则**一票否决**。

### 3.2 规模验证：Positioning 方式解决“买多少” 
*   **动作**：根据个股先天基因定风险敞口。
*   **落地**：结合 **“波段调整基因”**（最大回撤极值）、**“极端性格基因”** 和 **“波动振幅基因”**，为这只股票核算出一个专用的 `Risk_Weight`（风险乘数）。
    - 波动大、爱深V、历史最大回撤高达60%的股票 -> 缩小仓位（轻仓博弈）。
    - 趋势连贯、日内振幅小、历史阴跌可控的中军型标的 -> 放大仓位（重仓配置）。

---

## 4. 结论与演进路线

**结论**：个股基因库（Stock Gene Bank）将成为所有前置漏斗之后的最后一道“滤镜”。它不再去判断“今天是个什么好日子”，而是苛刻地质检“你到底是不是个好角儿”。这也标志着整个系统从“宏观配置型”向“微观实战型”的彻底转变。

**后续跟进规划**：
- [ ] 量化基准：把上述“六大基因”翻译成可算式的 DuckDB/Pandas 计算 SQL。
- [ ] 数据构建：定义并生成一张大宽表 `dim_stock_gene_profile` 进行每日/每周滚动更新。
- [ ] 策略平替：探索完全抛除大盘系数，仅使用基因过滤进行回测的可行性。
