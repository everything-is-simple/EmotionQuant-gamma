## 1. 目标

把当前 `P0 / P1` 主链收口到：

`可安全进入 Phase 2 / IRS 的稳定状态`

本卡不新增算法能力，也不提前实现 `Phase 2 / Phase 3` 的正文目标。

本卡只回答一件事：

`当前主链是否已经具备可信开工 Phase 2 的工程前提。`

---

## 2. 范围

当前 phase 只做：

1. `build_l3` 的 `MSS / IRS` 进度锚拆分
2. `run_id / trace` 清理与保留策略收口
3. `quality` 当前定位冻结与回归保护
4. pytest 环境下 `get_settings()` 缓存污染修复
5. 一次显式 `MSS overlay` 激活验证
6. `l3_signals` 在实验链路中的角色定性

当前 phase 明确不做：

1. 不升级 `IRS` 因子正文
2. 不升级 `MSS phase / risk_regime`
3. 不把 `quality` 写回 `final_score`
4. 不重写 `P4` 全套归因脚本
5. 不在本卡内追加新的交易语义
