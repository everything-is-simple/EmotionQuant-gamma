`在多个 Gene 字段已经各自完成 isolated win 之后，哪一个最小组合值得进入正式组合验证，而不是继续停留在口头联想。`

当前它不能开工，原因很简单：

1. 当前已经有 `3` 个 isolated winner：`duration_percentile >= 95`、`reversal_state == CONFIRMED_TURN_DOWN -> exit-preparation only`、`context_trend_direction_before -> parent-context negative guard`

4. `wave_role` 已完成，但 ruling 是 `retain_sidecar_only`，不能偷带进组合

因此当前真实状态是：

`Phase 9C 已不再受“winner 数量不足”阻塞，但仍没有开工资格。`

## 3. 开工门槛

只有同时满足下面条件，这张卡才允许切成 `Active`：

1. 至少 `2` 个 Gene 字段已经拿到 isolated win
2. 每个字段的运行角色都已被 formal record 写死
3. [`../../gene/03-execution/19-phase-gx8-three-level-trend-hierarchy-card-20260318.md`](../../gene/03-execution/19-phase-gx8-three-level-trend-hierarchy-card-20260318.md) 已完成或被正式裁定为 `non-blocking`
4. 组合对象必须来自已经过 isolated round 的字段，不能偷带新字段

当前门槛状态应读作：

1. 第 `1` 条已满足
2. 第 `2` 条已满足
3. 第 `3` 条未满足
4. 第 `4` 条尚未冻结成正式组合对象

## 4. 当前显式禁止

当前先写死这些禁止项：

1. 在只有 `duration_percentile` 一个 winner 的情况下偷开组合
2. 把 `wave_role` 这种当前已被裁定 `retain_sidecar_only` 的字段偷偷并入组合
3. 把 `current_wave_age_band / mirror / conditioning / gene_score` 一起塞进组合
4. 把组合验证包装成“其实只是新一轮单变量改名”

这张卡未来若被激活，必须留下：

2. isolated winner 引用链
3. 正式 replay evidence
4. formal ruling
