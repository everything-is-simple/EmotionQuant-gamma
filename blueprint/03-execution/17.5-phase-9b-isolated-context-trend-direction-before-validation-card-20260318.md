## 1. 本轮问题

这张卡只回答一个问题：

`如果只把 context_trend_direction_before 以 parent-context guard 身份接进当前 validated baseline，它能不能形成一条和 wave_role 不同、且值得保留的单变量 runtime 规则？`

## 2. 为什么这张卡不是下一张 active 卡

这张卡现在只开出来，不立即进入实现，原因有三条：

1. `wave_role` 刚完成 isolated validation，当前先要让 `reversal_state` 接下一棒
2. `context_trend_direction_before` 与 `wave_role` 共享父趋势参照语义，存在明显的“换名重复验证”风险
3. `GX8 / three-level trend hierarchy` 仍是这个字段后续是否值得升格的重要外部阻塞项

因此当前真实状态是：

`17.5 已开卡，但仍保持 Planned。`

本轮只允许改一个主变量：

`add context_trend_direction_before parent-context guard semantics to the validated baseline`

1. `context_trend_direction_before + wave_role`
2. `context_trend_direction_before + duration_percentile`
3. `context_trend_direction_before + reversal_state`
4. 任意 `mirror / conditioning / gene_score` 伴随进入
