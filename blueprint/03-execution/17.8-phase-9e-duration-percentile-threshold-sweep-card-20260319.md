# Phase 9E / duration_percentile 阈值扫描卡

- 状态: `Active`
- 日期: `2026-03-19`
- 类型: `single-variable follow-up validation`
- 范围: `把 duration_percentile 从 p65 到 p95 按 5 点步长做正式 sweep`

## 1. 这张卡回答什么

这张卡只回答一个问题：

`如果真正把 duration_percentile 做细，而不是只看 p95 与 p65 两个端点，那么寿命轴上真实的转折点到底在哪里？`

---

## 2. 为什么必须开这张卡

当前仓库只留下了：

1. `p95` formal round
2. `p65` sensitivity reference

这不够回答下面这些问题：

1. `p95` 是真实 turning point，还是只是极端端点
2. `p65 ~ p95` 中间是否存在更诚实的阈值
3. 改善曲线在哪个区段开始变钝
4. 斜率是否发生反转
5. 你真正要的低频、高胜率、高盈亏比机会，是否集中在更窄或更早的寿命区间

因此当前必须补：

`p65 / p70 / p75 / p80 / p85 / p90 / p95`

另外，`2026-03-19` 的 `GX12` 已正式裁定：

1. `GX10` 已改变 duration 轴的参考基础
2. 因此本卡不能继续沿整改前 duration surface 往下跑
3. 当前任何未完成的 sweep 尝试，都应视为：
   `pause legacy-surface attempt and reopen on remediated duration surface`

---

## 3. 当前固定 baseline

这张卡所依附的固定 baseline 仍然是：

`legacy_bof_baseline + FIXED_NOTIONAL_CONTROL + FULL_EXIT_CONTROL + Gene sidecar only`

本卡不允许重开：

1. `IRS-lite / MSS-lite`
2. Gene exit modulation
3. Gene sizing overlay
4. multi-field Gene package

---

## 4. Sweep Boundary

本卡只允许改变一个主变量：

`duration_percentile threshold`

当前冻结的 sweep 轴严格只有：

1. `p65`
2. `p70`
3. `p75`
4. `p80`
5. `p85`
6. `p90`
7. `p95`

本卡当前默认先沿用的规则形态只有这一种：

`block when current_wave_duration_percentile >= threshold`

本卡可以研究：

1. 曲线拐点
2. 斜率变化
3. “加速度反向点”是否存在

但本卡**不能**直接跳成新的 runtime rule 形态，例如：

1. `p70 ~ p85` 区间带状规则
2. 动态 percentile rule
3. duration + another Gene field 组合 rule

若扫描结果真的证明需要新区间规则，必须在本卡结论里明说，并另开更小 card。

---

## 5. Required Outputs

本卡完成时，必须正式留下：

1. 每个 threshold 一份 formal evidence JSON
2. 一份汇总表，至少包括：
   - `signals_count`
   - `buy_filled_count`
   - `trade_count`
   - `reject_rate`
   - `expected_value`
   - `profit_factor`
   - `max_drawdown`
3. full-window 对比
4. split-window 对比
5. turning-point / slope-change 判断
6. 一个明确结论：
   - `duration remains a truthful runtime candidate`
   - 或 `duration should return to sidecar only at package level`
