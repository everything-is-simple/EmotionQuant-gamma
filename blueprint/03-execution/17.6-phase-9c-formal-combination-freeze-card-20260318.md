# Phase 9C / Formal Combination Freeze

- 状态: `Completed`
- 日期: `2026-03-18`
- 类型: `formal combination freeze 子阶段`
- 范围: `先冻结最小合法组合范围，再决定组合 replay 是否需要另开 follow-up`

## 1. 这张卡回答什么

这张卡当前只回答一个问题：

`在 Phase 9B 已经出现多个 isolated winners 之后，Phase 9C 现在到底允许哪几个组合候选进入后续验证，以及哪些字段必须继续排除在外。`

## 2. 当前真实角色

截至 `2026-03-19`，这张卡已经完成的事情是：

`freeze the smallest lawful combination surface`

它没有完成的事情是：

1. 正式组合 replay
2. package-level promotion closeout
3. duration 阈值曲线细化

因此这张卡的 truthful 角色始终只是：

`freeze`

不是：

`freeze + pretend replay already happened`

## 3. Frozen Source Field Pool

当前只允许从下面 `3` 个已经赢下 isolated round 的字段里选组合候选：

1. `duration_percentile`
2. `reversal_state`
3. `context_trend_direction_before`

单字段结论在本卡中的角色固定为：

`upstream isolated evidence only`

## 4. Frozen Allowed Combination Set

当前 Phase 9C 明确允许的组合候选只有 `4` 个：

1. `duration_percentile + reversal_state`
2. `duration_percentile + context_trend_direction_before`
3. `reversal_state + context_trend_direction_before`
4. `duration_percentile + reversal_state + context_trend_direction_before`

除这 `4` 个之外，本卡不允许：

1. 把单字段重新伪装成“组合候选”
2. 临时加第四个字段
3. 用隐藏附加条件、解释性补丁、排序补偿或 tie-break 偷带额外变量

## 5. Explicitly Forbidden

当前明确禁止偷带以下字段：

1. `wave_role`
2. `current_wave_age_band`
3. `mirror`
4. `conditioning`
5. `gene_score`

这些禁止项在本卡内同时适用于：

1. 组合定义
2. 候选排序
3. tie-break
4. 补充 gate
5. 解释性“顺手加一条”

## 6. Boundaries

本卡当前明确不做下面这些事：

1. 不在本卡里扩写 runtime / SQL / 脚本字段映射
2. 不在本卡里发明新别名
3. 不在本卡里定义 package promotion / retention / defer 结论
4. 不在 freeze 之外口头授权额外组合 replay
5. 不因为本卡处于 `Completed`，就自动视为“组合 replay 已获准”

## 7. Formal Outputs

本卡完成时，只允许留下下面 `2` 种正式裁定之一：

1. `no combination replay opened`
2. `open only frozen combinations for replay`

如果正式裁定是：

`no combination replay opened`

则它只意味着：

1. 当前 `17.6` 内部没有组合 replay
2. 当前 `Phase 9C` 不存在 formal combination winner
3. 若后续仍要做组合 replay，必须另开明确 follow-up card

如果正式裁定是：

`open only frozen combinations for replay`

则它只意味着：

1. 后续 replay 只允许发生在本卡已冻结的 `4` 个组合里
2. 不允许在 replay 阶段新增字段、别名、隐藏附加条件或解释性补丁
3. package ruling 必须等待 replay evidence

## 8. Final Ruling

截至 `2026-03-19`，本卡的正式裁定仍然是：

`no combination replay opened`

原因现在应读成：

1. 本卡职责是 `freeze + decision`，不是直接补完 replay
2. 仓库当前不存在任何 formal combination replay evidence
3. `duration_percentile` 的阈值面本身还没有完成 `p65 ~ p95, step 5` sweep
4. 因此当前即使要开 replay，也更适合放进后续更小、更明确的 follow-up card，而不是把 replay 硬塞回本卡

因此当前正式为真的是：

1. `Phase 9C` 当前不存在 formal combination winner
2. frozen combination surface 已被锁死为 `4` 个组合
3. 后续若要 replay，只能在这个 surface 上进行

本轮没有声称的是：

1. 这些组合永远没有价值
2. 后续永远不得新开更小的组合 follow-up
3. `Gene` 已经整体完成 runtime promotion

## 9. Completion Definition

`17.6 / Phase 9C` 只有在下面条件全部写定后，才允许从 `Active` 转为 `Completed`：

1. 组合源字段池仍然严格等于：
   `duration_percentile`、`reversal_state`、`context_trend_direction_before`
2. 允许组合集仍然严格等于本卡已冻结的 `4` 个组合
3. `wave_role`、`current_wave_age_band`、`mirror`、`conditioning`、`gene_score` 仍被正式排除在组合定义、候选排序、tie-break、补充 gate 与解释性补丁之外
4. 本卡没有引入新的 runtime / SQL / 脚本字段别名，也没有引入隐藏规则
5. “是否打开 combination replay” 已被写成单一正式裁定，而不是口头保留
