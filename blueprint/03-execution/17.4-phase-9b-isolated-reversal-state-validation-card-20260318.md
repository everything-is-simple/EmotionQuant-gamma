# Phase 9B / 单变量 reversal_state 验证卡

- 状态: `Completed`
- 日期: `2026-03-18`
- 类型: `单变量 runtime 验证子阶段`
- 范围: `reversal_state 作为 exit-preparation only，单独验证`

## 1. 本轮问题

这张卡只回答一个问题：

`如果只把 reversal_state 以 exit-preparation only 身份接进当前 validated baseline，而且不改 entry backbone、不改 sizing，结果会不会比 baseline 更好？`

## 2. 当前 validated baseline

当前固定不动的 baseline 仍然是：

`legacy_bof_baseline + FIXED_NOTIONAL_CONTROL + FULL_EXIT_CONTROL + Gene sidecar only`

本轮不允许把 `duration_percentile >= 95` 偷带进 runtime。
它在包内已经赢过 isolated round，但不自动并入本轮。

## 3. 为什么下一张是它

当前顺序这样排才诚实：

1. `duration_percentile` 已完成第一轮 isolated win
2. `wave_role` 已完成 isolated validation，但 ruling 是 `retain_sidecar_only`
3. `current_wave_age_band` 仍是 `duration` 的分带读法，当前不应另开成独立卡
4. `reversal_state` 是剩余 `pending candidate` 里，最适合先用最窄语义试探 runtime 的字段
5. `context_trend_direction_before` 更接近父趋势参照字段，当前优先级后置

因此下一张 active 卡固定为：

`17.4 / isolated reversal_state validation`

## 4. 验证边界

本轮只允许改一个主变量：

`add reversal_state exit-preparation semantics to the validated baseline`

这意味着：

1. entry backbone 保持 `legacy_bof_baseline`
2. position sizing 保持 `FIXED_NOTIONAL_CONTROL`
3. default exit contract 仍以 `FULL_EXIT_CONTROL` 为基线
4. Gene 除这一个 `reversal_state` hook 外，继续保持 `sidecar only`

本轮显式禁止：

1. `reversal_state + duration_percentile`
2. `reversal_state + wave_role`
3. `reversal_state + current_wave_age_band`
4. `reversal_state + context_trend_direction_before`
5. 任意 `mirror / conditioning / gene_score` 伴随进入
6. 把 `reversal_state` 直接写成默认 entry hard filter
7. 把 `reversal_state` 直接写成 sizing overlay

## 5. 本轮默认先冻住的唯一候选

本卡开工默认只允许先检验这一条最窄规则：

`reversal_state == CONFIRMED_TURN_DOWN -> exit-preparation only`

这条规则当前只允许：

1. 作用在已有持仓上
2. 作为 `exit-preparation` 或 `de-risk` 语义试探
3. 以单一 rule 进入本轮实现

这条规则当前不允许：

1. 同时兼任 entry filter
2. 同时兼任 exit modulation + sizing modulation
3. 一口气把 `TWO_B_WATCH / COUNTERTREND_WATCH` 也一起打包进来

如果这条最窄规则都站不住，后续再决定是否需要更小 follow-up，而不是本轮开锅。

## 8. 最终裁决

本轮正式 ruling 是：

`promote_reversal_state_exit_preparation`

原因是：

1. `signals_count` 保持 `16` 不变
2. candidate 真实创建并成交了 `9` 笔 `GENE_REVERSAL_PREP` defensive exits
3. `buy_filled_count` 从 `13` 升到 `14`
4. `trade_count` 从 `13` 升到 `14`
5. full-window `expected_value` 从 `-0.0135471571` 升到 `+0.0054696770`
6. `profit_factor / max_drawdown / reject_rate` 同时改善

一句话解释：

`reversal_state` 这轮不是靠少做交易取胜，而是靠更早退出高风险持仓，真实改写了主线路径并赢下了 baseline。`

## 9. 本卡完成不代表什么

即使本卡已经完成，也不代表：

1. `Gene` 已整体进入默认 runtime
2. `Phase 9` 整包已经 closeout
3. `duration_percentile + reversal_state` 现在可以自动组合
4. `17.5 / context_trend_direction_before` 已自动变成 active
5. `17.6 / Phase 9C` 已自动打开
