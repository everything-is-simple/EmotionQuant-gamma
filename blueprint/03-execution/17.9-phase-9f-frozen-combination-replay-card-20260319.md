# Phase 9F / 冻结组合 replay 卡

- 状态: `Planned`
- 日期: `2026-03-19`
- 类型: `formal combination follow-up`
- 范围: `严格只回放 Phase 9C 已冻结的组合面`

## 1. 这张卡回答什么

这张卡只回答一个问题：

`在 duration 轴完成正式 threshold sweep 之后，Phase 9C 已冻结的 4 个组合里，到底有没有组合值得被正式视为 truthful winner？`

---

## 2. Hard Dependency

本卡当前不能直接开跑，依赖于：

1. [`17.6-phase-9c-formal-combination-freeze-card-20260318.md`](./17.6-phase-9c-formal-combination-freeze-card-20260318.md) 已完成
2. [`17.8-phase-9e-duration-percentile-threshold-sweep-card-20260319.md`](./17.8-phase-9e-duration-percentile-threshold-sweep-card-20260319.md) 已完成
3. `17.8` 已明确写出：
   - 哪个 duration threshold candidate 允许继续进入组合 replay
   - 或 duration 轴已不再值得继续进入组合 replay

因此本卡当前状态应读作：

`blocked-by-17.8`

---

## 3. Frozen Allowed Combination Set

本卡只允许 replay `17.6` 已冻结的 `4` 个组合：

1. `duration_percentile + reversal_state`
2. `duration_percentile + context_trend_direction_before`
3. `reversal_state + context_trend_direction_before`
4. `duration_percentile + reversal_state + context_trend_direction_before`

除这 `4` 个之外，不允许：

1. 新增第四字段之外的新字段
2. 偷带 `wave_role`
3. 偷带 `current_wave_age_band`
4. 偷带 `mirror`
5. 偷带 `conditioning`
6. 偷带 `gene_score`

---

## 4. Duration Input Boundary

本卡里的 `duration_percentile` 不允许默认沿用旧的 `p95`。

它只能使用 `17.8` 正式写出来的 duration 结论：

1. 一个候选 threshold
2. 或最多两个明确排序过的候选 threshold

如果 `17.8` 给不出明确 narrowing，本卡不得偷偷开锅跑一大矩阵。

---

## 5. Required Outputs

本卡完成时，必须正式留下：

1. 每个组合 replay 的 evidence JSON
2. 一个组合汇总表
3. 明确写出：
   - 是否存在 `formal combination winner`
   - 若存在，是哪一个
   - 若不存在，明确写 `none`
4. 对 `17.7 / package ruling` 的回灌建议：
   - `promote`
   - `retain`
   - `continue defer`

---

## 6. Explicitly Forbidden

本卡显式禁止：

1. 用隐藏 alias 扩字段
2. 在 replay 中临时补 tie-break
3. 只因为 isolated winners 看起来都不错，就把三字段 bundle 口头当作已验证
4. 跳过单个组合的正式 evidence，而只写摘要结论

---

## 7. Completion Definition

`17.9` 只有在下面条件全部满足后，才允许从 `Planned` 转为 `Completed`：

1. 所有实际打开的组合 replay 都严格属于冻结 surface
2. duration 侧输入严格来自 `17.8`
3. 每个实际 replay 的 evidence JSON 已落地
4. formal combination winner 是否存在已被明确裁定
5. `17.7` 的后续 package reruling 所需输入已写明
