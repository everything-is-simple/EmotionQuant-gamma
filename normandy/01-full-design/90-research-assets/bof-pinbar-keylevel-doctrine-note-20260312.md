## 3. 三套资料真正重合的 entry 语义

### 3.1 关键位先于 K 线

三套资料虽然市场、周期和表达方式不同，但共同点非常稳定：

1. 先有 `关键位 / 水平位 / range boundary / S/R`
2. 再有 `试探 / 突破 / 假突破 / 回抽`
3. 最后才看 `pinbar / rejection bar / failure bar` 这种可视化结果

因此当前更稳的结论是：

`Pinbar 不是 entry 的起点，关键位上的失败行为链才是起点。`

### 3.2 BOF 的核心不是“跌回去了”，而是“错误方向被套”

三套资料共同强调的链路可以压缩为：

1. 价格接近关键位
2. 市场尝试突破
3. 突破未能延续
4. 价格回到原结构
5. 错误方向仓位被迫退出
6. 反向订单流加速价格

这正是当前 `BOF` baseline 的来源层根基。

### 3.3 Pinbar / 合成 Pinbar 更适合当 BOF family 的表达层

从当前资料抽出来，更稳的定位是：

1. `Pinbar` 是 rejection / failure 的 K 线表达
2. `合成 Pinbar` 是多根 K 线合并后的 rejection 表达
3. 它们都更适合作为 `BOF quality branch`，而不是独立替代 `BOF` 的正式 taxonomy

所以当前不该做的事是：

`把 Pinbar 从 BOF 行为链里硬剥出来，单独当一个新的正式 detector 家族。`

## 4. 三套资料真正重合的出场语义

### 4.1 初始止损必须贴着失效点

三套资料虽然细节不同，但都在强调同一件事：

1. 初始风险要尽量小
2. 止损应贴着 `信号 bar 极值 / 结构失效点 / critical point`
3. 这不是为了追求“看起来止损小”，而是为了让交易逻辑一旦失效就尽快认错

换句话说：

`小止损的前提不是勇敢，而是失效点清楚。`

### 4.2 目标位可以预设，但 trailing 必须单向推进

三套资料共同认可：

1. 可以有 `预设目标位 / 第一目标`
2. 也可以有 `分批处理 / 部分实现`
3. 但一旦开始 trailing，止损只能朝盈利方向移动
4. 不能因为主观希望而重新放宽风险

这和当前 Broker 里的固定 `stop_loss + trailing_stop` 语义在方法论上是相容的。

### 4.3 金字塔加仓属于第二层，不属于首轮 alpha 证明

`许佳冲` 和 `YTC` 都允许在顺势环境中做更积极的持仓管理。

但从 Normandy 的问题边界来看：

1. `金字塔加仓` 的前提是初始头寸已经健康
2. 它讨论的是 `持有放大`，不是 `entry provenance`
3. 因此它不该并入当前首轮 `entry alpha` 证明

## 5. A 股日线 + T+1 Open 的翻译边界

### 5.1 不能照搬盘中微观止损

当前必须写死：

1. `YTC / Volman` 里很多漂亮的微观止损，是建立在盘中可即时处理之上的
2. A 股日线 `T+1 Open` 无法原样复制这种 intrabar 反应
3. 因此当前只能保留它们的语义，不直接复制它们的 tick 级动作

### 5.2 语义可保留，执行要翻译

对当前系统更稳的翻译是：

1. 保留 `失效点清楚`
2. 保留 `止损只朝盈利方向移动`
3. 保留 `不要把已被证伪的交易继续拿着`
4. 把触发方式翻译为：`当日收盘检查 -> T+1 Open 执行`

### 5.3 当前 Normandy 应冻结“同一套 Broker 出场语义”

这轮研究更合理的边界是：

1. `MSS / IRS` 暂时拿掉
2. `Broker` 仍然保持同一套退出核心
3. 不在 entry provenance 阶段同时改 entry 和改 exit

这样才可以回答：

`到底是 entry 本身有 alpha，还是只是换了退出讲法。`
