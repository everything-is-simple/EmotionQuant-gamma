# scripts 目录规范（强制）

`scripts/` 用于放置“非系统运行时、但研发/运维必须”的工具。

分类要求：

- `scripts/data/`：数据拉取、导入、修复、迁移、校验工具
- `scripts/backtest/`：回测批处理、参数扫描、批量评估工具
- `scripts/report/`：报告导出、证据汇总、审计生成工具
- `scripts/ops/`：运维脚本（环境检查、目录修复、巡检）

约束：

1. `src/` 是系统运行时；`scripts/` 是工具层。两者职责不能混用。
2. 工具脚本不得成为 `src/` 的强依赖。
3. 新增工具必须放在对应分类目录，禁止散落仓库根目录。
