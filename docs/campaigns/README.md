# 三大战役总图

**状态**: `Active`  
**日期**: `2026-03-12`  
**对象**: `EmotionQuant 三大战役与共用层统一导航`

---

## 1. 定位

本文不是新的设计 SoT。

本文只做一件事：

`把 EmotionQuant 现在的文档世界收成“四块”：三大战役 + 一块共用层。`

以后不再靠记忆判断：

1. 哪些文件属于 `v0.01`
2. 哪些文件属于 `v0.01-plus`
3. 哪些文件属于 `Normandy`
4. 哪些文件只是跨战役共用资产

---

## 2. 四块入口

| 区块 | 路径 | 角色 |
|---|---|---|
| 共用层 | `00-shared/README.md` | 跨战役共用治理、参考、流程与理论来源 |
| 第一战役 | `01-v0.01-frozen/README.md` | `v0.01 Frozen` 历史战役 |
| 第二战役 | `02-v0.01-plus/README.md` | `v0.01-plus` 主线战役 |

---

## 3. 当前一句话口径

1. `v0.01` 是已经打完并冻结的历史战役。
2. `v0.01-plus` 是已经完成 Gate 裁决的主线战役。
3. `Normandy` 是当前继续找 alpha 的研究战役。
4. 跨战役共用的东西统一留在共用层，不再散落在战役目录里冒充当前主路。

---

## 4. 使用顺序

1. 先看本文件，确认自己现在进入哪一场战役。
2. 再进对应战役 `README`，确认该战役的主入口、归档口径和已废弃路线。
3. 最后才进入该战役的 SoT、evidence、records 或研究资产。

---

## 5. 维护规则

1. 新增文档前，先判断它属于哪一场战役。
2. 如果它不属于任何单一战役，就放进共用层。
3. 如果它只是某场战役的旧方案、失败路线或临时计划，就放进该战役自己的 `archive`，不要再堆回共享目录。
4. `campaigns/` 只做导航，不承担设计正文。

从 `2026-03-12` 起，三张战役入口页统一按五栏维护：

1. 战役定位
2. 当前裁决
3. 正式档案入口
4. 已淘汰路线
5. 下一步
