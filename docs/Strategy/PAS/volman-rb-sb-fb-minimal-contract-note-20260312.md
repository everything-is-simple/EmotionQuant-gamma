# Volman RB_FAKE / SB / FB 最小契约说明

**文档版本**：`Draft v0.01`  
**文档状态**：`Active`  
**创建日期**：`2026-03-12`  
**变更规则**：`允许继续细化字段、边界与失败语义；不直接变成当前主线 formal Signal 契约。`

> **定位说明**：本文只服务 `Normandy / N1.5`。  
> 它的目标不是写一套正式线上契约，而是先把 `BOF_CONTROL / RB_FAKE / SB / FB` 四个对象的最小结构语义写清楚，保证后续实验不会一边写 detector 一边改对象定义。

---

## 1. 当前对象

`N1.5` 当前只允许出现四个对象：

1. `BOF_CONTROL`
2. `RB_FAKE`
3. `SB`
4. `FB`

其中：

1. `BOF_CONTROL` 是控制组
2. `RB_FAKE / SB / FB` 是候选组

---

## 2. 统一判定原则

四个对象必须统一遵守下面五条：

1. 都必须依附位置语义，不能脱离关键位或结构背景单独判定
2. 都必须有完整行为链，而不是单根 K 线触发
3. 都必须能写出独立失效点
4. 都必须可改写到 `A 股日线 + T+1 Open` 语义
5. 都不得偷带 `IRS / MSS / account state`

---

## 3. `BOF_CONTROL`

### 3.1 核心语义

`BOF_CONTROL` 固定代表：

`一次已经发生的关键位突破尝试，随后无法延续并重新跌回原结构，由错误方向退出触发反向加速。`

### 3.2 最小行为链

1. 价格接近关键阻力或突破边界
2. 出现向上突破尝试
3. 突破未能获得有效跟进
4. 价格重新回到边界内或边界下方
5. 反向方向获得加速

### 3.3 最小失效语义

下面任一条成立，`BOF_CONTROL` 视为弱或失效：

1. 突破本身不清楚
2. 回落不够明确
3. 没有反向加速痕迹
4. 回落后立刻重新站回突破上方并继续走强

### 3.4 和 `RB_FAKE` 的关系

`BOF_CONTROL` 是当前仓库已存在的 baseline detector。  
`RB_FAKE` 不是它的别名，而是更偏 Volman 语义的候选版本。

---

## 4. `RB_FAKE`

### 4.1 核心语义

`RB_FAKE` 固定代表：

`一段可识别区间边界上的突破尝试，因为缺乏有效 follow-through 或属于假突破路径，最终回撤并形成反向机会。`

### 4.2 最小行为链

1. 先存在可识别的区间或平台边界
2. 区间边线附近出现突破尝试
3. 突破后跟进不足，或迅速回撤
4. 价格重新回到原区间
5. 错误方向仓位退出，形成反向机会

### 4.3 与 `BOF_CONTROL` 的边界

`RB_FAKE` 相对 `BOF_CONTROL` 多一层更强的前置结构要求：

1. 必须先有可识别的 `range / box / boundary`
2. 必须区分 `true breakout / fake breakout / teasing breakout`
3. 不能把所有“冲高回落”都算成 `RB_FAKE`

换句话说：

1. `BOF_CONTROL` 更像当前系统可运行的广义假突破失败
2. `RB_FAKE` 更像 `Volman` 书里的区间边界假突破失败子集

### 4.4 最小失效语义

下面任一条成立，`RB_FAKE` 视为不成立：

1. 前面没有清楚区间边界
2. 突破前没有边界可见性
3. 回撤不够深，仍属真突破延续
4. 更接近从区间中部直冲边线的 `teasing breakout`

---

## 5. `FB`

### 5.1 核心语义

`FB` 固定代表：

`在一段已经爆发的趋势中，第一次有序回撤后，价格在回撤末端恢复原方向的 continuation entry。`

### 5.2 最小行为链

1. 先有清楚的单边推进或趋势爆发
2. 随后出现第一次回撤
3. 回撤深度和节奏保持相对有序
4. 回撤末端出现恢复原方向的触发
5. 原趋势重新延续

### 5.3 与 `PB / BPB` 的边界

`FB` 不能直接等同于泛化 `PB` 或 `BPB`。

它的最小独立性来自：

1. `first pullback` 必须明确
2. 趋势爆发必须先存在
3. 回撤应是顺势恢复前的第一次主要喘息

因此：

1. 不是所有回调恢复都叫 `FB`
2. 第二次、第三次回撤不能偷算 `FB`

### 5.4 最小失效语义

下面任一条成立，`FB` 视为不成立：

1. 前面没有明确爆发趋势
2. 当前回撤并非第一次主要回撤
3. 回撤后没有恢复原趋势
4. 更接近横盘震荡而非单边中的第一次喘息

---

## 6. `SB`

### 6.1 核心语义

`SB` 固定代表：

`在第一次回撤恢复尝试失败后，价格再次回到关键恢复区域，并在第二次恢复时触发的 continuation entry。`

### 6.2 最小行为链

1. 先有趋势背景
2. 第一次恢复尝试失败或未走出来
3. 价格再次回到关键恢复区域
4. 第二次恢复触发出现
5. 前面两次失败累积的 opposite-side pressure 被清算

### 6.3 与 `FB` 的边界

`SB` 绝不能和 `FB` 混写。

最重要的边界是：

1. `FB` 依赖第一次回撤恢复
2. `SB` 依赖第一次恢复没有成功走出来，随后第二次恢复才成立

因此：

1. 如果没有第一次失败，不能叫 `SB`
2. 如果只是更长一点的第一次回撤，也不能偷写成 `SB`

### 6.4 与 `CPB / TST` 的边界

`SB` 可能视觉上和 `CPB / TST / W型恢复` 接近，但当前实验要固定区分：

1. `SB` 的重点是 `first failure -> second attempt -> liquidation`
2. `CPB` 的重点是复杂回调结构本身
3. `TST` 的重点是支撑/阻力测试本身

也就是说：

`SB` 的 alpha 假说重点不是“形状像 W”，而是“第二次失败后的 forced exit 是否更强”。`

### 6.5 最小失效语义

下面任一条成立，`SB` 视为不成立：

1. 看不到第一次失败
2. 第二次恢复不是独立第二次尝试
3. 更像单纯箱体震荡
4. 没有出现第二次恢复后的清算加速

---

## 7. 四者的相对关系

用一句话压缩：

1. `BOF_CONTROL` = 当前系统广义假突破 baseline
2. `RB_FAKE` = Volman 化的区间边界假突破失败
3. `FB` = 首次回撤恢复 continuation
4. `SB` = 第二次失败后的再恢复 continuation

它们不是一层关系：

1. `BOF_CONTROL / RB_FAKE` 更偏 reversal / failure family
2. `FB / SB` 更偏 continuation family

因此后续 matrix 不能只问“谁收益高”，还要问：

`它们到底是在替代 BOF，还是在补 BOF 没覆盖的另一条 alpha 语义。`

---

## 8. 当前一句话契约

`N1.5` 当前只允许把 BOF_CONTROL / RB_FAKE / FB / SB 当成四个边界清楚的实验对象来比较，其中 SB 必须作为首批核心候选保留。
