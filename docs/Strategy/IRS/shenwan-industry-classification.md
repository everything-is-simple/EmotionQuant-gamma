## 1. 文档定位

本文档是 `IRS` 的**行业口径参考资料**，作用是回答三件事：

1. 当前仓库使用哪一套行业分类标准。
2. 个股与行业之间的正式映射应以什么资料为准。
3. 行业分类资料在数据层应如何维护、更新和审计。

本文档**不是当前主线执行 SoT**。当前可执行设计以 `blueprint/` 为准，当前主线固定链路仍是：

`Selector 初选 -> BOF 触发 -> IRS 排序 -> MSS 控仓位 -> Broker 执行`

其中 `IRS` 在当前主线里只承担**行业级后置排序增强**，不承担前置硬过滤，也不承担市场级仓位调节。

---

## 2. 当前主线采用边界

当前主线只把本文用于以下三类事项：

1. 确认正式行业口径为 `SW2021 一级行业`。
2. 维护 `个股 -> 行业` 映射和行业成分变更。
3. 审计行业样本覆盖、名称一致性和历史成员变动。

当前主线**不直接采用**本文中的以下内容作为执行规则：

1. 行业生命周期分类与 `PE / PB` 权重映射。
2. 牛熊阶段下的行业配置建议。
3. 历史轮动经验总结。
4. 任意“某阶段应超配/减配某行业”的静态表。

这些内容仍有研究价值，但只保留为**研究性观察**，不构成当前 `IRS` 算法正文。

## 3. 核心资料与正式口径

### 3.1 基础资料

- **分类标准**：申万行业分类（Shenwan Industry Classification）
- **正式版本**：`SW2021`
- **使用层级**：一级行业
- **核心文件**：
  - `SwClassCode_2021.xls`
  - `StockClassifyUse_stock.xls`
  - `最新个股申万行业分类(完整版-截至7月末).xlsx`

### 3.2 为什么当前主线只使用一级行业

当前主线固定使用一级行业，原因是：

1. 一级行业样本容量更稳定，适合日频横截面排序。
2. 二级、三级行业在 A 股环境下更容易出现样本过小和流动性不足问题。
3. `IRS` 当前主线目标是“最小可交易排序层”，不追求过度细分。

### 3.3 正式映射口径

当前主线中，行业正式映射应优先来自：

1. `index_classify`
2. `index_member`
3. 本地维护的 `l1_sw_industry_member`

以下资料**不应再被当作正式行业归属真相源**：

1. `stock_basic.industry`
2. 随手整理的行业简称表
3. 任何未注明版本的“旧申万分类”

原因很直接：这些来源容易偏离 `SW2021` 正式行业成员口径，导致行业桶数量异常或归属漂移。

## 5. 个股行业归属规则

### 5.1 基本原则

当前参考口径保留以下三条归属原则：

1. **主营业务原则**：个股应归属到主营业务最主要的一级行业。
2. **动态调整原则**：行业归属不是永久不变，需跟随官方成员调整维护。
3. **唯一归属原则**：当前主线一只股票只归属一个一级行业。

### 5.2 特殊情况

以下处理方式保留为数据维护参考：

| 情况 | 处理建议 |
|---|---|
| `ST` 股票 | 保留行业归属，但统计时可按策略规则选择排除 |
| 新股 | 上市后按正式行业归属维护；样本窗口不足时可在排序层排除 |
| 停牌股票 | 保留行业归属，但当日统计时不计入有效交易样本 |
| 退市股票 | 从成员关系中移除，不再参与行业统计 |
| 多元化公司 | 以正式口径为准，不在本地重复造“复合行业” |

## 6. 数据维护建议

### 6.1 推荐数据接口

申万分类与成员维护优先使用以下接口：

```python
# 获取申万行业分类
df = pro.index_classify(level="L1", src="SW2021")

# 获取行业成分股
df = pro.index_member(index_code="801010.SI")

# 获取行业指数日线
df = pro.index_daily(
    ts_code="801010.SI",
    start_date="20200101",
    end_date="20241231",
)
```

### 6.2 维护频率

建议保持以下更新节奏：

1. **行业成员**：按季度或官方口径变更后更新。
2. **行业指数日线**：按交易日更新。
3. **名称与代码映射**：每次成员更新后做一次一致性审计。

### 6.3 当前仓库中的落地边界

当前主线可接受的职责分工是：

1. `L1` 保存正式行业成员和行业指数原始数据。
2. `L2` 聚合行业日快照，不发明新的行业口径。
3. `L3` 只消费行业结果，不回写行业归属定义。
