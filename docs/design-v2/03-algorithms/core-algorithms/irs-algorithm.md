## 1. 定位与理论基础

### 1.1 系统定位

在 v0.01 中，IRS 的职责是：

`给 Selector 提供行业相对优先级。`

它不负责：

- 判断市场该不该做
- 判断个股今天买不买
- 直接输出交易动作

### 1.2 理论来源

IRS 的核心理论来自申万行业分类体系和行业轮动实践：

1. **申万行业分类标准（SwClass）**（`docs/Strategy/IRS/shenwan-industry-classification.md`）
   - SW2021 一级 31 行业标准
   - 行业生命周期分类（周期/消费/成长/金融）
   - 行业轮动规律（经济周期驱动、政策驱动、资金驱动、技术驱动）

**核心设计原则**：
- 行业分类标准化（申万一级为唯一事实源）
- 相对强度优先（行业 vs 基准）
- 资金流向验证（量价配合）
- 不依赖宏观叙事（纯数据驱动）

### 1.3 行业轮动四驱动力映射

| 理论驱动力 | 对应因子 | 观测指标 |
|-----------|---------|---------|
| 经济周期驱动 | RS（相对强度） | 行业涨跌幅 vs 基准 |
| 资金驱动 | CF（资金流向） | 成交额变化、换手率 |
| 政策驱动 | （v0.02+） | 政策事件、行业新闻 |
| 技术驱动 | （v0.02+） | 龙头股强度、牛股基因 |

**v0.01 范围**：仅实现前两个驱动力（RS + CF）

## 2. 正式行业口径

自 `2026-03-06` 起，v0.01 正式行业口径固定为：

`SW2021 一级 31 行业`

运行时优先级为：

1. `l1_sw_industry_member`（申万行业成分股表）
2. 无匹配时记为 `未知`

解释：

- `stock_basic.industry` 不是申万一级事实源
- 只有 `index_classify + index_member` 链路才是当前系统认可的正式行业口径
- `未知` 只允许作为 L2 聚合残余桶存在，不允许进入 IRS 正式排名

## 3. 输入与数据边界

IRS 当前读取：

- `l2_industry_daily`（行业日线聚合数据）
- `l1_index_daily`（基准指数，默认 `000001.SH` 上证指数）

`l2_industry_daily` 的正式生成方式为：

- 先按 `l1_sw_industry_member` 确定个股所属申万一级行业
- 无 SW 匹配时归入 `未知`
- 再把个股日线聚合到行业日线
- 非交易日不进入 `L2/L3`

## 4. 当前算法定义

### 4.1 两因子框架

当前 v0.01 只保留两因子：

1. **RS（相对强度，55%）**
   - 公式：`industry_pct_chg - benchmark_pct_chg`
   - 含义：行业相对大盘的超额收益
   - 对应"经济周期驱动"

2. **CF（资金流向，45%）**
   - 公式：`flow_share + amount_delta_10d`
   - 含义：资金流入强度 + 成交额变化趋势
   - 对应"资金驱动"

### 4.2 权重设计

- `RS = 55%`
- `CF = 45%`

综合得分：

```text
total_score = 0.55 * rs_score + 0.45 * cf_score
```

**权重设计原则**：
- 相对强度优先（趋势跟随）
- 资金流向验证（量价配合）
- 两者结合避免单一维度误判

### 4.3 排名规则

当前排序固定为：

1. 按 `score` 降序
2. 若分数相同，按 `industry` 升序
3. 最终输出唯一排名 `1..N`

这条规则属于 v0.01 正式契约：

`排名必须唯一、稳定、可复现。`

## 5. 日级有效性契约

IRS 自 `2026-03-06` 起新增日级有效性约束：

1. 非交易日不得进入 `l2_industry_daily`
2. `industries_per_day < IRS_MIN_INDUSTRIES_PER_DAY` 时，当日 IRS 视为无效，不写入 `l3_irs_daily`
3. 缺基准指数时，当日评分仍可算，但必须在证据中标注

当前默认：

- `IRS_MIN_INDUSTRIES_PER_DAY = 25`

## 6. 当前已知现实问题

### 6.1 旧实现曾误用 111 桶

Week2 审计暴露出旧实现曾直接使用 `stock_basic.industry`，形成 `111` 个现实桶。

该问题已被纠正：

- 当前运行链路已切回申万一级优先
- `111` 桶不再是正式设计口径

### 6.2 SW31 主链已收口

当前真实状态已经变成：

- raw 源库已补采并清洗到 `31` 个申万一级行业
- `l1_sw_industry_member` 已恢复到 `31` 行业覆盖
- `l3_irs_daily` 已回到 `31` 个正式行业桶
- `未知` 不再进入 IRS 排名

因此，IRS 的主阻塞已从"行业口径错误"转移到"过滤强度待验证"。

### 6.3 过滤强度仍待校准

即使口径改回申万一级，IRS 仍需重新验证：

- `IRS_TOP_N=10` 是否过窄
- `+IRS` 是否会把机会压空

## 7. v0.01 正式方向

IRS 在 v0.01 的正式方向不是恢复旧版 6 因子全集，而是：

`先把两因子 IRS 做成现实可用、口径正确、可消融的行业过滤器。`

这意味着优先级必须是：

1. 行业口径正确（SW31 已完成）
2. 数据覆盖可信（已完成）
3. 排名契约稳定（已完成）
4. 过滤强度合理（待验证）

而不是一边口径没收口，一边继续加新因子。

## 8. 版本演进路径（v0.01-v0.06）

### 8.1 v0.01：两因子 + Top-N 硬过滤

**当前状态**：
- RS + CF 两因子
- 固定权重（55/45）
- Top-N 硬过滤（默认 Top 10）
- 申万一级 31 行业

**已知问题**：
- Top-N 可能过窄
- 硬过滤误杀机会
- 两因子信息量有限

### 8.2 v0.02：软评分模式 + 龙头强度

**计划改进**：
- IRS 从硬过滤改为软评分（参考 `down-to-top-integration.md`）
- 新增龙头强度因子（行业内涨停股占比、行业内新高股占比）
- IRS 作为后置评分项，不再前置过滤
- Top-N 敏感性测试（10/15/20）

**验收标准**：
- soft_score 模式改善 baseline 指标
- 龙头强度因子 IC >= 0.05
- trade_count >= 60

### 8.3 v0.03：牛股基因聚合

**计划改进**：
- 新增牛股基因聚合因子（行业内"5牛5衰"分布）
- 基于历史交易样本反推行业特征
- Gene 仅作为事后分析，不前置过滤
- 行业轮动状态识别（强势/平衡/弱势）

**验收标准**：
- Gene 聚合因子 IC >= 0.03
- 行业轮动状态识别准确率 >= 60%
- 不同状态的策略表现差异显著

### 8.4 v0.04：行业生命周期

**计划改进**：
- 新增行业生命周期判定（成长/成熟/衰退）
- 基于长期趋势 + 估值水平 + 盈利增速
- 不同生命周期调整权重
- 行业轮动预测（提前1-2周）

**验收标准**：
- 生命周期判定准确率 >= 55%
- 轮动预测命中率 >= 40%
- 提前布局改善收益

### 8.5 v0.05：政策事件驱动

**计划改进**：
- 新增政策事件因子（行业政策、监管变化）
- 基于新闻文本分析 + 事件日历
- 政策敏感行业加权
- 事件驱动快速响应

**验收标准**：
- 政策事件识别准确率 >= 50%
- 事件驱动交易胜率 >= 55%
- 政策敏感行业表现改善

### 8.6 v0.06：自适应权重

**计划改进**：
- 因子权重不再固定，基于滚动窗口 IC 动态调整
- 不同市场环境（牛/熊/震荡）使用不同权重
- 引入机器学习优化权重
- 行业轮动策略组合

**验收标准**：
- 自适应权重相对固定权重改善 EV >= 10%
- 不同环境权重差异显著
- 权重调整逻辑可解释

## 9. 下一步校准路径（v0.01 → v0.02）

### 9.1 行业口径固定

当前行业口径已固定为 SW2021 一级 31 行业，后续只允许：

- 申万官方更新时同步更新
- 成分股调整时同步更新
- 不允许回退到其他分类标准

### 9.2 过滤强度校准

下一步必须重跑并比较：

- `Top 10`
- `Top 15`
- `Top 20`

并观察：

- 候选池大小
- 信号数量
- 回测结果（EV/PF/MDD）
- 是否仍可交易

### 9.3 软硬开关决策

在过滤强度校准后，才进入：

- 软评分模式（IRS 作为后置评分项）
- 龙头强度因子
- 牛股基因聚合

因为这一步讨论的前提已经从"先收口行业分类"变成了"先确认 Top-N 是否合理"。

## 10. 权威结论

当前 v0.01 的 IRS：

- 两因子框架成立（RS + CF）
- 排名契约成立（唯一、稳定、可复现）
- 正式行业口径已收回到 `SW2021 L1`（31 行业）
- 执行链路已经恢复（l1_sw_industry_member → l2_industry_daily → l3_irs_daily）

但还未完成：

1. `Top N` 强度重新校准
2. `+IRS` 消融收益验证

所以 IRS 的下一步不是扩因子，而是：

`围绕正式 SW31 链路重新跑 +IRS 消融，再决定 Top N 是否放宽（v0.02）。`
