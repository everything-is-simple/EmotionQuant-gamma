# EmotionQuant 沙盘评审标准（v0.01）

**状态**: Active（v0.01 执行期）  
**生效日期**: 2026-03-03  
**适用范围**: Data / Selector / Strategy / Broker / Backtest / Report 全链路

---

## 1. 目的

把沙盘模拟从“经验排查”升级为“可重复执行的评审流程”，避免单点修复后在调用链其他位置再次失效。

本标准是 `rebuild-v0.01.md` 的配套评审规范；若与主文档冲突，以 `rebuild-v0.01.md` 为准。

---

## 2. 七维评审框架（固定口径）

### 2.1 Schema 契合（骨骼）

检查 DDL、SQL 查询、契约对象（pydantic）三方字段是否一致：字段名、类型、非空约束、语义。

### 2.2 调用链完整性（神经）

检查深层调用链的参数签名与传递是否完整，重点验证 `today_date`、`broker_state`、`execute_date` 等时序关键参数。

### 2.3 幂等与确定性（免疫）

检查重跑是否产生重复写入。ID 必须确定性，写入必须 upsert。

### 2.4 状态机完整性（内分泌）

检查状态迁移是否可达、可回退、无死路径。重点关注信任分级 `ACTIVE/OBSERVE/BACKUP`。

### 2.5 时序语义（心跳）

检查全链路是否严格满足 `signal_date=T -> execute_date=T+1 -> price=T+1 open`，并显式声明回测末日强平例外。

### 2.6 边界冲突与去重（排异）

检查同日同标的多规则同时触发时是否冲突，是否可能生成重复 SELL、负仓位或重复撮合。

### 2.7 报告口径真实性（体检）

检查统计指标是否基于净收益（含手续费与滑点），配对逻辑是否支持分批成交/分批平仓。

---

## 3. 证据模板（每个偏差必须填写）

每个偏差必须包含以下 5 项：

1. 触发条件（什么输入会触发）
2. 失败表现（报错/错误行为）
3. 根因定位（具体函数与文件行）
4. 修复点（改动的函数、字段或断言）
5. 回归断言（如何证明修复有效且不回归）

---

## 4. 风险分级（固定）

1. `S0-阻断`：会导致崩溃、未来函数、数据污染、状态机失效
2. `S1-高危`：结果显著偏差，可能误导交易决策
3. `S2-中危`：局部功能失效或可观测偏差
4. `S3-低危`：文档/注释/可维护性问题

处置要求：

1. `S0/S1` 必须在定稿前清零
2. `S2` 必须有明确 owner 与截止日期
3. `S3` 可进入后续迭代，但需登记

---

## 5. 定稿门禁（Go/No-Go）

满足以下条件才可判定“可定稿”：

1. 七维检查全部完成，且有证据记录
2. `S0/S1 = 0`
3. 三组消融链路完整可复现：`BOF baseline -> BOF+MSS -> BOF+MSS+IRS`
4. 幂等重跑通过（同输入重跑不新增脏数据）
5. T+1 语义与末日强平例外在文档中显式并列
