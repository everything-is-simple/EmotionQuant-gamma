## 交易所概览

中国 A 股主要交易所包括：

| 交易所 | 简称 | 常见板块 |
|---|---|---|
| 上海证券交易所 | `SSE` | 主板、科创板 |
| 深圳证券交易所 | `SZSE` | 主板、创业板 |
| 北京证券交易所 | `BSE` | 北交所市场 |

说明：

1. 不同交易所 / 板块的涨跌停、新股限制与细则可能不同。
2. 仓库中做交易规则判断时，必须区分板块类型，而不能只看股票代码前缀的粗略经验。

## 标准交易时段参考

常见交易时段如下：

| 时段 | 时间 | 说明 |
|---|---|---|
| 开盘集合竞价 | `09:15 - 09:25` | 含可撤单阶段与不可撤单阶段 |
| 上午连续竞价 | `09:30 - 11:30` | 连续撮合 |
| 午间休市 | `11:30 - 13:00` | 无交易 |
| 下午连续竞价 | `13:00 - 14:57` | 连续撮合 |
| 收盘集合竞价 | `14:57 - 15:00` | 收盘价撮合 |

说明：

1. 具体细则以交易所当期规则为准。
2. 仓库若只做日线 / 低频系统，重点是理解“信号生成日”和“可成交执行日”的边界，而不是盘中分秒级撮合细节。

### TuShare / 交易日历参考

低频系统最关键的外部规则之一是“交易日历”。

示意：

```python
import tushare as ts

pro = ts.pro_api("YOUR_TOKEN")
df_trade_cal = pro.trade_cal(exchange="SSE", start_date="20250101", end_date="20251231")
```

仓库使用建议：

1. 所有 `T+1` 推进应基于交易日历，而不是自然日。
2. 节假日、周末、停牌都会影响“下一交易日”的判断。

## T+1 制度参考

A 股现货股票常见规则是：

1. `T` 日买入
2. 最早 `T+1` 才能卖出

仓库内的固定执行语义是否进一步约束为 `T 日收盘信号 -> T+1 Open 执行`，属于系统设计问题，应以：

1. `blueprint/`
2. `docs/design-v2/01-system/system-baseline.md`

为准，而不是由本文决定。

## 停牌与不可成交场景参考

在交易制度层面，仓库实现时通常需要关注：

1. 停牌
2. 一字涨停导致买入不可成交
3. 一字跌停导致卖出不可成交
4. 无下一交易日

这些属于“外部制度边界”，但系统如何落成：

1. `REJECTED`
2. `EXPIRED`
3. 顺延
4. 强平

是实现设计问题，不由本文直接定义。

## 集合竞价参考

集合竞价是开盘和收盘阶段的撮合制度。

对低频系统真正重要的是：

1. 开盘价和收盘价是特殊价格点
2. 若系统固定使用 `T+1 Open` 执行，则开盘价有效性很关键
3. 回测若简化为“次日开盘成交”，必须明确这是一种系统执行语义，而不是交易所全文规则的完整复制
