## 设计要点

### 组合硬约束

- 日/周/总在险 R 预算
- 行业与主题暴露上限
- 现金底线
- 硬约束不满足时直接拒单，不进入个股风控

### 动态降仓

- 按回撤和连亏分级调整开仓系数
- 必须定义触发条件、恢复条件、冷却窗口
- 降仓逻辑不改 `T+1 Open` 撮合语义

### 冲突优先级（固定）

1. 时序守卫（T+1）
2. 组合硬约束（预算/暴露/现金）
3. 动态降仓（系数调整）
4. 个股风控（已有 risk.py）
