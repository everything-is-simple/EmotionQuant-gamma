## 使命

单策略能赚钱不等于组合能活。组合层解决：**买多少、暴露多少、什么时候该收手**。

一句话：**总风险预算 + 暴露限制 + 动态降仓 + 候选池分层，让系统从"单笔交易正确"进化到"整体组合可持续"。**

## 定位说明

v0.06 是 **Type-B 核心功能阶段**：在 Broker/Risk 之上新增组合层（Portfolio Layer），不替换现有 risk.py 的个股级风控，而是叠加组合级约束。这是系统从"单笔交易系统"向"完整投资组合管理系统"的关键跃迁。

## 设计要点

### 总风险预算

v0.01–v0.05 的风控是个股级的（单笔 0.8% 账户风险 + 单只上限 10%）。v0.06 新增组合级约束：

| 约束 | 默认值 | 说明 |
|------|--------|------|
| 每日 R 限额 | 3R | 单日新开仓总风险不超过 3 个 R 单位 |
| 每周 R 限额 | 8R | 单周累计新开仓总风险不超过 8 个 R 单位 |
| 总在险 R | 15R | 全组合同时在险总 R 不超过 15 |
| 现金最低比例 | 20% | 永远保留至少 20% 现金（极端市场自我保护） |

当日 R 额度用完 → 后续信号排队到次日。周 R 额度用完 → 本周停止开仓。

### 行业 / 主题暴露限制

| 约束 | 默认值 | 说明 |
|------|--------|------|
| 单行业最大持仓 | 3 只 | 同一申万一级行业最多持有 3 只 |
| 单行业最大暴露 | 30% NAV | 同一行业持仓市值不超过 NAV 的 30% |
| 单主题最大暴露 | 20% NAV | 同一概念板块（如"人工智能"）不超过 NAV 的 20% |

暴露超限时：新信号被 REJECTED（原因 `EXPOSURE_LIMIT`），不影响已有持仓。

### 仓位计算升级

v0.01–v0.05 使用固定 R 风险仓位（`quantity = account_risk / (entry - stop_loss)`）。v0.06 新增分数凯利模式：

| 模式 | 公式 | 适用场景 |
|------|------|----------|
| R_FIXED（默认） | `qty = risk_per_trade / (entry - stop)` | 保守模式 |
| HALF_KELLY | `qty = 0.5 × kelly_fraction × account / entry` | 激进模式 |

kelly_fraction 基于该形态最近 100 笔的胜率和盈亏比实时计算。两种模式取**较小值**作为最终仓位（凯利提供进攻上限，R 风险提供硬上限约束）。

### 动态降仓

| 触发条件 | 行动 | 恢复条件 |
|----------|------|----------|
| 组合回撤 10%–15% | 新开仓 R 减半 | 净值恢复至回撤前 95% |
| 组合回撤 15%–20% | 停止开仓（不清仓） | 净值恢复至回撤前 90% |
| 组合回撤 > 20% | 清仓 + 冷却（同 v0.01 §5） | 冷却期结束 |
| 连亏 ≥ 3 笔 | 新开仓 R 减半 | 首笔盈利 |
| 连亏 ≥ 5 笔 | 停止开仓 3 天（同 v0.01 §5） | 冷却期结束 |

注：v0.01 的第二级（15% 全清仓）和第三级（5 连亏暂停）被 v0.06 的梯度降仓替代，但极端情况下仍保留硬清仓。

### 组合层冲突优先级（实现前固定）

同日多约束并发触发时，按以下顺序处理，避免重复拒单或语义冲突：

1. 交易时序守卫：先检查 `signal_date=T -> execute_date=T+1 -> price=T+1 Open` 是否满足。
2. 组合硬约束：总在险 R / 行业暴露 / 现金底线（不满足直接 REJECTED）。
3. 动态降仓：在可开仓前提下再调整仓位系数（减半或暂停开仓）。
4. 个股风控：最后进入既有 `risk.py` 个股级检查与下单。

### 候选池分层（视角四）

候选池从"平坦列表"变为"分层列表"，用于量化"拥挤度"：

| 分层维度 | 分层方式 | 说明 |
|----------|----------|------|
| `liquidity_tier` | L1 / L2 / L3 | 按 20 日平均成交额分位（Top 10% = L1, 10–40% = L2, 40%+ = L3） |
| `attention_tier` | HOT / MID / COLD | 指数成分 + 成交额 Top 分位 + 行业热度代理 |
| `scan_batch` | A / B / C | 每日扫描 A+B（高关注度），周末扫描 C（低关注度） |

分层目的：
- 验证"中腰部（L2/MID）是否比头部（L1/HOT）更适合低频结构交易"
- 产出"拥挤度—表现"对照表

## 验收标准

1. R 限额生效：单日超 3R 后的信号被排队到次日（单测验证）
2. 行业暴露限制生效：同行业第 4 只信号被 REJECTED（单测验证）
3. 动态降仓梯度正确：10% 回撤 → R 减半，15% → 停止开仓，20% → 清仓
4. 分数凯利仓位 ≤ R 风险仓位（取较小值，单测验证）
5. 候选池分层标签正确（L1/L2/L3 按成交额分位，单测验证）
6. `l4_tier_stats` 输出不同 tier 的信号数/摩擦/胜率/期望值
7. 组合风控报告输出：暴露分布、R 使用率、降仓记录
8. 全链路回测在新组合层约束下可运行且指标合理
9. 七维评审 S0/S1 = 0

## 评审证据与 Go/No-Go 门禁（补充）

除 S0/S1 清零外，v0.06 还必须提交：

1. 七维证据表：重点覆盖“边界冲突去重、调用链完整性、时序语义”三维。
2. 连环失效回放：至少 1 条“组合预算 + 动态降仓 + 个股风控” kill-chain 回放通过。
3. 幂等重跑结果：同输入重跑两次，`l4_portfolio_snapshot/l4_tier_stats/l4_position_control_log` 主键集合一致。
4. 并发优先级回归：验证“组合 REJECTED 不会再进入个股重复拒单”。
5. Go/No-Go 结论：显式给出“可进入 v0.07 规划 / 不可进入”。

## 后续展望（v0.07+）

v0.06 完成后，系统具备完整的"信号生成 → 风控 → 组合管理 → 监控 → 进化"闭环。后续方向视运行数据而定，可能包括：

- **纸上交易转实盘对接**（券商 API 接入）
- **GUI 可视化仪表盘**（Streamlit）
- **多周期协同**（日线 + 周线信号融合）
- **机器学习辅助**（特征筛选 / 参数优化，不替代规则引擎）

以上均不预先展开，待 v0.06 评审通过后根据运行数据决定。

## 已知风险

| 级别 | 风险 | 缓解措施 |
|------|------|----------|
| 高 | R 限额/暴露限制过严导致信号大量排队 | 参数可配置，消融对照确定最优值 |
| 高 | 分数凯利在极端市场下仓位波动大 | 凯利仅作上限，R 风险兜底 |
| 中 | 动态降仓恢复条件可能导致"震荡区反复触发" | 恢复条件设滞后（95%/90%），避免频繁切换 |
| 中 | 候选池分层增加选股复杂度 | 分层仅标签，不改变核心漏斗逻辑 |
| 低 | 主题暴露限制需要维护概念板块映射 | 用申万行业分类替代，不额外维护概念板块 |
