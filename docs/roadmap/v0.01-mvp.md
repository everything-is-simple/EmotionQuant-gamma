## 版本晋级门槛

引用自 `docs/design-v2/rebuild-v0.01.md` §7 + §6.2，所有版本升级必须同时满足：

1. 当前版本连续两个评估窗通过 §6.2 通过阈值（expected_value≥0, profit_factor≥1.05, max_drawdown≤25%, trade_count≥60）且无强制回退。
2. 新增形态在单形态回测中先独立通过，再进入组合评估。
3. 任何新增模块不得破坏 `T+1 Open` 执行语义与结果契约字段冻结规则。
4. 新漏斗（如 +MSS、+IRS、+GENE）相对前一配置触发回退条件时（EV 下降>10%、max_dd 恶化>20%、任一环境中位数路径由正转负且连续两个评估窗未恢复）必须回退。
5. 七维评审 S0/S1 = 0（`docs/design-v2/sandbox-review-standard.md` §6）。

## 跨版本评审证据落点（统一门禁）

所有 v0.02+ 版本在“七维评审 S0/S1=0”之外，必须补齐以下证据并可追溯：

1. 七维证据清单：Schema/调用链/幂等/状态机/时序/冲突去重/报告真实性各至少 1 组可复现证据。
2. 连环失效回放：至少 1 条 kill-chain 组合场景（2+偏差联动）回放通过。
3. 幂等重跑证明：同输入重跑两次，关键表主键集合一致且无新增脏数据。
4. 证据模板完整：每个偏差项按 `sandbox-review-standard.md` §4 的 8 项模板记录。
5. Go/No-Go 结论：是否满足 §6 定稿门禁，必须有明确结论与日期。

## 关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 回测引擎 | backtrader 单引擎 | 旧版三引擎都没实现，只留一套做到底 |
| 存储 | DuckDB 单库 | 替代旧版L1-L4四层门禁架构 |
| 数据源 | TuShare主 + AKShare备 | 简化旧版双TuShare通道 |
| 执行策略 | YTC/Volman形态检测 | 替代旧版打板/短线战法 |
| IRS 第1迭代 | 2因子（相对强度+资金流向） | 信号最强的2个，其余后续加回 |
| GUI | 延后 | 第1迭代命令行足够 |
| 赛道选择 | 低频形态触发 | 避开主流高频/拥挤因子赛道的算力与速度绞杀 |
