## 设计要点

### TST 支撑/阻力测试检测器

TST 包含两种子形态：
- **DD（Double Doji / 双十字星）**：价格在支撑位附近形成双十字星，暗示卖压枯竭
- **SB（Second Breakout / 二次突破）**：首次突破后回落，二次突破确认有效

触发条件：
1. 明确的支撑/阻力位（由 CriticalPointManager 提供）
2. 价格到达支撑位附近（±1% 容差）
3. DD 子形态：连续 2 根 K 线实体 < ATR × 0.3
4. SB 子形态：首次突破失败后再次突破，第二次突破量能 ≥ 首次

### PB 趋势回调检测器

趋势中的标准回调买入：
1. 明确趋势（MA20 方向一致 ≥ 10 日）
2. 回调至 MA20 附近（±2% 容差）或 0.382–0.618 回撤区间
3. 回调量能萎缩（回调段成交量 < 趋势段均值 × 0.7）
4. 出现反转 K 线（阳包阴 / 锤子线 / 十字星）

### CPB 复杂回调检测器

CPB 包含两种子形态：
- **SB+ARB（Springboard + Anti-Rally Base）**：多次回调后形成平台，平台突破
- **M/W 形态 + 头肩底**：经典技术形态适配

触发条件：
1. 回调波数 ≥ 2（至少两次下探）
2. 各波低点在容差范围内形成支撑带
3. 最后一波出现量能放大确认
4. W 底 / 头肩底 / 三重底等经典形态匹配

### CriticalPointManager（Volman 第14章）

临界点动态管理核心：
- **维护临界点列表**：自动识别近期支撑/阻力位（swing high/low + 成交密集区）
- **临界点强度评分**：触碰次数 × 时间衰减 × 成交量确认
- **临界点状态转换**：ACTIVE → TESTED → BROKEN → FLIPPED
- TST/PB/CPB 的触发判定需引用 CriticalPointManager 提供的临界点
- **边界约束**：CriticalPointManager 只参与 Strategy 内部判定，不改变跨模块结果契约字段

### 不利条件预警（Volman 第15章）

在信号生成前检查是否存在不利条件，若存在则降低 strength 或直接跳过：
- **阻力位压制**：做多信号上方有强阻力（临界点强度 ≥ 阈值）
- **冲击效应**：突然的大幅波动后信号可靠性下降
- **真空效应**：快速拉升后无支撑，回调风险高

### IRS 6 因子扩展

| 因子 | 权重 | 说明 |
|------|------|------|
| 相对强度 | 30% | industry_pct_chg - benchmark_pct_chg |
| 资金流向 | 25% | 10 日滚动净流入 + 资金占比 |
| 连续性 | 15% | 行业连续跑赢基准天数比例 |
| 估值 | 10% | 行业 PE 分位 |
| **龙头效应** | 10%（新增） | 行业内龙头股涨跌幅领先度 |
| **基因库因子** | 10%（新增） | 行业内匹配"牛股画像"个股占比 |

权重调整须经消融对照验证。基因库因子仅在 `ENABLE_GENE_FILTER=True` 时生效。

### PAS_COMBINATION 三种组合模式

| 模式 | 逻辑 | 适用场景 |
|------|------|----------|
| ANY | 任一形态触发即入场（取 strength 最高者） | 信号覆盖率优先 |
| ALL | 全部活跃形态同日触发才入场 | 精确度优先（极端保守） |
| VOTE | 加权投票超过阈值才入场（默认 ≥ 0.6） | 平衡模式 |

v0.03 消融对照须分别评估三种模式 vs 单形态基线。

## 验收标准

1. `python main.py backtest --patterns=tst` / `--patterns=pb` / `--patterns=cpb` 各自独立回测可运行
2. `python main.py backtest --patterns=bof,bpb,tst,pb,cpb --combination=VOTE` 五形态组合回测可运行
3. 每种形态单形态回测输出同口径指标，可与 BOF/BPB 直接对比
4. CriticalPointManager 输出临界点列表，可视化验证（抽样 10 只股票人工检查）
5. 不利条件预警可降低 strength（抽样验证命中率）
6. PAS_COMBINATION 三种模式各有独立回测结果
7. 报告新增：多形态重叠率统计、临界点命中率统计
8. IRS 6 因子 vs 4 因子消融对照完成
9. 输出信号拥堵度指标（同日 Top-N 强度分布 + 同票同日多形态重叠率）
10. 七维评审 S0/S1 = 0

## 评审证据与 Go/No-Go 门禁（补充）

除 S0/S1 清零外，v0.03 还必须提交：

1. 七维证据表：至少覆盖“多形态并发触发”“组合模式合并”“风控并发 SELL 去重”三类场景。
2. 连环失效回放：至少 1 条“ANY/ALL/VOTE + RiskManager + Matcher” kill-chain 回放通过。
3. 幂等重跑结果：同一区间执行两次五形态组合，`l3_signals/l4_orders/l4_trades` 主键集合一致。
4. 状态机一致性证据：若引入临界点状态机，需提供“状态可达 + 无死路径”回归结果。
5. Go/No-Go 结论：显式给出“可晋级 v0.04 / 不可晋级”。

## 已知风险

| 级别 | 风险 | 缓解措施 |
|------|------|----------|
| 高 | 五形态同日重叠信号导致过度交易 | VOTE 模式 + 组合阈值控制 |
| 高 | CriticalPointManager 计算量大（全市场 × 每日更新） | 仅对候选池计算，非全市场 |
| 中 | CPB 子形态（M/W/头肩底）识别准确率低 | 严格几何约束 + 人工抽样验证 |
| 中 | 消融矩阵组合爆炸 | 优先对照关键维度，非穷举 |
| 低 | 不利条件预警过度过滤导致信号不足 | 预警仅降低 strength，不硬拒 |
