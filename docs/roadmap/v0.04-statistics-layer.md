## 使命

把脑子里模糊的"感觉"全部变成**统计与规则**。

一句话：**用更细的环境分桶、执行摩擦统计和风险归因，让系统从"我觉得行"变成"数据告诉我行不行"。**

## 定位说明

v0.04 是 **Type-A 监控覆盖层**：不新增交易逻辑，不改变信号生成或执行语义，仅在现有 report 模块上叠加更深入的统计分析。所有新增内容均为"读数据 → 算指标 → 出报告"，不影响回测/交易链路。

## 设计要点

### 环境分桶矩阵深化（视角二）

v0.01–v0.03 仅有 MSS 三挡（BULLISH/BEARISH/NEUTRAL）。v0.04 深化为多维分桶：

| 维度 | 分桶方式 | 来源 |
|------|----------|------|
| 市场温度 | BULLISH / NEUTRAL / BEARISH | MSS（已有） |
| 流动性周期 | HIGH / LOW | `amount_sma20 >= amount_sma60` → HIGH |
| 波动周期 | HIGH / LOW | `ATR% 20日` 分位 ≥ 70% → HIGH |

每个桶内统计：
- `win_rate`, `expectancy`, `max_dd`, `avg_hold_days`, `fail_type_share`
- 按形态分组（BOF/BPB/TST/PB/CPB × 桶 = 最多 60 个格子）

产出物：**环境—形态有效性矩阵**（表格或 heatmap），回答"BOF 在什么市场环境下才有正期望"。

### 执行摩擦统计深化（视角五）

v0.01 已有基础摩擦判定（停牌/涨跌停 REJECTED）。v0.04 新增：

| 指标 | 计算方式 |
|------|----------|
| `unfilled_rate` | REJECTED 订单数 / 总订单数，按原因分组 |
| `unfilled_by_reason` | HALTED / LIMIT_UP / LIMIT_DOWN / LIQUIDITY 各占比 |
| `slippage_p50` | 实际成交价 vs 理论成交价（adj_open）的中位数偏差 (bps) |
| `slippage_p95` | 同上，95 分位 |
| `delay_fill_days` | 因 REJECTED 导致的延迟成交天数 |

产出物：**执行可行性报告**，决定是否需要提高流动性阈值或修改撮合假设。

### 风险归因深化（视角六）

v0.01 已有逐形态统计。v0.04 新增**失败类型标签**和 Pareto 归因：

| 失败类型 | 说明 | 适用形态 |
|----------|------|----------|
| `confirm_failed` | 触发后未确认（回踩失败） | BPB/TST |
| `no_follow_through` | 确认后不延续（动能不足） | ALL |
| `true_breakdown` | 假破位其实是真破位 | BOF |
| `trend_reversal` | 趋势反转导致回调变转折 | PB/CPB |
| `gap_risk` | 隔夜跳空导致止损滑点 | ALL |

归因交叉维度：
- **环境桶** × **失败类型** × **形态** × **liquidity tier**
- 输出 Pareto 表：前 20% 原因解释 80% 亏损

产出物：**亏损贡献表**，回答"亏损主要来自哪类错误"。

## 验收标准

1. `python main.py report --env-matrix` 输出环境—形态有效性矩阵
2. 环境分桶矩阵覆盖 3（温度）× 2（流动性）× 2（波动）= 12 个桶
3. `unfilled_rate` 按原因分组统计正确（与 l4_orders REJECTED 记录一致）
4. `slippage_p50/p95` 在合理范围内（A 股日内滑点通常 < 50bps）
5. 亏损归因 Pareto 表输出 top-5 亏损原因及其贡献比例
6. 所有新增统计不影响原有回测链路（纯读取 + 统计，不修改 l4 交易数据）
7. 新增 l4 表有数据且可查询
8. 七维评审 S0/S1 = 0

## 评审证据与 Go/No-Go 门禁（补充）

除 S0/S1 清零外，v0.04 还必须提交：

1. 只读链路证明：新增模块不写 `l3_signals/l4_orders/l4_trades`，仅写统计表（SQL 审计或单测证据）。
2. 七维证据表：重点覆盖“报告真实性、Schema 契合、幂等重跑”三维。
3. 连环失效回放：至少 1 条“统计口径错误 -> 回退判定错误”的 kill-chain 被拦截。
4. 幂等重跑结果：同输入重跑两次，`l4_env_bucket_stats/l4_execution_stats/l4_loss_attribution` 主键集合一致。
5. Go/No-Go 结论：显式给出“可晋级 v0.05 / 不可晋级”。

## 已知风险

| 级别 | 风险 | 缓解措施 |
|------|------|----------|
| 中 | 分桶过细导致单桶样本不足 | 每桶最低 30 笔交易才出统计 |
| 中 | 失败类型标签需人工标注验证 | 先自动标注 + 抽样 50 笔人工校验 |
| 低 | 统计模块计算量增大 | 增量计算 + 缓存 |
