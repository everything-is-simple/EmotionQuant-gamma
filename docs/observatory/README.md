# Observatory

## 定位

`docs/observatory/` 只保留：

1. 观察框架
2. 评审标准

这里回答“怎么观察、怎么评审、怎么复盘”，不回答“当前系统怎么设计”。

| 文件 | 角色 |

## 使用边界

1. `god_view` 用于观察和规划，不直接改写当前执行口径。
2. `sandbox-review-standard` 用于评审方法，不替代版本证据。
3. 历史基线看 `docs/design-v2/01-system/system-baseline.md`。
4. 当前治理状态看 `docs/spec/common/records/development-status.md`。
5. 当前设计看 `blueprint/`，当前治理看 `docs/spec/`。
