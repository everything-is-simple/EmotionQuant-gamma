## 0. 核心判断

基于所有已有证据，我们现在可以确定：

1. **Alpha 主要来自 PAS 入场层**（尤其是 PB/CPB，见 `v0.01-plus-pas-ablation-20260310.md`）
2. **IRS 是排序增强，不是主 alpha**（见 `v0.01-plus-phase-2-exit-20260311.md`）
3. **MSS 不是 alpha，是风险控制**（见 `v0.01-plus-phase-4-gate-decision-20260311.md`）
4. **Legacy 赢是因为伤害少，不是因为发现了更多 alpha**

当前最大瓶颈：**还没把"谁在创造 alpha、谁在吞掉 alpha"拆干净**

### Phase 2：Exit Decomposition（2 周）

**目标**：确认收益是被"买错"还是"卖坏"

**实验设计**：

```python
# 固定配置
PAS_ENTRY = "Phase1 最优形态"  # 使用 Phase 1 找到的最强入场
BROKER_MODE = "minimal"

# 变量：退出规则
EXIT_VARIANTS = [
    "baseline",      # 止盈 +8% / 止损 -5% / 持有 10 天
    "tight",         # 止盈 +5% / 止损 -3% / 持有 5 天
    "loose",         # 止盈 +12% / 止损 -8% / 持有 15 天
    "trailing",      # 移动止盈（从最高点回撤 3%）
    "time_only",     # 只用时间退出（持有 10 天）
    "profit_only",   # 只用止盈（+8%），无止损
]
```

**对比维度**：

| 维度 | 指标 |
|------|------|
| 止盈效率 | 触发止盈的交易占比 / 平均盈利幅度 |
| 止损保护 | 触发止损的交易占比 / 平均亏损幅度 |
| 时间衰减 | 持有天数 vs 收益率曲线 |
| 整体表现 | EV / PF / MDD |

**交付物**：

1. `evidence/normandy-phase2-exit-decomposition-20260311.json`
2. `records/normandy-phase2-decision-20260311.md`
3. 结论：当前退出规则是否在吞 alpha

**验收标准**：

- 找出最优退出规则组合
- 量化"退出伤害"（baseline vs optimal 的 EV 差）
- 如果 EV 差 > 1%，则退出规则需要优化

## 4. 技术实现

### 4.1 最简 Broker 定义

```python
# src/broker/minimal_broker.py

class MinimalBroker:
    """最简 Broker：只做撮合，不做风控干预"""
    
    def __init__(self, max_positions: int = 10):
        self.max_positions = max_positions  # 固定上限，不动态调整
        self.positions = {}
        self.orders = []
        self.trades = []
    
    def process_signals(self, signals: List[Signal], date: str):
        """处理信号：简单的 FIFO 逻辑"""
        # 1. 处理退出
        self._process_exits(date)
        
        # 2. 处理新入场
        available_slots = self.max_positions - len(self.positions)
        
        for signal in signals[:available_slots]:
            self._create_buy_order(signal, date)
    
    def _process_exits(self, date: str):
        """统一退出规则"""
        for position in list(self.positions.values()):
            # 止盈
            if position.unrealized_return >= 0.08:
                self._close_position(position, date, "TAKE_PROFIT")
            # 止损
            elif position.unrealized_return <= -0.05:
                self._close_position(position, date, "STOP_LOSS")
            # 时间
            elif position.holding_days >= 10:
                self._close_position(position, date, "TIME_EXIT")
```

## 5. 决策树

```
Phase 1 完成
├─ 如果没有任何形态 EV > +2%
│  └─ 停止，回到 v0.01 baseline，重新审视 PAS 定义
│
├─ 如果有 1-2 种形态 EV > +2%
│  └─ 进入 Phase 2，用最优形态做 exit 分解
│
└─ 如果有 3+ 种形态 EV > +2%
   └─ 进入 Phase 2，选择样本量最大的形态

Phase 2 完成
├─ 如果退出优化能提升 EV > 1%
│  └─ 采用最优退出规则，进入 Phase 3
│
└─ 如果退出优化提升 EV < 0.5%
   └─ 保持 baseline 退出规则，进入 Phase 3

Phase 3 完成
├─ 如果 IRS 改善 EV > 0.5%
│  └─ 保留 IRS，采用最优权重，进入 Phase 4
│
└─ 如果 IRS 改善 EV < 0.3%
   └─ 移除 IRS，直接进入 Phase 4

Phase 4 完成
├─ 如果有 MSS 模式在 MDD < 12% 下 EV 损失 < 0.5%
│  └─ 采用该 MSS 模式，完成诺曼底登陆
│
└─ 如果所有 MSS 模式 EV 损失 > 1%
   └─ 不使用 MSS，完成诺曼底登陆
```

## 6. 风险控制

### 6.1 回退条件

| Phase | 回退条件 | 回退目标 |
|-------|----------|----------|
| Phase 1 | 所有形态 EV < +1% | 回到 v0.01，重新审视 PAS 定义 |
| Phase 2 | 最优退出 EV < baseline | 保持 baseline 退出规则 |
| Phase 3 | IRS 改善 EV < 0 | 移除 IRS 模块 |
| Phase 4 | 所有 MSS 模式 EV 损失 > 2% | 不使用 MSS |

- 单个 Phase 超过计划时间 50%，触发复盘
