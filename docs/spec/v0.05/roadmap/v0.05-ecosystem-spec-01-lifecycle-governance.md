## 需求摘要

在保持 v0.01 核心执行语义不变的前提下，建立可进化策略生态：

1. 生命周期状态机可达且无死路径
2. 重叠度控制与冻结规则可测且可回归
3. strength 分位有效性有统计结论
4. walk-forward + 参数敏感性 + 成本压力三套稳健性完成

## 设计要点

### 生命周期状态机

- 目标状态：`NEW -> VALIDATING -> ACTIVE -> DEGRADED -> FROZEN -> DEPRECATED`
- 必须证明关键路径可达、异常路径可回退、无死路径
- 默认建议模式，不自动硬禁用（需人工确认后执行）

### 重叠度治理

- 同股同日多形态重叠率超阈值时进行合并或降权
- 冻结规则触发后必须保证不再产生新信号
- 规则日志需可追溯到单笔信号
