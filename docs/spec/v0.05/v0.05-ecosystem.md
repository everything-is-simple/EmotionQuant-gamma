## 设计要点

### 注册表生态管理（视角七）

形态不再是"注册即永久在线"，而是有完整的生命周期：

```
NEW → VALIDATING → ACTIVE → DEGRADED → FROZEN → DEPRECATED
                     ↑                      |
                     └──────────────────────┘ (解冻)
```

**生命周期规则**：

| 状态 | 含义 | 触发条件 |
|------|------|----------|
| NEW | 刚注册，未验证 | 新形态写入 registry |
| VALIDATING | 单形态回测验证中 | 手动启动验证 |
| ACTIVE | 在线参与信号生成 | 单形态回测通过 §6.2 门槛 |
| DEGRADED | 降权（strength × 衰减系数） | 滚动 100 笔 EV 下降 > 20% |
| FROZEN | 冻结不参与信号生成 | 滚动 100 笔 EV 为负 |
| DEPRECATED | 淘汰（保留代码但永久关闭） | 连续 2 个评估窗 FROZEN 未恢复 |

**滚动窗口淘汰**：每个形态维护最近 100 笔交易的滚动期望值。EV 为负且持续两个评估窗 → 冻结。

**重叠度控制**：同日同股多形态命中时，计算重叠率。若重叠率 > 阈值（默认 40%），合并信号或仅保留 strength 最高者，避免伪独立的"投票幻觉"。

**环境启用矩阵**：基于 v0.04 环境—形态有效性矩阵，某形态在特定环境桶内 EV 为负 → 先产出“禁用建议”；默认人工确认后生效（`AUTO_ENV_DISABLE=False`）。

### 信号强度有效性验证（视角三）

回答"strength 到底有没有用"：

| 分析维度 | 方法 |
|----------|------|
| 强度分位回测 | Top 20% vs Bottom 20% 的胜率/期望值/回撤对比 |
| 组成因子贡献 | ClosePos / VolMult / 触边次数等各因子分层统计 |
| 参数敏感性 | break_pct / touch_band / volume_mult 在 ±20% 下的表现变化 |

如果 Top 20% 和 Bottom 20% 无统计显著差异 → strength 无效 → 标记为需重构。

### 稳健性验证框架

| 验证类型 | 方法 | 目的 |
|----------|------|------|
| Walk-Forward | 滚动训练/测试窗口（如 2 年训练 + 6 个月测试） | 防止过拟合 |
| 参数敏感性 | 关键参数 ±20% 网格扫描 | 确认参数不在"悬崖边" |
| 成本压力测试 | 佣金 ×2 / 滑点 ×3 / 印花税上调 | 确认利润不依赖低成本假设 |

---

## 验收标准

1. `l4_pattern_lifecycle` 表记录每个形态的当前状态和滚动指标
2. FROZEN 状态的形态不参与 `generate_signals`（单测验证）
3. 重叠度 > 40% 时同日同股信号被合并（单测验证）
4. 信号强度 Top20% vs Bottom20% 统计对比报告可输出
5. Walk-Forward 验证在 3 年数据上可运行（至少 3 个窗口）
6. 参数敏感性报告覆盖 BOF 全部关键参数（break_pct / volume_mult / close_pos_threshold）
7. 成本压力测试：佣金 ×2 条件下 EV 仍 > 0
8. 七维评审 S0/S1 = 0

## 评审证据与 Go/No-Go 门禁（补充）

除 S0/S1 清零外，v0.05 还必须提交：

1. 生命周期状态机证据：`NEW -> VALIDATING -> ACTIVE -> DEGRADED -> FROZEN -> DEPRECATED` 路径可达且无死路径。
2. 七维证据表：重点覆盖“状态机完整性、边界冲突去重、报告真实性”三维。
3. 连环失效回放：至少 1 条“生命周期状态 + 组合模式 + 信号合并” kill-chain 回放通过。
4. 幂等重跑结果：同输入重跑两次，`l4_pattern_lifecycle/l4_signal_quality/l4_robustness_results` 主键集合一致。
5. Go/No-Go 结论：显式给出“可晋级 v0.06 / 不可晋级”。

## 已知风险

| 级别 | 风险 | 缓解措施 |
|------|------|----------|
| 高 | 滚动淘汰误杀暂时低谷的形态 | 冻结前需两个评估窗确认，支持手动解冻 |
| 高 | 环境启用矩阵过度拟合历史环境分布 | 默认建议模式（AUTO_ENV_DISABLE=False）；需人工审核后启用 |
| 中 | 重叠度计算在多形态场景下耗时增大 | 仅对候选池 × 活跃形态计算，非全市场全形态 |
| 中 | Walk-Forward 窗口选择影响结论 | 至少用 3 种窗口长度交叉验证 |
| 低 | strength 验证结论可能导致大范围重构 | 先标记为"需关注"，不立即删除 |
