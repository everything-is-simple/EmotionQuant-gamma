## 设计要点

### BPB 突破回踩检测器

BPB（Breakout Pullback）：趋势突破后回踩结构上沿，确认支撑有效后入场。

- **与 BOF 的区别**：BOF 是区间内假突破反杀（Spring），BPB 是趋势启动后的回踩确认
- BPB 有"确认期"（1–2 日回踩确认），BOF 无确认期（触发即买）
- **时序冻结口径**：确认完成当日记为 `signal_date=T`，统一在 `execute_date=T+1` 以 `T+1 Open` 执行；确认期不得改写 Broker 撮合语义
- `detect()` 签名与 PatternDetector ABC 一致，输入 df + code + signal_date，输出 Signal | None
- strength 计算：回踩深度 + 量缩比 + 突破力度

### IRS 4 因子扩展

| 因子 | 权重 | 说明 |
|------|------|------|
| 相对强度 | 40%（原 55%） | industry_pct_chg - benchmark_pct_chg |
| 资金流向 | 30%（原 45%） | 10 日滚动净流入 + 资金占比 |
| **连续性** | 15%（新增） | 行业连续 N 日跑赢基准的天数比例 |
| **估值** | 15%（新增） | 行业 PE 分位（低估值加分） |

权重调整须经消融对照验证。

### 基因库（gene.py）实现

- **仅做事后分析**：基于 v0.01 BOF 历史交易样本反推候选特征（"5 牛 5 衰"画像）
- v0.02 阶段 `ENABLE_GENE_FILTER` 仍默认 **False**
- 只有反推验证通过后才可考虑在 v0.03 开启硬过滤
- 输出：每只个股的基因画像评分（暂不进入实时漏斗）

### BB 箱体增强

所有形态的信号强度加入 BB 箱体位置因子加权（Bollinger Band 位置 0–1 归一化），用于区分"箱体底部假突破"vs"箱体中部震荡"。

## 验收标准

1. `python main.py backtest --patterns=bpb` 单形态回测可独立运行
2. `python main.py backtest --patterns=bof,bpb --combination=ANY` 双形态组合回测可运行
3. BPB 单形态回测输出同口径指标（胜率/盈亏比/期望值/最大回撤/分环境中位数）
4. BPB 与 BOF 的回测结果可直接对比（同区间、同费用模型）
5. IRS 4 因子 vs 2 因子消融对照完成，结论有数据支撑
6. gene.py 输出 v0.01 样本的"5 牛 5 衰"画像，可解释
7. `ENABLE_GENE_FILTER` 默认关闭，手动开启不崩溃
8. 输出 `trigger_to_confirm_rate`（BPB 触发→确认转化率，按月统计）
9. 七维评审 S0/S1 = 0

## 评审证据与 Go/No-Go 门禁（补充）

除 S0/S1 清零外，v0.02 还必须提交：

1. 七维证据表：至少覆盖 `l3_signals`、`l4_orders`、`l4_trades` 的 Schema/幂等/时序三维对齐证明。
2. 连环失效回放：至少 1 个“BPB 确认期 + 组合模式 + Broker 风控”联动场景，证明无重复下单/重复成交。
3. 幂等重跑结果：同一区间执行两次 `--patterns=bpb`，`signal_id/order_id/trade_id` 集合一致。
4. 偏差模板记录：若出现偏差，按 `sandbox-review-standard.md` §4 的 8 项模板落库到评审记录。
5. Go/No-Go 结论：显式给出“可晋级 v0.03 / 不可晋级”。
