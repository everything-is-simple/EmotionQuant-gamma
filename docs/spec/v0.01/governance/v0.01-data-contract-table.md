# v0.01 数据契约表（执行模板）

**版本**: v0.01 执行补充  
**状态**: Active（模板 + 最小落地口径）  
**日期**: 2026-03-04  
**基线**: `docs/design-v2/system-baseline.md`, `docs/design-v2/data-layer-design.md`

## 1. 使用方式

1. 每个核心表至少填：主键、幂等键、as-of 规则、缺失策略。  
2. 字段说明做到“可被实现者直接写 SQL/代码”。  
3. 缺失策略必须四选一：`FAIL / SKIP / FILL / REJECT`。

## 2. 核心契约（v0.01）

| 表名 | 用途 | 主键/幂等键 | As-Of 规则 | 缺失策略 | 口径备注 |
|---|---|---|---|---|---|
| `l1_stock_daily` | 原始日线 | PK:`(ts_code,date)` | 仅 Data 层写入；下游读取需 `date<=asof_date` | `SKIP` 空行；字段缺失 `FILL` | raw 价格与停牌/涨跌停真相源 |
| `l2_stock_adj_daily` | 后复权与派生指标 | PK:`(code,date)` | Strategy/Selector 必须 `date<=asof_date` | `adj_factor` 缺失 `FILL=1.0` | `adj = raw * adj_factor` |
| `l1_trade_calendar` | 交易日序列 | PK:`date` | 所有 T+1 推进都依赖该表 | 缺失交易日 `FAIL` | 交易日历一等公民 |
| `l1_stock_info` | 基础信息/行业/ST | PK:`(ts_code,effective_from)` | 读取时取 `effective_from<=asof_date` 最近记录 | 行业缺失 `FILL='未知'` | 避免幸存者偏差的关键维表 |
| `l1_stock_status_daily`* | 股票日状态（上市/退市/停牌/ST） | PK:`(code,date)` | Selector 股票池必须 `date=asof_date` 过滤 | 缺状态 `FAIL` | G6 强制项；可先用稳定视图实现 |
| `l3_signals` | 信号输出 | PK:`signal_id` | `signal_date<=asof_date` | 字段缺失 `FAIL` | `signal_id=f"{code}_{signal_date}_{pattern}"` |
| `l4_orders` | 订单状态机 | PK:`order_id` | 仅按执行时间推进，不逆时读取 | 不可成交 `REJECT`，超时 `EXPIRED` | 状态：`PENDING/FILLED/REJECTED/EXPIRED` |
| `l4_trades` | 成交记录 | PK:`trade_id` | 回测/报告读取 `execute_date<=asof_date` | 无成交不写行 | `trade_id=f"{order_id}_T"` |
| `_meta_runs` | 运行追溯 | PK:`run_id` | 不受交易 as-of 约束 | 关键元数据缺失 `FAIL` | 必含 `config_hash/data_snapshot/git_commit` |
| `_meta_schema_version` | Schema 版本门 | PK:`id=1` | 启动即检查 | 不一致 `FAIL` | 阻断不兼容运行 |

## 3. As-Of 实现约束（强制）

1. 业务模块读数据只能走 Store 封装接口（例如 `read_table_asof`）。  
2. 禁止在业务模块直接拼“无上限日期 SQL”。  
3. 审计建议：run 期间记录 as-of 查询数量与抽样 SQL。

\* `l1_stock_status_daily` 允许采用“物理表或同名视图”二选一，但口径必须稳定且可重跑复现。

## 4. 缺失策略字典（统一）

| 策略 | 含义 | 典型场景 |
|---|---|---|
| `FAIL` | 中断运行并报错 | schema/version 不一致、关键主键空值 |
| `SKIP` | 跳过记录，继续运行 | API 返回空批次 |
| `FILL` | 用确定默认值填充 | `adj_factor` 缺失填 1.0 |
| `REJECT` | 写业务拒绝状态 | 不可成交订单 |

## 5. Run 元数据最小字段

1. `run_id`  
2. `start_time`, `end_time`  
3. `modules`, `status`, `error_summary`  
4. `config_hash`, `data_snapshot`, `git_commit`, `runtime_env`
