# v0.01 边界条件与确定性规则（执行版）

**版本**: v0.01 执行补充  
**状态**: Active（不改 Frozen 主干语义）  
**日期**: 2026-03-04  
**基线**: `docs/design-v2/system-baseline.md`

## 1. 使用说明

1. 本文件用于把“可能争议的真实市场边界”固化为确定分支。  
2. 若与 Frozen 主干冲突，以 `system-baseline.md` 为准。  
3. 本文件可以在实现阶段补充勘误，但不得改写主干执行语义。

## 2. 边界规则总表（v0.01）

| 场景 | BUY 处理 | SELL 处理 | 状态写入 | 备注 |
|---|---|---|---|---|
| `is_halt=true`（停牌） | 不成交 | 不成交 | `REJECTED(HALTED)` | 当日直接拒单，不自动延期 |
| 开盘触及涨停（买入） | 不成交 | — | `REJECTED(LIMIT_UP)` | 按原始价口径 `raw_open/open` 对 `up_limit` |
| 开盘触及跌停（卖出） | — | 不成交 | `REJECTED(LIMIT_DOWN)` | 按原始价口径 `raw_open/open` 对 `down_limit` |
| 一字涨停 / 一字跌停 | 不成交 | 不成交 | `REJECTED(ONE_WORD_LIMIT)` | 可合并到 LIMIT_UP/DOWN 原因码 |
| 无行情 bar（数据缺失） | 不成交 | 不成交 | `REJECTED(NO_MARKET_DATA)` | 计入缺失率 |
| 订单超过有效期 | 不再撮合 | 不再撮合 | `EXPIRED` | 见 §3 订单生命周期 |
| 退市整理期（持仓） | 禁止新开仓 | 强制退出优先 | `FILLED/REJECTED` | 若无法成交，保留持仓并持续尝试 |
| ST 股票 | 默认禁止新开仓 | 已有仓位按卖出规则 | `REJECTED(ST_FILTER)` | 由 Selector 粗筛优先拦截 |

## 3. 订单生命周期（v0.01 固定）

1. 初始状态：`PENDING`（`execute_date = next_trade_date(signal_date)`）。  
2. 可成交时：`FILLED` 并生成 `Trade`。  
3. 不可成交且满足拒绝条件：`REJECTED`。  
4. 超过有效期：`EXPIRED`（默认建议 `MAX_PENDING_TRADE_DAYS = 1`；可配置）。  
5. v0.01 不启用“自动延期重挂”，避免回测与实盘语义漂移。

## 4. 成交价与成本（v0.01 最小模型）

1. 成交价：`T+1 Open`（买卖均如此），再叠加固定滑点。  
2. 滑点：买入 `+slippage_bps`，卖出 `-slippage_bps`。  
3. 成本：佣金 + 过户费 + 卖出印花税。  
4. 所有成本参数必须来自 `config.py`，禁止硬编码。

## 5. 估值与持仓边界

1. 正常估值：持仓按当日可得收盘价估值。  
2. 停牌估值：沿用最近可得价格并打标 `stale_price=true`。  
3. 长期停牌：报告需单列“停牌持仓占比”。  
4. 回测末日：执行强平流程；若无可成交价格，按最近可得价估值并标记异常。

## 6. 最小验收要求（与 Gate 联动）

1. 每个 `REJECTED/EXPIRED` 必须有 `reject_reason` 或 `expire_reason`。  
2. 报告必须输出：`reject_rate`、`expired_rate`、`missing_market_data_rate`。  
3. 同输入重跑两次，订单状态分布与关键指标不可漂移（容忍浮点微差）。
