### 1.2 v0.01 当前唯一触发器就是 BOF

以下结论已经被代码与证据同时支持：

- 后续所有 selector 改造，都应该围绕“如何更好地服务 BOF”来做，而不是偏离到“替换 BOF”。

### 1.3 历史 GO baseline 说明 BOF 不是坏策略

`2026-03-05` 的历史 `GO baseline` 已经证明：

- `trade_count = 256`
- `expected_value = 0.018140`
- `profit_factor = 2.7126`
- `max_drawdown = 0.108484`
- `participation_rate = 0.7926`

同时还暴露出真实组合约束：

- `opportunity_count = 323`
- `filled_count = 256`
- `skip_maxpos_count = 59`
- `failure_reason_breakdown = {"MAX_POSITIONS_REACHED": 59, "ALREADY_HOLDING": 6, "SIZE_BELOW_MIN_LOT": 2}`

含义：

- BOF 不是“没有 alpha”的坏策略。
- 当前更像是“有机会，但候选拥挤、容量有限、组合层选择还不够精细”。
- 这天然说明后续增强方向应该是优化“机会怎么选”，而不是先换掉 `BOF`。

### 1.4 IRS 结构健康，taxonomy 已收口到正式 SW31

以下结论已经被 `2026-03-06` 的权威分布审计支持：

- `distinct_industries = 31`
- `rank_range = 1..31`
- `industries_per_day min/median/max = 31/31/31`
- `duplicate_rank_days = 0`

含义：

- IRS 当前的问题不是“行业表坏了”或“排名写乱了”。
- IRS 结构层已经健康，真正需要继续验证的是：它作为过滤/排序信号时，到底怎样用才不会误杀 BOF。

### 1.5 MSS 当前 zscore 版分布过窄，65/35 阈值下会极度稀疏

以下结论已经被 `MSS baseline calibration` 和 `variant comparison` 同时支持：

- `zscore_weighted6` 的分布中心明显挤在中间：
  - `p50 = 49.74`
  - `p95 = 61.62`
- 在 `65/35` 下：
  - `BULLISH = 20`
  - `BEARISH = 23`
  - `NEUTRAL = 714`
- `threshold = 65` 时：
  - `bullish_ratio = 2.64%`
  - `longest_dry_spell = 239`

含义：

- 不是 MSS 完全没波动，而是当前 `zscore + weighted6 + 65/35` 的组合把 regime 压得过窄。
- 这类输出更像“稀缺强顺风探测器”，不像“日常交易环境开关”。

### 1.6 “MSS 没影响 baseline”这个说法不成立；准确说法是“影响在不同 gate 语义下完全不同”

全周期 reduced A/B 已经证明：

- `bof_baseline_fullcycle`
  - `trade_count = 294`
  - `EV = -0.00716`
  - `PF = 2.1261`
  - `MDD = 0.2281`
- `zscore_weighted6 + bullish_required`
  - `trade_count = 16`
  - `EV = +0.03935`
  - `MDD = 0.0141`
- `zscore_weighted6 + soft_gate + IRS top10`
  - `trade_count = 133`
  - `EV = +0.01962`
  - `PF = 2.9306`
  - `MDD = 0.0507`
- `percentile_weighted6 + soft_gate + IRS top10`
  - `trade_count = 119`
  - `EV = +0.02185`
  - `PF = 2.7528`
  - `MDD = 0.0380`

含义：

- MSS 不是没作用，而是在 `bullish_required` 下作用过强，在 `bearish_only` / `soft_gate` 下作用形态不同。
- 真正把 full-cycle 修好的，不是 `MSS hard gate`，而是 `IRS + MSS soft overlay`。

### 1.7 baseline 的主要拖累发生在 NEUTRAL / BEARISH，不在 BULLISH

`bof_baseline_fullcycle` 的 `environment_breakdown` 很关键：

- `BULLISH EV = +0.0549`
- `NEUTRAL EV = -0.0086`
- `BEARISH EV = -0.0224`

含义：

- BOF 在强环境下本身并不差。
- 当前系统不是“缺少牛市识别”，而是“在大量 NEUTRAL/BEARISH 环境里，低质量交易把边际收益吃掉了”。
- 这进一步支持“IRS 负责横截面筛选、MSS 负责环境约束”的方向。

## 2. 未证实

### 2.1 `2026-03-06` 当前代码路径下的正式 baseline 还没有重新封板

当前状态是：

- `2026-03-05` 的历史 `GO baseline` 已封板，且为正 EV。
- `2026-03-06` 当前代码路径下已有 full-cycle reduced A/B，但还没有新的“正式 baseline 证据包 + 双跑幂等 + G1-G7 风格汇总”。

这意味着：

- 不能直接把 `2026-03-05` 的正 EV 基线当成“今天代码仍然有效”的正式结论。
- 也不能只凭 `2026-03-06` reduced A/B 就宣布新的默认口径已封板。

### 2.2 IRS 的 alpha 贡献尚未被单独隔离证明

虽然 `IRS + soft_gate` 的 full-cycle 组合明显改善了结果，但目前仍未正式证明：

- IRS 单独作为后置排序分数时，是否稳定优于 `IRS hard filter`
- IRS 的最优强度是否真的固定在 `top10`
- IRS 的改善主要来自“过滤行业”还是“缓解候选拥挤”

目前能证实的是：

- IRS 结构健康
- IRS 与 soft MSS 组合有价值

但还不能证实：

- IRS 当前公式本身已经是最佳
- `top10` 已经是稳定默认值

### 2.3 percentile MSS 是否应取代 zscore MSS 作为默认口径，尚未被正式确认

当前证据只支持：

- `percentile` 分布更适合做 regime 切分
- `percentile + soft_gate + IRS top10` 在 full-cycle reduced A/B 中略高 EV、略低 MDD

但还没有证实：

- `percentile` 在正式 baseline 双跑、幂等、短窗/全周期切换下整体优于 `zscore`
- `percentile` 的优势不是窗口偶然性

### 2.4 “自下而上（down-to-top）链路”还没有实验结果

当前证据全部来自 `top-to-down` 家族：

- `basic filters -> MSS gate -> IRS filter -> candidate rank -> BOF`

尚未被证实的，是你真正偏好的这条路线：

- `basic filters -> BOF -> stock/industry/market score -> final rank`

也就是说：

- 当前证据已经说明旧链路的边界和问题。
- 但新链路还没有实证结果，不能提前宣布它一定更好。

## 3. 风险点

### 3.1 baseline 口径分裂风险

当前同时存在两套重要数字：

- 历史 `GO baseline`（`2026-03-05`）
- 当前代码路径 full-cycle reduced A/B（`2026-03-06`）

风险在于：

- 如果不显式区分“历史封板口径”和“当前代码路径对照口径”，后续文档和讨论会持续混淆。

### 3.2 MSS 职责错位风险

当前 evidence 支持的不是“重写 MSS 六因子”，而是：

- MSS 当前更适合做 `soft regime overlay`
- 不适合继续承担 `hard global gate`

如果继续把它当硬闸去调阈值，风险是：

- 交易机会被过度压缩
- 样本变薄
- 看似 EV 变好，实则只是把样本删到只剩少量顺风期

### 3.3 IRS 前置硬过滤误杀 BOF 的风险

短窗证据已经显示：

- `+MSS+IRS` 在部分窗口里会把交易压到极少甚至归零

风险在于：

- BOF 是个股触发器，但 IRS 当前在它之前就裁掉了样本母体。
- 这会让 IRS 从“行业偏好因子”越界成“前置否决器”。

### 3.4 G6 源侧快照仍有稀疏边界

G6 已通过，但源侧 `raw_stock_basic` 快照依然稀疏：

- `snapshot_days = 68`
- `avg_gap_days = 87.69`
- `p95_gap_days = 334.7`
- `max_gap_days = 1036`

风险在于：

- 当前证据足够支持 v0.01，不阻塞 Gate。
- 但这仍然是高版本继续提升 `G6` 证据强度时必须保留的边界说明。

### 3.5 Week2 全周期 exhaustive ablation 仍有性能阻塞

Week2 证据页已明确记录：

- 在复用 L3 的前提下，全周期三场消融超过 `75` 分钟仍未收口

风险在于：

- 当前 full-cycle 结论更多依赖 reduced A/B。
- 如果后续实验矩阵继续扩大，而不先处理性能，证据生产会越来越慢。

说明：

### 4.4 当前最值得优先验证的主张

如果只保留一句下一步主张，本报告建议优先验证：

> `BOF trigger + IRS cross-sectional score + MSS soft regime overlay`

原因：

- 它是当前 full-cycle 证据中最接近“既保留交易参与，又改善 EV/PF/MDD”的方向。
- 它也最贴近你当前对系统的真实偏好：先个股，再行业，最后大盘。

## 5. 总结结论

把这 `12` 份 evidence 压成一句话，就是：

> v0.01 的工程底座已经可信，BOF baseline 也不是没有 edge；当前主要问题不在数据诚信，而在 MSS 作为硬门控时分布过窄、职责错位，以及 IRS 作为前置硬过滤时容易误杀 BOF 样本。现阶段最有前途的方向，不是继续强化 `MSS hard gate`，而是让 `BOF` 负责触发、`IRS` 负责横截面筛选/加权、`MSS` 负责软环境约束。
