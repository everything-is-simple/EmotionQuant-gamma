## 4. 历史保留价值

本文最值得保留的历史信息有三类：

### 4.1 它保留了 `v0.01` 的阶段式推进方式

原路线图最重要的历史信息，不是具体周计划本身，而是它体现了当时的推进思路：

1. 先把数据层和基础契约跑通。
2. 再把 `Selector / Strategy` 接成最小可交易主干。
3. 然后补 `Broker / Backtest`。
4. 最后补 `Report` 与联调闭环。

### 4.2 它保留了 `v0.01` 的验收锚点

本文仍然值得保留的另一部分，是它为 `v0.01` 留下了历史验收索引：

1. `BOF baseline` 跑通的阶段背景。
2. 七维校对和门禁收口的历史上下文。
3. 与 `runbook / errata / evidence / gate checklist` 的配套关系。

### 4.3 它解释了为什么本文只保留为历史参考

从今天回看，本文不再充当当前执行入口，原因主要在于：

1. `v0.01` 已退回历史基线角色。
2. 当前主线设计、实现与执行入口已经转移到 `blueprint/`。
3. 旧的 `Week1-Week4` 口径继续保留为正文，容易让人误以为它仍是当前任务计划。
