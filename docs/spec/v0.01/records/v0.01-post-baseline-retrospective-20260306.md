## 4. 历史保留价值

本文最值得保留的历史信息有三类：

### 4.1 它保留了“主干没错，但落点要修”的判断

原文最重要的历史结论，不是某个具体修订项，而是：

1. `BOF`、`T+1 Open`、结果契约、`Broker` 单内核这些主干不改。
2. 但数据路径、验收口径、运维记录和诚信指标等说明需要更贴近真实跑通路径。

### 4.2 它保留了当时最关心的修订方向

原文里关注的核心，不外乎这几类：

1. 路线图要从“实现前口吻”切到“基线跑通后口吻”
2. Data Layer 要承认“本地历史库优先”的现实路径
3. Selector / Broker / Report 要把已验证的边界写成正式约束
4. 运维记录和勘误需要进入正式追溯链

这些内容今天主要保留为历史回望素材。

### 4.3 它保留了“哪些内容不该回写进 v0.01”

原文同时保留了另一条很有价值的约束：

1. `BPB`
2. `GENE` 实时过滤
3. `IRS` 扩因子
4. 多形态组合

这些不应借回望之名塞回 `v0.01` 主干。
