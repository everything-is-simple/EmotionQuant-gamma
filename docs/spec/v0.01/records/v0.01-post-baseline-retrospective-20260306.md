# v0.01 基线跑通后的回望审视（2026-03-06）

**版本**: v0.01 回望审视  
**状态**: Active  
**日期**: 2026-03-06  
**目的**: 在 `BOF baseline` 已跑通并通过 `G1-G7` 后，判断 `v0.01` 路线图、实现卡与 `design-v2` 是否需要基于实战经验做回望修订。  

---

## 1. 审视结论

结论分两层：

1. **主线继续遵循 `v0.01` 路线图 + 实现卡推进。**
2. **必须在继续大规模实现前，做一轮“实战回望修订”。**

原因：

1. `docs/design-v2/01-system/system-baseline.md` 已冻结，`BOF` 单形态、`T+1 Open`、结果契约、Broker 单内核等主干语义不能在 `v0.01` 内随意改写。
2. 但本轮基线跑通不是按“理想化流程”一把直过，而是经过数据重建、幂等修复、G6 证据补齐、资金硬约束修正、诚信指标补齐后才稳定通过。
3. 这说明 **主干设计没错，但若干实现细节、验收口径、运维路径写得不够实战化**，需要回写到文档里，避免后续实现继续按“旧理想口径”漂移。

## 3. 明确保留不动（v0.01 不改）

以下内容属于 `v0.01` 的执行合同，不因本轮跑通过程曲折而改动：

1. **BOF 单形态闭环**：`BPB/TST/PB/CPB` 仍不进入 `v0.01` 实盘口径。
2. **执行语义固定为 `T+1 Open`**：`signal_date=T`，`execute_date=T+1`，成交价 `T+1 open`。
3. **模块边界与结果契约冻结**：`MarketScore / IndustryScore / StockCandidate / Signal / Order / Trade` 不扩字段口径。
4. **Broker 单内核**：回测与纸上交易共用同一个 Broker 交易内核。
5. **MSS/IRS 消融顺序不改**：`BOF baseline -> BOF+MSS -> BOF+MSS+IRS`。
6. **Gene 在 v0.01 继续禁用**：只允许事后分析，不进入实时漏斗。

## 4. v0.01 需要立即修订的文档点

以下项目建议在 `v0.01` 内修订，性质为“实现与验收口径补清楚”，不是改主干语义。

### 4.1 路线图状态需要从“实现前”切换到“基线跑通后”

目标文件：

1. `docs/spec/v0.01/roadmap/v0.01-mvp-roadmap.md`

需要修订：

1. 把“代码层验证待实现阶段执行”的表述改成“BOF baseline 已跑通，进入按实现卡收口与固化阶段”。
2. 在路线图中增加“当时基线证据”链接，至少指向：
   - `docs/spec/v0.01/evidence/v0.01-bof-baseline-evidence-20260305.md`
   - `docs/spec/v0.01/evidence/v0.01-idempotency-check-20260305-v4.json`
   - `docs/spec/v0.01/evidence/v0.01-g6-asof-evidence-20260305-v2.json`
3. 把 Week1-Week4 从“纯计划”升级为“计划 + 已验证现实约束”。

原因：

1. 当前路线图 still 偏“实现前总校对”口吻，与基线已跑通的现实不一致。

### 4.2 Data Layer 需要补“本地历史库优先”这一实战路径

目标文件：

1. `docs/spec/v0.01/roadmap/v0.01-mvp-spec-01-data-layer.md`
2. `docs/design-v2/02-modules/data-layer-design.md`

需要修订：

1. 明确 `v0.01` 的首选数据路径是“优先复用本地 `raw_*` 历史库映射到执行库”，而不是每次都重新拉取三年数据。
2. 把 `fetch` 的定位从“首轮必须完成全量拉取”修正为“增量更新 + 断点续传 + 备源兜底”。
3. 明确 `raw_stock_basic` / `list_status` 快照稀疏性是已知现实边界，需配合 `as-of` 与变更点写入。
4. 补充 `amount` 单位来自 TuShare `千元`，相关流动性阈值必须显式注明单位。

原因：

1. 本轮基线并非靠重拉三年数据跑通，而是靠本地历史库复用 + 重建链路跑通。
2. 如果不回写，后续 Data 实现仍会错误地把“重拉三年”当成默认主路。

### 4.3 Selector 需要补“在市过滤 + 单位口径 + 候选解释性”

目标文件：

1. `docs/spec/v0.01/roadmap/v0.01-mvp-spec-02-selector.md`

需要修订：

1. 明确 `list_status='L'` + `asof_date` 过滤属于强制条件，不只是 G6 证据附属品。
2. 明确流动性阈值使用的 `amount` 单位和注入位置，避免把千元当元。
3. 补充候选解释性最小落点：`filters_passed` / `reject_reason` / `liquidity_tag` 进入追踪表或报告，而不是停留在口头描述。

原因：

1. 本轮 G6 跑通靠的不是抽象原则，而是明确的在市过滤和快照边界控制。

### 4.4 Broker 的确定性与状态守卫应从“控制点”升级为“确认约束”

目标文件：

1. `docs/spec/v0.01/roadmap/v0.01-mvp-spec-04-broker.md`
2. `docs/design-v2/02-modules/broker-design.md`

需要修订：

1. 明确 `order_id = signal_id`、`trade_id = f"{order_id}_T"` 是已验证生效的确定性约束，不再只是实现提示。
2. 明确 `date.today()` 禁用范围：所有回测路径必须显式传入模拟日期。
3. 明确 BUY 不触发 `update_trust`、仅 SELL 更新信任是已验证规则。
4. 明确 paper 持仓也必须走止损/止盈检查，不得绕过风险内核。

原因：

1. 这些不是“可选控制点”，而是本轮真正修复幂等、连亏熔断和信任状态机失真的关键点。

### 4.5 Backtest/Report 需要把“诚信指标”写成强制验收项

目标文件：

1. `docs/spec/v0.01/roadmap/v0.01-mvp-spec-05-backtest-report.md`
2. `docs/design-v2/02-modules/backtest-report-design.md`

需要修订：

1. 在验收标准中显式增加：
   - `reject_rate`
   - `missing_rate`
   - `exposure_rate`
   - `failure_reason_breakdown`
   - `opportunity_count`
   - `filled_count`
   - `participation_rate`
2. 在幂等验收中明确要求 `l4_orders/l4_trades` 主键集合一致，而不是只看指标近似相等。
3. 在终止结算例外中明确“回测末日强平不属于交易语义修改，只是回测定界”。

原因：

1. 本轮基线真正回答了“能不能参与机会、有没有漏单、是不是假盈利”，这些已经不该再作为隐含要求。

### 4.6 运维记录与勘误需要从“旁支”提升为路线图的正式追溯入口

目标文件：

1. `docs/spec/v0.01/roadmap/v0.01-mvp-roadmap.md`

需要修订：

1. 在“可追溯索引”中显式纳入：
   - `data-rebuild-runbook-20260303.md`
   - `rebuild-v0.01-errata-20260304.md`
   - `v0.01-bof-baseline-evidence-20260305.md`
2. 把它们定义为“基线跑通后的正式证据链”，而不是附属记录。

## 5. 不建议回写到 v0.01，应保留到 v0.02+ 的内容

以下经验可以保留为研究结论或后续需求，但不应借机回写成 `v0.01` 主干要求：

1. `BPB` 形态落地与并行回测
2. `GENE` 硬过滤进入实时漏斗
3. `IRS` 从 2 因子扩到 4/6 因子
4. `PAS_COMBINATION` 多形态组合实盘化
5. 更细环境分桶矩阵、生态治理、组合层预算

原因：

1. 这些都属于版本扩展，不是本轮基线跑通后对 `v0.01` 的必要修补。

---

## 6. 推荐执行顺序

建议下一步不是直接开新功能，而是做一个短修订轮：

1. 先修 `v0.01-mvp-roadmap.md`
2. 再修 `spec-01 / spec-02 / spec-05`
3. 最后修 `design-v2/data-layer-design.md` 与 `design-v2/backtest-report-design.md` 的说明性口径

完成标准：

1. `v0.01` 主干语义不变
2. 文档能解释“为什么这次基线是这样跑通的”
3. 后续实现不再依赖口头记忆补上下文

---

## 7. 最终判断

**要继续按 `v0.01` 路线图 + 实现卡推进。**  
但推进前，**有必要做一轮基于实战证据的文档回望修订**。

一句话总结：

**主干不推翻，落点要校正；打法不换，招式要去掉纸上谈兵的部分。**
