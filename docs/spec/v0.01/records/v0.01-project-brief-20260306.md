# v0.01 项目简报（2026-03-06）

## 1. 项目定位

EmotionQuant 当前定位已明确：这是一个面向中国 A 股的低频结构交易系统，不做高频，不追拥挤赛道。系统只解决四个问题：买什么、何时买、买多少、何时卖。

对应设计上：

1. Selector 负责从全市场缩池找候选。
2. Strategy 负责用 BOF 单形态在 T 日收盘后生成信号。
3. Broker 负责按 `T+1 Open` 执行、控制仓位和处理退出。
4. Backtest/Report 负责回测验证和复盘统计。

## 2. 当前进展

当前项目不再处于纯设计阶段。`docs/design-v2` 已完成封版，v0.01 基线 `BOF baseline` 已真实跑通，并通过 `G1-G7` 验证，说明最小闭环已经被代码和证据证明可执行。

截至 `2026-03-05` 的历史 `GO` 基线结果为：

- `expected_value = 0.018140`
- `profit_factor = 2.7126`
- `max_drawdown = 10.85%`
- `participation_rate = 79.26%`

同时，最小测试已通过。

补充说明：该简报生成于 `2026-03-06` 当日早期；当前代码路径下的后续全周期对照，应以 `docs/spec/v0.01/evidence/v0.01-selector-ablation-mss-fullcycle-ab-20260306.json` 为准。

## 3. 当前阶段判断

项目当前阶段是 `Rebuild Week 1`。Week1 Data Layer 的差距审计已经完成，并修正了若干关键实现口径；但按四周交付计划，Week2 的 Selector/Strategy、Week3 的 Broker/Backtest、Week4 的 Report/联调/纸上交易，仍在后续推进中。

这意味着，系统已经从“文档定稿”走到“基线跑通”，正在进入“按实现卡逐周收口交付”的阶段。

## 4. 管理结论

当前结论不是“项目还在概念设计”，也不是“v0.01 已完整交付”，而是：

`设计已冻结，基线已跑通，Week1 正在收口，后续 Week2-Week4 仍需按实现卡继续完成。`
