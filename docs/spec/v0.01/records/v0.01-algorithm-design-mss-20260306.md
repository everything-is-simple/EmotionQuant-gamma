# v0.01 当前版算法设计：MSS

日期：`2026-03-06`

## 1. 定位

在当前 v0.01 里，MSS 的定位不是“完整市场全景系统”，而是：

`一个可开关的市场 gate`

它回答的问题只有一个：

`今天市场环境是否允许做多形态交易？`

当前输出契约只有：

- `date`
- `score`
- `signal`（`BULLISH / NEUTRAL / BEARISH`）

不输出旧版 `core-algorithms` 中的：

- `cycle`
- `trend`
- `position_advice`

这些内容属于未来扩展，不属于 v0.01 当前可执行范围。

## 2. 当前实现文件

- `src/selector/normalize.py`
- `src/selector/baseline.py`
- `src/selector/mss.py`

## 3. 输入与上游依赖

MSS 只读 `l2_market_snapshot`，不读行业、不读个股、不读 PAS。

当前依赖字段主要包括：

- `total_stocks`
- `rise_count / fall_count`
- `strong_up_count / strong_down_count`
- `limit_up_count / limit_down_count`
- `touched_limit_up_count`
- `new_100d_high_count / new_100d_low_count`
- `continuous_limit_up_2d / continuous_limit_up_3d_plus`
- `continuous_new_high_2d_plus`
- `high_open_low_close_count / low_open_high_close_count`
- `pct_chg_std`
- `amount_volatility`

## 4. 当前算法定义

### 4.1 六因子

当前实现保留 `design-v2` 的六因子框架：

1. `market_coefficient`
2. `profit_effect`
3. `loss_effect`
4. `continuity`
5. `extreme`
6. `volatility`

### 4.2 权重

当前权重是：

- 大盘系数：`17%`
- 赚钱效应：`34%`
- 亏钱效应：`34%`（翻转）
- 连续性：`5%`
- 极端因子：`5%`
- 波动因子：`5%`（翻转）

### 4.3 判定阈值

- `score >= 65` → `BULLISH`
- `score <= 35` → `BEARISH`
- 其余 → `NEUTRAL`

### 4.4 归一化

当前使用 `zscore -> 0~100`：

```text
z = (value - mean) / std
score = clip((z + 3) / 6 * 100, 0, 100)
```

`std == 0` 时，返回 `50`。

### 4.5 当前 baseline 来源

当前 `src/selector/baseline.py` 里的 baseline 仍然是占位式参数。

它的作用只是让管线能跑通，不代表当前已经完成真实校准。

这也是为什么 Week2 分布审计会出现：

`2023-01-03 ~ 2026-02-24 全部 768 个交易日均为 NEUTRAL`

## 5. 当前实现的真实问题

### 5.1 gate 没有信息量

这不是理论判断，而是已经被证据坐实：

- `signal_counts = {"NEUTRAL": 768}`
- `BULLISH = 0`
- `BEARISH = 0`

这意味着：

`ENABLE_MSS_GATE` 现在打开和关闭，几乎没有差别。

所以目前 `BOF baseline` 和 `BOF + MSS` 的结果相同，不是偶然，而是算法现实。

### 5.2 score 分布过窄

当前审计结果：

- `p05 = 50.6481`
- `p95 = 52.9998`

这说明当前 score 在 50 附近极度收缩，根本碰不到 `65/35` 的阈值。

## 6. 我们这版 MSS 的正确方向

我建议把当前 MSS 明确定义为：

`六因子市场温度 + 经验 baseline 校准 + 三态 gate`

而不是回到旧版那种“大一统市场状态机”。

原因很简单：

1. v0.01 需要的是可验证 gate，不是宏观叙事系统
2. 当前 Selector 只需要 `score/signal`
3. 先把 gate 做有效，再谈 `cycle/trend/position_advice`

## 7. 下一步校准计划

### 7.1 先做经验 baseline

先基于真实 `l2_market_snapshot` 全区间数据，为六个 raw 因子统计：

- `mean`
- `std`
- `p05/p25/p50/p75/p95`

然后替换掉当前占位式 `0/1` baseline。

### 7.2 再做阈值复核

baseline 换成经验参数之后，再看：

- `BULLISH / NEUTRAL / BEARISH` 是否都出现
- `MSS gate` 是否能改变候选池和交易结果

### 7.3 最后再决定是否需要滚动窗口

如果经验 baseline 有效，再进一步评估：

- 固定全样本 baseline
- 120 日滚动 baseline

哪个更稳定、更不容易漂移。

## 8. 本文档的明确结论

当前 v0.01 的 MSS 不是“设计没想清楚”，而是：

`框架是对的，baseline 还没校准。`

所以 MSS 的下一步不该是大改公式，而该是：

`先做 baseline 校准，再重新跑 +MSS 消融。`
