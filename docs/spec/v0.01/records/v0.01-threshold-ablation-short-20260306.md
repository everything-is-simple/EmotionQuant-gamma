# v0.01 MSS 阈值短窗消融记录

**日期**: 2026-03-06  
**窗口**: 2026-01-05 ~ 2026-02-24  
**状态**: completed

---

## 1. 目的

在正式 SW31 链路与已校准 MSS baseline 之上，验证：

1. `BOF baseline`
2. `BOF + MSS`
3. `BOF + MSS + IRS`

并把 MSS `bullish_threshold = 55 / 58 / 60 / 62 / 65` 一起纳入对照。

---

## 2. 结果摘要

### 2.1 `BOF baseline`

- `trade_count = 13`
- `signals_count = 13`
- `EV = -0.01306`
- `PF = 0.7859`
- `MDD = 0.02118`

### 2.2 `BOF + MSS`

对 `55 / 58 / 60 / 62 / 65` 五档阈值，结果完全一致：

- `trade_count = 13`
- `signals_count = 13`
- `EV = -0.01306`
- `PF = 0.7859`
- `MDD = 0.02118`

### 2.3 `BOF + MSS + IRS`

对 `55 / 58 / 60 / 62 / 65` 五档阈值，结果也完全一致：

- `trade_count = 9`
- `signals_count = 9`
- `EV = -0.01265`
- `PF = 0.3911`
- `MDD = 0.00828`

---

## 3. 关键结论

本轮最重要的发现不是“哪档阈值最好”，而是：

`在当前 v0.01 gate 语义下，bullish_threshold sweep 对交易结果是 no-op。`

原因是当前 Selector 的 MSS gate 规则是：

- `BEARISH` 当天不出手
- `NEUTRAL / BULLISH` 都允许继续进入下游

因此调整 `bullish_threshold` 只会改变：

- `BULLISH / NEUTRAL` 的环境标签分布

不会改变：

- 是否阻断交易
- 候选池
- 信号数
- 交易数

---

## 4. 对系统设计的含义

这说明当前真正待决策的问题不是：

`MSS 的 bullish 阈值到底是 55 还是 65`

而是：

`MSS 在 v0.01 究竟是 BEARISH-only 风险阀，还是 BULLISH-required 硬 gate，还是软 gate。`

如果维持当前 `BEARISH-only` 语义，那么后续更有信息量的实验应转向：

1. `bearish_threshold` 敏感性
2. `+IRS` Top N 强度
3. `BULLISH-required` / 软 gate 的独立实验

---

## 5. 下一步建议

1. 保持当前 `65 / 35` 为默认执行阈值，不急着只因短窗结果而改默认。
2. 新增一轮 `gate mode` 消融：
   - `bearish_only`（当前）
   - `bullish_required`
   - `soft_gate`
3. 在同一窗口下继续比较 `IRS Top 10 / 15 / 20`。
