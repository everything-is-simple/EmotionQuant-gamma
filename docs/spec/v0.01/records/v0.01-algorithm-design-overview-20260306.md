# v0.01 当前版算法设计总览

日期：`2026-03-06`

## 1. 这组文档为什么要新建

当前仓库里其实有两套不同层级的算法材料：

1. `docs/reference/core-algorithms/`
   这是一套历史上更完整、更理想化的算法全集，覆盖 `MSS / IRS / PAS / Validation / Integration` 五大块，粒度很细，但它不是当前 v0.01 的可执行口径。
2. `docs/design-v2/`
   这是当前系统级 SoT，负责冻结模块边界、执行语义和版本范围；但它不是“当前实现版算法细案”，尤其没有把我们最近暴露出来的 `MSS baseline`、`IRS 行业桶`、`PAS detector` 这些现实问题写透。

所以需要一套第三层文档：

3. `docs/spec/v0.01/records/` 下的“当前版算法设计”
   这套文档只回答一件事：
   **v0.01 今天真正跑在代码里的 MSS / IRS / PAS，到底是怎么定义的，卡在哪，下一步怎么校准。**

## 2. 文档边界

这组文档的边界是：

- 不改 `design-v2` 的 Frozen 执行语义
- 不回到旧版 `core-algorithms` 的大而全体系
- 只写当前能跑的算法定义、当前已经暴露的风险、以及下一步可执行的校准路径

因此它的定位不是新的系统 SoT，而是：

`v0.01 当前实现级算法基线`

## 3. 当前口径与旧版口径的差异

### 3.1 MSS

旧版 `core-algorithms` 中，MSS 不只是一个 gate，更像一套完整的市场温度全景系统，输出 `temperature / cycle / trend / position_advice`。

当前 v0.01 不是这样。

当前实现只保留：

- `score`
- `signal`（`BULLISH / NEUTRAL / BEARISH`）

也就是说，v0.01 的 MSS 是一个**可开关市场 gate**，不是一整套市场状态机产品。

### 3.2 IRS

旧版 `core-algorithms` 中，IRS 是多因子行业轮动系统，包含：

- 相对强度
- 连续性
- 资金流向
- 估值
- 龙头因子
- 行业基因库

当前 v0.01 只保留两个因子：

- 相对强度（55%）
- 资金流向（45%）

当前 IRS 的职责也很克制：

`给 Selector 提供行业排序，不直接下交易动作。`

### 3.3 PAS

旧版 `core-algorithms` 中，PAS 是一套个股机会评分系统，核心产物是：

- `pas_score`
- `opportunity_grade`
- `direction`
- `risk_reward_ratio`

当前 v0.01 已经明确放弃这条路线，改成了更好的架构：

`PAS = detector registry + pattern detectors`

这也是当前系统里最健康的一块设计。

也就是说，PAS 不再是“一个大分数”，而是“多个形态检测器统一装配”。

## 5. 这组文档怎么用

建议后续统一按这个顺序使用：

1. `system-baseline.md`
   看执行红线与版本范围
2. `selector-design.md` / `strategy-design.md`
   看系统级边界和职责
3. 本组“当前版算法设计”
   看当前实现到底怎么算、哪里还没收口
4. `core-algorithms`
   只作为历史参考，不直接拿来当 v0.01 可执行口径

## 6. 当前最关键的三个算法现实问题

截至 2026-03-06，这三个问题已经不是猜测，而是有证据的：

1. MSS 当前全区间 `768/768` 个交易日全部是 `NEUTRAL`
   说明 `baseline` 仍是占位参数，gate 目前没有信息量。
2. IRS 当前真实行业桶是 `111`
   这和 `design-v2` 中写的 `31` 个申万一级行业不一致，必须做口径决策。
3. 全周期三场消融超过 `75` 分钟未收口
   说明当前回测/selector 主链的性能还不足以支撑 Week2 的“全周期 A/B/C 对照”。

## 7. 结论

结论不是“旧版算法文档没用”，而是：

`旧版 core-algorithms 适合当参考全集；当前 v0.01 需要一套面向现实现实与现有代码的算法设计。`

这就是本组文档的目的。
