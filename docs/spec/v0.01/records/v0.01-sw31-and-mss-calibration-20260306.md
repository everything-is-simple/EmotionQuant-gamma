# v0.01 SW31 收口与 MSS baseline 校准记录

**日期**: 2026-03-06  
**阶段**: Week2 Selector / Strategy  
**状态**: completed

---

## 1. 背景

Week2 分布审计暴露出两类问题：

1. IRS 运行链路虽然已切到申万一级优先，但 raw 源库与执行库仍未真正收口到正式 `SW2021 L1 31` 行业。
2. MSS 已有 zscore 框架，但 `MSS_BASELINE` 仍是占位参数，导致 `all NEUTRAL` 结论缺乏解释力。

---

### 2.1 SW31 raw 补采与清洗

- `scripts/data/bulk_download.py` 改为对 `raw_index_classify/raw_index_member` 走 `SW2021 L1 + index_member_all(Y/N)` 正式链路

  - 优先保留显式 `out_date`
  - raw bootstrap 与运行时 helper 统一按同一规则收口

### 2.2 执行库同步

- 实际 raw 库完成 SW31 快照补采
- 实际执行库重新执行：
  - `fetch --from-raw-db ... --refresh-stock-info-only`
  - `build --layers l2,l3 --force`

### 2.3 IRS taxonomy 收紧

- `clean_industry_daily()` 不再把 `stock_basic.industry` 直接落入行业聚合

### 2.4 MSS baseline 校准

- `src/selector/mss.py` 抽出：
  - raw factor frame
  - baseline calibration
  - score / threshold sensitivity
- 新增 `scripts/backtest/run_mss_baseline_calibration.py`
- `src/selector/baseline.py` 已回写真实经验 baseline

## 4. 结论

1. IRS 的主问题已经从“行业桶定义错误”切换成“Top N 强度如何设置”。
2. MSS 的主问题已经从“baseline 占位”切换成“`65` 阈值是否过紧”。
3. 旧结论 `MSS all NEUTRAL` 已失效，新的正式结论是：

`MSS 已恢复有限区分度，但在 65 阈值下仍然过稀疏。`
