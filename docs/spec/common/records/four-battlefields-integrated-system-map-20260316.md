# 当前四战场集成系统图

**状态**: `Active`  
**日期**: `2026-03-16`  
**对象**: `第一战场到第四战场的当前集成系统、可用边界与后续整合工作`

---

## 1. 目标

本文只回答三件事：

1. 四个战场当前整合起来，到底已经形成了一套什么系统
2. 这套系统现在能不能直接上手用
3. 如果要把四个战场真正吸收成一个统一默认系统，还缺什么工作

---

## 2. 当前已经形成的系统

当前四个战场整合后，已经形成的是一套：

`BOF-centric + capital-disciplined + context-aware + evidence-gated`

的波段交易操作系统。

它不是“大一统打分机”，而是四层分工明确的系统：

1. 第一战场负责运行骨架与默认路径治理
2. 第二战场负责 `alpha provenance / quality gate / exit damage diagnosis`
3. 第三战场负责 `position sizing / partial-exit` 控制层
4. 第四战场负责 `historical wave ruler / mirror ruler / conditioning readout / MSS-IRS retire boundary`

---

## 3. 四个战场各自留下了什么

| 战场 | 回答的问题 | 当前正式留下来的结论 | 当前集成状态 |
|---|---|---|---|
| 第一战场 | 系统如何端到端运行 | 主线与研究线边界、执行骨架、治理账本、默认运行纪律 | `runtime backbone` |
| 第二战场 | 买什么、何时买、为什么打 | `BOF_CONTROL` 继续保持唯一稳定 baseline；主伤害不在“买错”而在部分 exit damage | `entry truth layer` |
| 第三战场 | 打多大、怎么退 | `FIXED_NOTIONAL_CONTROL = canonical sizing baseline`；`FULL_EXIT_CONTROL = canonical exit control baseline`；partial-exit 仅形成 retained queue | `capital control layer` |
| 第四战场 | 当前所处历史环境是什么 | `self-history ruler + market/industry mirror + five-pattern conditioning + MSS/IRS retire boundary` | `context and governance layer` |

---

## 4. 当前默认运行系统

当前真正已经升格为默认运行口径的，不是“四战场所有研究结论一次性全开”，而是：

`legacy_bof_baseline + no IRS + no MSS + current full-exit semantics`

若把第三战场已经收实的控制层一并纳入当前最稳的实际操作口径，可压成：

`Selector 初选 -> BOF baseline entry -> FIXED_NOTIONAL_CONTROL -> FULL_EXIT_CONTROL -> Broker 执行`

这意味着：

1. 现在已经可以上手用
2. 但当前可直接使用的是 `baseline operating system`
3. 不是“第二、三、四战场所有 retained 结论已自动升格为默认参数”

---

## 5. 当前还没有自动并入默认系统的部分

以下内容当前仍属于研究线结论，不得误写成默认运行参数：

1. 旧 `IRS / MSS` 的任何默认回归
2. `Normandy` retained / watch 分支的自动 promotion
3. `Positioning` partial-exit retained queue 的默认升格
4. `Gene` 条件层变成硬过滤器
5. `Gene` 镜像尺直接改写当前 Broker 或 Selector 默认逻辑

一句话说：

`四战场已经形成完整认知系统，但还没有完成“全量研究结论 -> 单一默认运行系统”的最终迁移。`

---

## 6. 如果要真正整合成一个统一默认系统，还需要什么工作

当前至少还需要四类工作：

1. `explicit_mainline_migration_package`
   - 把第二、第三、第四战场里真正要升格的子集显式迁回 `blueprint/` 与 `docs/spec/v0.01-plus/`
2. `promoted subset freeze`
   - 明确哪些研究结论只是解释层，哪些才是真正进入默认运行的参数和合同
3. `integrated end-to-end validation`
   - 对拟升格组合做正式长窗 replay、对照矩阵和 go/no-go 裁决
4. `operating runbook refresh`
   - 把运行链路、监控口径、风险开关、人工介入边界写成新的统一 runbook

如果不做这四步，研究线结论就仍然只是“知道了什么”，还不是“默认怎么跑”。

---

## 7. 现在能不能直接上手用

可以，但要分清两层意思。

### 7.1 作为当前交易系统去上手

可以。

当前可直接上手的系统是：

1. `legacy_bof_baseline`
2. `FIXED_NOTIONAL_CONTROL`
3. `FULL_EXIT_CONTROL`
4. `Gene` 作为环境解释与复盘标签

### 7.2 作为“四战场完全融合的一体化默认系统”去上手

还不可以。

因为当前还缺：

1. 显式 migration package
2. 升格子集冻结
3. 一体化 replay / gate
4. 新 runbook

---

## 8. 正式结论

当前四战场整合后的正式结论固定为：

1. 我们已经拥有一套可运行、可验证、可继续演化的交易系统
2. 这套系统当前的核心形态是 `BOF baseline + capital control + historical context + evidence governance`
3. 现在已经可以按当前 baseline 上手使用
4. 但若要把四个战场全部吸收成新的统一默认系统，下一步必须走 `explicit_mainline_migration_package`
