# EmotionQuant 系统演化历史

**状态**: `Active`  
**日期**: `2026-03-16`  
**对象**: `系统演化史 / 四个战场的开辟逻辑 / 产品拼装方法`

---

## 1. 这套系统不是按原设计长出来的

EmotionQuant 的形成，并不是一张总蓝图被顺利实现的结果。

它更接近另一种过程：
先试图把很多东西一次合成，发现合不起来；  
再被具体问题逼着拆开；  
每次只盯住一个局部，只解决一个局部；  
把这些被梳理通顺的小块，小心拼回一件能够运行、能够验证、能够继续演化的生产工具。

如果要用一句话概括这段历史，那就是：

`这不是先有完美设计，再去实现；而是先遇到真实问题，再被迫化整为零，一点一点把系统从混沌里拽出来。`

---

## 2. “战场”不是雄心设计，而是生存策略

最初的愿望其实很简单：希望把 `PAS + MSS + IRS` 直接合成成一个足够强的系统。

但现实很快证明，这条路并不成立。问题不在于完全没有成果，而在于：

1. 太多模块在同一层面纠缠在一起
2. 一个结论里同时混入 alpha、过滤、仓位、退出与市场背景
3. 一旦结果不好，很难知道到底是哪一层出了问题
4. 参数越来越多，但系统并没有因此变得更清楚

于是，所谓“几个战场”的开辟，并不是原本就想这样设计，而是被问题逼出来的拆解法。

这个拆解法的核心只有三个动作：

1. 把问题拆开，不让不同层次继续混在一起
2. 一次只回答一个问题，不追求一步到位
3. 每拆出一层，就先做局部验证，确认这块东西到底有没有真货

所以，四个战场不是四个平行爱好，而是系统在自救过程中形成的四个功能层。

---

## 3. 第一战场：系统骨架先被打出来，但两次主攻都失望了

第一战场原本承担的是“把系统直接做出来”的任务。

当时的设想是：

1. `PAS` 负责候选或结构性约束
2. `MSS` 负责市场状态与阶段判断
3. `IRS` 负责相对强弱与排序
4. 三者合成之后，应该能形成一个相当有力的主系统

但实际结果并不理想。无论是 `v0.01`，还是后来的 `v0.01-plus`，都没有证明这条主攻路线已经拿到了足够可靠的 alpha。

这两次战役可以说都输了。

但第一战场并不是空手而回。它留下了真正重要的东西：

1. 数据链路
2. 回测框架
3. 治理账本
4. 主线与研究线分离的结构
5. 一套能持续接新证据、做升级决策、控制默认路径的工程骨架

换句话说，第一战场没有交出最终答案，但交出了系统的骨架。

---

## 4. 第二战场：Normandy 重新回答“为什么打”

当第一战场没能证明 `MSS + IRS` 就是主 alpha 之后，第二战场才被迫开辟出来。

第二战场的任务不是做全系统，而是只回答一件事：

`为什么打。`

进一步说，就是：

1. 买什么
2. 何时买
3. 哪一种入场结构才真正带来 alpha

这条线后来逐渐收敛到 `Normandy`，也重新确立了 `BOF` 的地位。

第二战场的意义，不是证明系统已经完成，而是把“主 alpha / 主信号”从一团混合物里分离出来。它解决的是出手理由，而不是仓位大小，也不是退出编排。

它回答的是：`该不该打。`

---

## 5. 第三战场：Positioning 回答“打多大，怎么退”

当第二战场把“为什么打”分离出来之后，另一个问题马上浮现出来：

`即便该打，应该打多大，又该怎么退？`

此前系统对退出的处理比较粗：

1. 初始止损
2. 移动止损
3. 到点就卖

这种处理方式并不一定错，但它把所有头寸都压成了一种退出语言，没有区分：

1. 是否应该先落袋一部分
2. 是否应该保留核心头寸
3. 不同退出编排会如何改变收益分布、回撤和存活率

第三战场因此被开辟出来。

它不重新讨论买点，而是独立研究：

1. `position sizing`
2. `partial-exit`
3. `scale-out`
4. “买多少、卖多少、怎么退”的控制问题

所以第三战场回答的是：`打多大，怎么退。`

---

## 6. 第四战场：Gene 回答“这一段走势在它自己的历史里算什么级别”

第四战场的开辟，来源于另一个越来越明显的问题：

系统知道一些信号，但并不知道“这一段走势究竟处在什么历史位置上”。

这正是第四战场要补的空白。

它不直接回答买点，也不直接回答仓位，而是要建立一把历史标尺，去衡量：

1. 什么是趋势
2. 什么是波段
3. 什么是波段主流
4. 什么是趋势逆流
5. 什么是转折
6. 什么是新高 / 新低

第一版只抓最少的原语：

1. 波动幅度
2. 波动时间

在涨波段中记录：

1. 新高第几次
2. 新高发生时间
3. 新高价格
4. 新高总数量
5. 新高密度

跌波段则做对称处理。

然后再用简单统计去回答三个问题：

1. 当前波段在该股自身历史里排第几
2. 当前波段在全市场同类波段里排第几
3. `magnitude / duration / extreme_density` 这三个子因子，谁对后续延续或衰竭的解释力更强

所以第四战场回答的是：`当前这只股票的这段走势，在它自己的历史里算什么级别。`

---

## 7. 四个战场之间的关系

这四个战场不是并列的功能菜单，而是一条被问题倒逼出来的演化链：

1. 第一战场留下系统骨架
2. 第二战场分离主 alpha
3. 第三战场分离仓位与退出控制
4. 第四战场补上历史波段标尺

它们各自回答的是不同问题：

1. 第一战场：系统框架能否成立
2. 第二战场：买什么，何时买，为什么打
3. 第三战场：买多少，卖多少，怎么退
4. 第四战场：这段走势在其历史里算什么级别

这也意味着，任何一个战场都不应该假装自己包办全部问题。

---

## 8. 这套产品的真正形成方式

EmotionQuant 的产品化，不是“把一个大公式越调越精”。

它真正的形成方式是：

1. 先把大问题拆成几个不能再混的层次
2. 每一层只解决自己的问题
3. 把梳理通顺的小块重新拼起来
4. 拼完以后，不急着宣布胜利，而是谨慎验证

这个“谨慎验证”非常关键。

因为历史记录会被改写，市场也会变，任何局部成果都有可能在更大范围里失效。所以系统不能靠叙事成立，只能靠证据链成立。

换句话说，这套系统不是“设计出来”的，更准确地说，它是：

`被问题反复逼退、逼拆、逼校正之后，慢慢拼出来的。`

---

## 9. 这段历史应当如何保存

这段系统演化史最值得保存的，不是某个指标细节，而是这条方法论：

1. 遇到混乱，不再试图一次解释全部
2. 先拆层，先定边界，先回答单个问题
3. 每解决一点，就把这一点写清楚、冻结下来
4. 只有经过验证的部分，才允许进入生产工具

这就是 EmotionQuant 到目前为止最真实的演化路径。

它不是从一个宏伟设计中自然生长出来的，
而是在一系列失败、拆解、澄清、重组与验证之后，
才逐步拥有了今天这副形状。
