## 1. 目的

这份文件不是读书摘要，也不是某条研究线的阶段总结。

它的作用只有一个：

把四个战场在后续设计、代码、回测、复盘和治理中反复使用的核心词汇，先统一到同一套语言里，避免同一个词在不同文档、不同模块、不同阶段里指向不同东西。

本文件先做第一层冻结：

1. 统一系统级术语
2. 统一四战场级术语
3. 明确哪些词是正式术语，哪些只是叙述性说法

本文件暂不处理的内容：

1. `对象 / 指标 / 信号` 的边界细表
2. 第四战场的书本定义与代码偏移比对
3. 某个策略参数是否应当升格到默认主线

这些内容分别由后续两份文件处理：

1. `four-battlefields-object-indicator-signal-boundary-20260317.md`
2. `gene-foundational-definition-freeze-20260317.md`

## 3. 总原则

四战场统一术语，遵循以下五条原则：

1. 先定义对象，再定义对象上的状态、指标和信号。
2. 先定义概念，再定义设计，再定义代码。
3. 同一个词在全系统内只能保留一个主定义。
4. 若系统对原书做了工程转译，必须明确标注“这是系统定义，不是原书原句”。
5. 没有进入本表的词，不得默认当作全系统共识词。

## 4. 系统总词汇

### 4.1 系统

**定义**：由运行骨架、策略逻辑、仓位控制、历史语境和治理边界共同组成的完整交易操作系统。  
**理由**：系统不是单一策略，也不是单一指标；它包含从输入到执行再到审计的完整闭环。  
**当前口径**：当前正式系统描述为  
`legacy_bof_baseline + FIXED_NOTIONAL_CONTROL + FULL_EXIT_CONTROL + Gene sidecar boundary + MSS/IRS retirement boundary`

### 4.2 主线

**定义**：当前被治理层正式承认为默认开发与默认审计对象的设计链。  
**不是**：所有研究线的总和。  
**当前主线权威层**：`blueprint/` + `docs/spec/v0.01-plus/`

### 4.3 默认运行路径

**定义**：当前实际允许按正式口径运行、回测、审计和对外说明的 runtime 组合。  
**当前默认运行路径**：  
`BOF baseline + FIXED_NOTIONAL_CONTROL + FULL_EXIT_CONTROL`

### 4.4 研究线

**定义**：不直接改写主线 SoT、用于回答独立问题域的探索性设计与验证路径。  
**当前研究线**：`normandy`、`positioning`、`gene`

### 4.5 治理

**定义**：决定哪些内容可以进入正式口径、哪些只能停留在候选或观察层的规则与记录机制。  
**载体**：`docs/spec/common/` 与 `docs/spec/v0.01-plus/`

### 4.6 证据

**定义**：能支撑某个正式结论的可复核输出，包括回测结果、统计表、验证脚本输出、正式 evidence 文件。  
**不是**：口头印象、单次截图、未归档实验结果。

### 4.7 Record

**定义**：对某阶段结论、裁决、边界和状态切换的正式文字记录。  
**作用**：把“发生过”变成“可追溯”。

### 4.8 Promotion

**定义**：某个研究对象、参数、边界或设计子集，经过治理批准后进入正式主线口径。  
**注意**：被验证过，不等于被 promotion。

### 4.9 Sidecar

**定义**：可以伴随默认运行路径输出、用于解释、归因、监控或看板展示，但不参与 runtime 硬决策的附属层。  
**当前例子**：`Gene context sidecar`

### 4.10 Boundary

**定义**：系统明确规定某个模块“可以做什么 / 不可以做什么”的约束线。  
**当前重要边界**：
- `Gene = sidecar only`
- `MSS/IRS = retired from default runtime semantics`

## 5. 四战场总词汇

### 5.1 第一战场

**正式名称**：运行骨架与主线设计层  
**仓库位置**：`blueprint/`  
**核心问题**：系统如何被定义、运行、审计、迁移与升格。  
**职责**：回答“系统怎么跑，什么才算正式默认链路”。

### 5.2 第二战场

**正式名称**：Alpha 与伤害诊断层  
**仓库位置**：`normandy/`  
**核心问题**：该打什么、为什么打、伤害来自哪里。  
**职责**：回答“买什么 / 何时打 / alpha 真假如何验证”。

### 5.3 第三战场

**正式名称**：仓位与退出控制层  
**仓库位置**：`positioning/`  
**核心问题**：打多大、怎么退、如何控制损失与回吐。  
**职责**：回答“仓位怎么给 / 退出怎么做 / 风险怎么收”。

### 5.4 第四战场

**正式名称**：价格对象历史语境层  
**仓库位置**：`gene/`  
**核心问题**：当前这段价格对象，在它自己的历史里处于什么位置。  
**职责**：回答“这段走势在历史上算什么级别，以及这些历史语境如何解释其他战场”。

## 6. 跨战场核心术语

### 6.1 对象

**定义**：系统认为“真实存在、可被单独追踪、可被赋予属性”的单位。  
**例子**：标的、市场、行业、pivot、wave、trade、portfolio。  
**原则**：对象先于指标存在。

### 6.2 状态

**定义**：对象在某一时点或某一阶段所处的可枚举条件。  
**例子**：`UP / DOWN`、`active / completed`、`mainstream / countertrend`。

### 6.3 事件

**定义**：在某一时点发生、会改变对象状态或引发后续判定的离散发生。  
**例子**：突破、失败测试、确认拐点、入场、止损、退出。

### 6.4 指标

**定义**：对对象某一属性的数值化测量结果。  
**例子**：收益率、波动、百分位、rank、PF、MDD。  
**注意**：指标不是对象，也不是事件。

### 6.5 信号

**定义**：系统基于对象、状态、事件和指标，得出的可执行或可判断输出。  
**例子**：可买、拒单、减仓、退出、观察。  
**注意**：信号通常是决策输出，不是原始事实本身。

### 6.6 基线

**定义**：当前用来做对照、归因、迁移审计和默认运行的正式参照路径。  
**例子**：`legacy_bof_baseline`

### 6.7 候选

**定义**：已进入正式讨论或验证，但尚未被升格为默认的一组设计组合。  
**注意**：候选必须有边界，不允许无限膨胀。

### 6.8 观察层

**定义**：允许输出、允许看、允许复盘，但暂不允许直接驱动默认 runtime 的层。  
**当前例子**：`Gene sidecar`

### 6.9 硬门

**定义**：会直接阻断某笔交易或某段 runtime 流程的决策条件。  
**注意**：凡是硬门，都必须有比观察层更高的证据要求。

### 6.10 审计

**定义**：验证系统是否按声明的边界、配置和路径运行的检查机制。  
**关注点**：链路是否真实、trace 是否完整、是否发生越权引用。

## 7. 第一战场术语

### 7.1 运行骨架

**定义**：从数据到 Selector、到 Broker、到结果归档的最小可运行闭环。

### 7.2 默认链路

**定义**：当前正式允许生产、回测和对外描述时使用的主执行链。

### 7.3 Migration Package

**定义**：把某些研究结论从研究线迁回主线时使用的正式迁移包。  
**作用**：防止研究结论绕过治理直接入主线。

### 7.4 Promoted Subset

**定义**：研究线中被挑选出来、允许进入主线候选验证的子集。  
**注意**：不是研究线的全部。

---

## 8. 第二战场术语

### 8.1 Alpha

**定义**：某个 setup 或 trigger 所携带的、可在重复样本中验证的超额收益能力。  
**注意**：alpha 不是单笔赚钱，也不是“看起来顺眼”。

### 8.2 Setup

**定义**：一类可重复识别的市场结构场景。  
**例子**：某类突破、某类测试、某类回调接受。  
**作用**：回答“什么局面值得关注”。

### 8.3 Trigger

**定义**：在某个 setup 内，具体触发入场判定的动作或价格行为条件。  
**作用**：回答“什么时候真正出手”。

### 8.4 Provenance

**定义**：一笔交易为何出现、来自哪条检测路径、由什么结构与条件构成的可追溯来源说明。

### 8.5 伤害诊断

**定义**：把系统亏损、回吐或失真拆解到入场、过滤、退出、仓位、执行等层次上的分析过程。

---

## 9. 第三战场术语

### 9.1 风险单位

**定义**：系统分配仓位与评估单笔容忍损失时使用的统一度量单位。

### 9.2 测试单

**定义**：在尚未确认应大规模暴露前，以较小风险暴露先行试探市场接受度的仓位。

### 9.3 正式单

**定义**：在 setup、trigger、环境或已有利润条件更充分时允许配置的标准风险暴露。

### 9.4 仓位法

**定义**：从风险预算映射到实际下单数量或资金规模的方法。  
**例子**：`FIXED_NOTIONAL_CONTROL`、`Williams fixed risk`

### 9.5 分批

**定义**：把一次交易拆成多个执行腿，以分层进入或分层退出。  
**注意**：分批是执行结构，不自动等于更优。

### 9.6 退出

**定义**：交易生命周期内使持仓归零或降低暴露的动作集合。  
**类型**：止损、止盈、分批退出、移动止损、全平。

## 10. 第四战场术语

### 10.1 价格对象

**定义**：第四战场中被视为可计算历史结构的对象。  
**当前对象层**：个股、行业、市场。

### 10.2 趋势

**定义**：价格对象在某一层级上表现出的方向性结构。  
**注意**：趋势先是结构，不是指标。

### 10.3 趋势层级

**定义**：同一价格对象上并行存在的不同时间层级的趋势视角。  
**当前状态**：系统已确认这是必需概念，但代码尚未完整升格为一等对象。

### 10.4 拐点

**定义**：经过确认、足以参与后续波段构造和结构判定的局部高低点。  
**注意**：拐点是工程实现的基础对象，不等于肉眼随意看到的高低点。

### 10.5 波段

**定义**：两个相邻异侧确认拐点之间的单向价格段。  
**作用**：是第四战场幅度、期限、结构和历史分位的最基础统计对象。

### 10.6 主流 / 逆流

**定义**：某个波段相对于上一级趋势方向的归属。  
**注意**：这一定义必须依附于趋势层级，不能只看单段方向。

### 10.7 1-2-3

**定义**：用于确认趋势改变的一组三条件结构语法。  
**当前治理口径**：它首先是“趋势改变确认法”，其次才可能被交易触发器消费。  
**注意**：不得把它简化成“任何三段波形态”。

### 10.8 2B

**定义**：对前高或前低的失败性穿越与回收，属于失败测试事件。  
**当前治理口径**：它首先是失败确认事件，不是神奇预测器。  
**注意**：时间窗与趋势层级有关。

### 10.9 历史寿命

**定义**：某类波段在历史统计中常见的期限与幅度分布。  
**作用**：用于回答“这段走势更像历史早段、中段还是末段”。  
**注意**：它是概率尺，不是确定性预言。

### 10.10 历史尺

**定义**：把当前对象放回其历史样本分布中，衡量其相对位置的统计标尺。  
**当前系统口径**：第四战场已实现的是“个股自历史尺”的工程转译版本。

### 10.11 镜像

**定义**：把个股层已定义好的历史尺语义，映射到行业和市场对象上的过程。  
**注意**：镜像不是简单复制分数，而是同语义的对象级迁移。

### 10.12 条件层

**定义**：用于解释某类 trigger 在什么历史环境下更值得打、更不值得打的附属分析层。  
**当前作用**：服务第二战场，不直接充当默认硬门。

## 11. 当前冻结结论

截至本文件版本，以下口径正式成立：

1. 四战场是分工系统，不是四套互相覆盖的系统。
2. 第一战场定义运行与治理边界，第二战场定义 alpha 与伤害诊断，第三战场定义仓位与退出控制，第四战场定义历史语境。
3. `Gene` 当前正式身份是 `sidecar / context / attribution layer`，不是默认 runtime hard gate。
4. `MSS / IRS` 的旧默认 runtime 语义已退役，不得在术语层被悄悄复活。
5. 所有新设计、新代码、新卡片，若要跨战场共用词汇，必须优先服从本表。
