## 4. 冻结原则

### 4.1 对象优先

先定义对象，再定义对象上的状态、事件、指标。

### 4.2 书定义优先于工程近似

如果代码为了机械化而采用了近似定义，必须明确标注“这是系统实现定义”，不能假装等同于原书原义。

### 4.3 第四战场先做语义层，不先做交易层

第四战场首先提供的是历史语义、结构语义和条件语义；它不是默认 runtime 的直接开关层。

### 4.4 统计层不得反向篡改定义层

某个分位标签即使统计上看起来有用，也不能反过来重写 `趋势 / 波段 / 1-2-3 / 2B` 的原始定义。

## 5. 必须冻结的基础定义

## 5.1 趋势层级

**书定义**

趋势不是单层对象。短期、中期、长期三种趋势可以同时存在；投机者通常以中期趋势为主要观察对象，但不能因此否认层级结构本身。

**系统冻结定义**

`趋势层级` 是第四战场的一级基础概念。  
系统至少要承认：

1. 趋势是分层的
2. 同一时刻可以并存不同层级趋势
3. `主流 / 逆流` 必须相对于更高一层趋势来判定

第一版实现可以先聚焦中期层，但术语和 schema 不能否认层级存在。

**当前代码实现**

[`gene.py`](../../../../src/selector/gene.py) 当前只有方向和单层 `major_trend` 语义，没有正式 `trend_level` 对象。

**偏移 / 问题**

这意味着当前实现能表达“当前方向”，但还不能严格表达“相对于哪一层趋势的方向”。

**冻结结论**

`trend_level` 是正式基础定义，当前代码尚未完整实现。  
后续所有 `mainstream / countertrend / 2B 时间窗 / 1-2-3` 语义，都必须允许挂接到趋势层级。

---

## 5.2 趋势

**书定义**

趋势首先是结构，不是分数；上升趋势对应更高高点和更高低点，下降趋势相反。

**系统冻结定义**

`趋势` 是价格结构对象上的方向性状态，不是指标，不是分位，不是一个综合分数。

第四战场里的趋势，至少要满足：

1. 有方向
2. 有层级归属
3. 可通过一组已确认 pivot / wave 结构被追踪

**当前代码实现**

当前实现通过 `wave direction + major_trend` 来近似表达趋势状态。

**偏移 / 问题**

这是一种可运行的工程简化，但还不是完整的结构化趋势定义。

**冻结结论**

趋势的正式定义保留为“结构状态”；  
`gene_score`、percentile、band 一律不得代替趋势定义。

---

## 5.3 拐点 Pivot

**书定义**

趋势判断依赖先确认高低点。没有可确认的结构拐点，就没有可以被稳定统计的趋势段。

**系统冻结定义**

`pivot` 是第四战场的基础结构对象，用于：

1. 切分波段
2. 识别新高新低
3. 识别失败测试
4. 支撑 `1-2-3 / 2B` 的后续确认

**当前代码实现**

当前代码采用 `5-bar fractal + 2-bar confirmation` 的机械定义，并在相邻同侧极值中保留更极端者。

**偏移 / 问题**

这不是原书唯一指定的做法，而是为程序化而采用的机械确认法。

**冻结结论**

`pivot` 作为基础对象成立；  
当前机械定义允许继续使用，但必须明确标注为“系统机械实现定义”，不是书中唯一合法定义。

---

## 5.4 波段 Wave

**书定义**

幅度、期限、主流/逆流、反转与延续，都必须依附于具体走势段落。没有波段对象，后续统计没有着力点。

**系统冻结定义**

`wave` 是第四战场的核心基础对象。  
统一定义为：

`由两个相邻、方向相反、已确认的 pivot 所界定的单向价格段`

它是后续几乎全部统计的基底：

1. 幅度
2. 持续天数
3. 新高新低计数
4. 主流/逆流
5. 结构标签
6. 历史分位

**当前代码实现**

当前实现正是按相邻异侧确认 pivot 构造 wave。

**偏移 / 问题**

这部分和设计基本一致，偏移相对最小。

**冻结结论**

`wave` 正式保留为第四战场最基础统计对象。  
后续任何历史尺、镜像尺、条件层，都不得绕过 `wave` 直接凭日线分数硬造语义。

---

## 5.5 主流 / 逆流

**书定义**

逆流不是噪声，而是相对于上一级趋势的重要折返。难点不在于发现涨跌，而在于区分“次级折返”和“更小级别噪声”。

**系统冻结定义**

`mainstream / countertrend` 是波段相对于更高层趋势的结构状态，不是独立对象。

它必须回答的是：

`这段 wave 是顺着上一级趋势，还是逆着上一级趋势`

**当前代码实现**

当前实现按单层 `major_trend` 判定 `MAINSTREAM / COUNTERTREND`。

**偏移 / 问题**

由于没有显式 `trend_level`，当前主流/逆流判定过粗，容易把复杂回调、局部噪声和真正逆流混在一起。

**冻结结论**

主流/逆流这个概念保留，但其正式语义必须绑定“上一级趋势”。  
当前代码可作为第一版近似实现，不得被误读为最终语义完成版。

---

## 5.6 新高 / 新低

**书定义**

新高新低必须相对于先前已确认的重要高低点，而不是仅仅相对于临近日线波动。

**系统冻结定义**

`new_high / new_low` 是相对于已确认 pivot 或已确认结构极值的事件性判断基础。

它们本身不是买卖信号，而是：

1. 趋势延续的证据来源
2. `2B` 失败测试的前置条件
3. wave 极端密度统计的上游事件

**当前代码实现**

当前实现会记录极值候选、极端刷新与失败回落。

**偏移 / 问题**

整体方向正确，但严格参照层级仍然受限于当前单层趋势近似。

**冻结结论**

新高新低保留为事件性基础语义；  
它们不是独立对象，更不是直接交易信号。

---

## 5.7 2B

**书定义**

`2B` 本质上是失败性新高 / 新低：价格先越过前高或前低，然后在一个与趋势层级相适应的时间窗内重新回到关键参考位之内，构成失败测试。

原始时间窗具有层级差异：

1. 短期通常约 `1` 天内
2. 中期通常约 `3-5` 天
3. 长期通常约 `7-10` 天

**系统冻结定义**

`2B` 是结构失败事件，不是神奇预测器。  
它回答的是：

`原趋势的继续是否被失败性测试所否定`

系统里 `2B` 的正式语义包含四点：

1. 必须先有前高/前低参考
2. 必须先发生越界
3. 必须再发生回落/回升失守
4. 确认时间窗必须允许因趋势层级不同而不同

**当前代码实现**

当前代码已经能识别失败性新高 / 新低，但确认窗固定为 `3` 根 bar。

**偏移 / 问题**

事件语义基本成立，但时间定义被压平了，没有体现层级差异。

**冻结结论**

`2B` 正式冻结为“层级相关的失败极值事件”。  
当前固定 `3` bar 的实现只能视为中期层近似版，不能视为第四战场最终定义。

---

## 5.8 1-2-3

**书定义**

`1-2-3` 不是简单的三段波形态，而是趋势改变的三条件确认法：

1. 趋势线被破坏
2. 对前高/前低的再测试失败
3. 反向突破前一个重要短期回档点

三条件可以有不同先后，但三者共同构成“趋势已经改变”的确认语法。

**系统冻结定义**

`1-2-3` 是结构性确认框架，不是任意的 `A-B-C` 三段图形。

系统里它的正式定义应拆成三个可落库的组成部分：

1. `trendline_break`
2. `failed_extreme_test`
3. `prior_pivot_breach`

而 `123_STEP1 / STEP2 / STEP3` 只是系统对这三步的落库标签，不应反过来篡改书定义。

**当前代码实现**

当前代码主要通过三段相邻 wave 的结构关系来近似识别 `1-2-3`。

**偏移 / 问题**

这是一种可用的工程化简，但它把“确认语法”压缩成了“三段波形态”，语义比原书更窄，也更容易误判。

**冻结结论**

`1-2-3` 的正式冻结定义，服从“三条件确认法”；  
当前代码实现只能视为第一版近似 detector，不得被当作最终语义定义。

---

## 5.9 历史寿命

**书定义**

趋势或波段的“寿命”不是精确预测，而是通过历史样本的幅度与期限分布，估计当前走势更像处于早段、中段还是末段，并作为风险判断参考。

**系统冻结定义**

`历史寿命` 是波段对象在其历史分布参照系中的位置解释，不是直接交易信号。

它至少包含两个维度：

1. 幅度位置
2. 期限位置

系统对“寿命”的使用方式是概率性的、分布性的，不承诺精确倒计时。

**当前代码实现**

当前代码已经按同代码、同方向、已完成 wave 样本构造幅度分位、年龄分位和 band。

**偏移 / 问题**

这一步已经从原书的“历史分布参考尺”转译成“个股自历史尺”，属于系统改写，而不是原书照搬。

**冻结结论**

`历史寿命` 正式保留；  
但所有文档必须明确区分：

1. 原书原义：历史分布参考
2. 系统转译：个股自历史尺

---

## 5.10 自历史尺 Self-History Ruler

**书定义**

原书强调的是“通过历史统计分布得到参考标准”，并没有直接给出我们当前这种个股自历史数据库实现。

**系统冻结定义**

`self-history ruler` 是第四战场的系统级转译定义。  
它的正式含义是：

`把同一股票、同一方向、已完成波段构成的历史样本池，当作当前 active wave 的参照系`

它的作用是：

1. 解释当前波段在本股票历史中的位置
2. 给第二战场 trigger 提供环境切片
3. 给第三战场和治理层提供归因背景

它的非作用是：

1. 不直接生成默认 runtime 硬门
2. 不替代第二战场 trigger
3. 不替代第三战场仓位法

**当前代码实现**

当前 `G1 / G2 / G4 / G5 / G6` 整体已经围绕这个定义展开。

**偏移 / 问题**

它不是书的直接原定义，而是系统工程转译；如果不标明这一点，就会造成“借书义给新定义背书”的风险。

**冻结结论**

`self-history ruler` 正式成立，但必须永远带着“系统转译定义”标签存在。

## 6. 当前代码的正式定位

对 [`gene.py`](../../../../src/selector/gene.py) 当前实现，正式定性如下：

1. 它已经足够支撑第四战场 `G0-G8` 的研究闭环

3. 它已经证明 `自历史尺 / 镜像尺 / 条件层` 作为系统转译是可运行的
4. 但它还没有完全实现第四战场最终语义冻结版

当前最明确的语义缺口是：

1. 缺正式 `trend_level`
2. `mainstream / countertrend` 仍是单层近似
3. `2B` 时间窗未按层级区分
4. `1-2-3` 仍是三段波近似，而非完整三条件确认法

所以当前代码的正式口径应写成：

`研究闭环已完成，基础语义冻结已完成第一版，最终语义实现尚未完成。`

## 7. 对四个战场的约束

### 7.1 对第一战场

第一战场不得把第四战场当前标签直接偷升格成默认 runtime 硬门，除非另开正式 migration package。

### 7.2 对第二战场

第二战场的 `BOF / BPB / PB / TST / CPB` 必须被理解为 trigger family；  
第四战场提供的是趋势、转折与寿命语义，而不是替代 trigger。

### 7.3 对第三战场

第三战场可以消费第四战场解释层做 cohort、归因、sidecar 监控；  
未经正式治理，不得把 `gene band / gene score / mirror rank` 直接升格成默认仓位硬规则。

### 7.4 对第四战场自身

第四战场后续若继续开发，优先级应是：

1. 先补定义层缺口
2. 再修 detector 语义
3. 最后才考虑更多统计层

不允许再倒过来先加新 band、新镜像、新打分，再回头补定义。

## 8. 第一版正式结论

第一版冻结结论如下：

1. `wave` 正式确认为第四战场最基础对象
2. `trend_level` 正式确认为不可缺失的基础概念
3. `1-2-3` 正式冻结为“三条件确认法”，不是任意三段波
4. `2B` 正式冻结为“层级相关的失败极值事件”
5. `历史寿命` 正式冻结为概率性历史分布解释，不是精确预测
6. `self-history ruler` 正式冻结为系统转译定义，不是假装等同于原书原义

一句话收口：

`第四战场已经找到可运行的骨架，但要让骨架真正站成“定义学”而不只是“统计学”，还必须把趋势层级、1-2-3、2B 三处语义缺口继续补完。`
