# v0.01-plus 路线图（当前主开发线）

- **执行基线**: `docs/design-v2/01-system/system-baseline.md`（`v0.01 Frozen` 历史基线，用于对照与回退参考）
- **当前设计入口**: `docs/design-v2/03-algorithms/core-algorithms/down-to-top-integration.md`
- **状态来源**: `docs/spec/common/records/development-status.md`
- **文档状态**: Active（当前主开发线；用于替代 legacy top-down）
- **冲突规则**: 涉及 `v0.01` 历史口径时，以 `system-baseline.md` 为准；涉及 `v0.01-plus` 当前主开发线时，以本文件和对应实现卡为准。

## 1. 目标

`v0.01-plus` 的目标不再是“独立实验版验证”，而是作为 `v0.01` legacy top-down 的主线替代版本，完成 `down-to-top / DTT` 链路的实现、证据收口与默认运行路径切换。

本版本的核心使命：

1. 让 `Selector` 退回基础过滤与规模控制。
2. 让 `Strategy` 承担 `BOF -> 后置评分 -> 排序` 的主职责。
3. 让 `legacy_bof_baseline` 降级为对照组与回退路径，而不是继续充当默认主链。

## 2. 版本边界

1. `v0.01 Frozen` 保持不动：`BOF` 单形态、`T+1 Open`、历史正式结果契约字段冻结。
2. `v0.01-plus` 是当前主开发线，默认目标是 `DTT` 主链，而不是继续维护 legacy top-down。
3. `legacy top-down` 仅保留为 compare / rollback 模式，不再作为当前主开发线的默认交付目标。
4. 当前阶段采用“兼容优先”的迁移策略：下游 `Broker / Backtest / Report` 先保持消费兼容，`DTT` 排序与明细通过 `l3_signal_rank_exp` 承载；是否升级正式 `Signal` 契约，后续单独决策。
5. `v0.02` 不承接本次链路替代；当前主线替代工作全部留在 `v0.01-plus` 内完成。

## 3. 主线切换验证矩阵

| variant | Selector | Strategy | 用途 |
|---|---|---|---|
| `legacy_bof_baseline` | `基础过滤 + MSS/IRS 漏斗` | `BOF 单形态触发` | 历史对照组 / 回退参考 |
| `v0_01_dtt_bof_only` | `基础过滤后直接进 BOF` | 仅用 `bof_strength` 排序 | 验证纯后置排序基线 |
| `v0_01_dtt_bof_plus_irs_score` | `基础过滤后直接进 BOF` | `bof_strength + irs_score` | 当前优先候选主链 |
| `v0_01_dtt_bof_plus_irs_mss_score` | `基础过滤后直接进 BOF` | `bof_strength + irs_score + mss_score` | 验证市场/行业双后置加权 |

## 4. 交付范围

1. `roadmap/v0.01-plus-spec-01-selector-strategy.md`：主线实现卡与模块边界。
2. `governance/v0.01-plus-gate-checklist.md`：主线切换 Gate 与回退门。

4. `governance/v0.01-plus-run-artifact-rules.md`：`run_id / variant / artifact` 固定规则。
5. 主线证据统一落在 `docs/spec/v0.01-plus/evidence/`。
6. 主线记录统一落在 `docs/spec/v0.01-plus/records/`。

## 5. Go / No-Go

只有在以下条件同时满足时，才允许把 `v0.01-plus` 切成默认运行路径：

1. 四组对照均能在同一数据快照、同一参数快照、同一成本模型下重跑。
2. 新主链相对 `legacy_bof_baseline` 的改进不是靠样本显著收缩或过滤路径失真换来的假象。
3. `legacy` 回退路径仍然可用，且回退成本清楚可控。
4. `docs/observatory/sandbox-review-standard.md` 的七维审查结论可支撑切换。

若上述条件不满足，则结论为：`v0.01-plus` 继续留在当前主开发线内修正，但默认运行路径暂不切换；不得把这次链路替代工作改名塞进 `v0.02`。
