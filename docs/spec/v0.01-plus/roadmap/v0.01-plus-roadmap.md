# v0.01-plus 路线图（当前主开发线）

- **执行基线**: `docs/design-v2/01-system/system-baseline.md`（`v0.01 Frozen` 历史基线，用于对照与回退参考）
- **当前设计入口**: `docs/design-v2/03-algorithms/core-algorithms/down-to-top-integration.md`
- **状态来源**: `docs/spec/common/records/development-status.md`
- **文档状态**: `Active`（当前主开发线；用于替代 legacy top-down）
- **冲突规则**: 涉及 `v0.01` 历史口径时，以 `system-baseline.md` 为准；涉及 `v0.01-plus` 当前主开发线时，以本文件和对应实现卡为准。

## 1. 目标

`v0.01-plus` 的目标不再是“独立实验版验证”，而是作为 `v0.01` legacy top-down 的主线替代版本，完成 `down-to-top / DTT` 链路的实现、证据收口与默认运行路径切换。

本版本的核心使命：

1. 让 `Selector` 退回基础过滤与规模控制。
2. 让 `Strategy` 承担 `BOF -> IRS 排序 -> 排名输出` 的主职责。
3. 让 `Broker / Risk` 承担 `MSS` 市场级控仓位职责。
4. 让 `legacy_bof_baseline` 降级为对照组与回退路径，而不是继续充当默认主链。
5. 把当前过薄的 `IRS-lite` 升级到可玩的主线排序器，而不是继续只靠两因子硬撑。

## 2. 版本边界

1. `v0.01 Frozen` 保持不动：`BOF` 单形态、`T+1 Open`、历史正式结果契约字段冻结。
2. `v0.01-plus` 是当前主开发线，默认目标是 `DTT` 主链，而不是继续维护 legacy top-down。
3. `legacy top-down` 仅保留为 compare / rollback 模式，不再作为当前主开发线的默认交付目标。
4. 当前阶段采用“兼容优先”的迁移策略：下游 `Broker / Backtest / Report` 先保持消费兼容，`DTT` 排序与明细通过 `l3_signal_rank_exp` 承载；是否升级正式 `Signal` 契约，后续单独决策。
5. `v0.02` 不承接本次链路替代，但 `IRS-upgrade` 的第一批关键因子提前纳入 `v0.01-plus`。

## 3. 主线切换验证矩阵

| variant | Selector | Strategy | Broker / Risk | 用途 |
|---|---|---|---|---|
| `legacy_bof_baseline` | `基础过滤 + MSS/IRS 漏斗` | `BOF 单形态触发` | 固定仓位规则 | 历史对照组 / 回退参考 |
| `v0_01_dtt_bof_only` | `基础过滤后直接进 BOF` | 仅用 `bof_strength` 排序 | 固定仓位规则 | 验证纯后置排序基线 |
| `v0_01_dtt_bof_plus_irs_score` | `基础过滤后直接进 BOF` | `bof_strength + irs_score` | 固定仓位规则 | 当前优先候选主链 |
| `v0_01_dtt_bof_plus_irs_mss_score` | `基础过滤后直接进 BOF` | `bof_strength + irs_score` | `MSS` 动态控仓位 | 验证 `IRS 排序 + MSS 风控` 主线形态 |

## 4. 交付范围

1. `roadmap/v0.01-plus-spec-01-selector-strategy.md`：主线实现卡与模块边界。
2. `roadmap/v0.01-plus-spec-02-irs-upgrade.md`：把 `IRS-lite` 升级为 `5` 层可实现排序器的专项设计。
3. `governance/v0.01-plus-gate-checklist.md`：主线切换 Gate 与回退门。
4. `governance/v0.01-plus-data-contract-table.md`：兼容优先迁移期的数据契约补充。
5. `governance/v0.01-plus-run-artifact-rules.md`：`run_id / variant / artifact` 固定规则。
6. 主线证据统一落在 `docs/spec/v0.01-plus/evidence/`。
7. 主线记录统一落在 `docs/spec/v0.01-plus/records/`。

## 5. Go / No-Go

只有在以下条件同时满足时，才允许把 `v0.01-plus` 切成默认运行路径：

1. 四组对照均能在同一数据快照、同一参数快照、同一成本模型下重跑。
2. 新主链相对 `legacy_bof_baseline` 的改进不是靠样本显著收缩或过滤路径失真换来的假象。
3. `legacy` 回退路径仍然可用，且回退成本清楚可控。
4. `docs/observatory/sandbox-review-standard.md` 的七维审查结论可支撑切换。

若上述条件不满足，则结论为：`v0.01-plus` 继续留在当前主开发线内修正，但默认运行路径暂不切换；不得把这次链路替代工作改名塞进 `v0.02`。

## 6. 当前进展（2026-03-08）

已完成：

1. 固定四组矩阵已跑通：
   - `legacy_bof_baseline`
   - `v0_01_dtt_bof_only`
   - `v0_01_dtt_bof_plus_irs_score`
   - `v0_01_dtt_bof_plus_irs_mss_score`
2. 默认主线 `v0_01_dtt_bof_plus_irs_score` 的幂等验证已通过。
3. 每组 run 均已写入 `config_hash / data_snapshot / git_commit / variant`。
4. 四个焦点执行日的逐笔归因已完成，已经能解释 `IRS` 改了哪些票、哪些数量、哪些天变好、哪些天变差。
5. 长窗口敏感性已改成“分窗口 + 分场景”执行，并新增批处理脚本 `scripts/backtest/run_v001_plus_windowed_sensitivity.py`。
6. `BOF` 执行路径已做第一轮内存优化：批量预取历史、分批计算、DTT 临时表落地、最后统一排名。
7. `MSS` 已接入 `Broker Risk`，可按交易日动态调整 `max_positions / risk_per_trade_pct / max_position_pct`。
8. `candidate_top_n / preselect_score_mode` 消融已完成，已确认初选不只是工程近似，而会直接改变交易数量、收益结构与回撤。
9. `IRS-upgrade` 已从路线图占位项升级为当前主线专项，第一批关键因子确定提前纳入 `v0.01-plus`：
   - 多周期相对强度
   - 相对量能
   - 连续性
   - 扩散度
   - 牛股基因轻量层

当前结论：

1. `legacy` 与 `DTT` 已切开，`DTT` 在该短窗内的交易数明显高于 `legacy`。
2. `2026-02-10 / 02-11 / 02-13` 的 raw 与 execution 覆盖已修复，`31` 行截断不再存在。
3. 排序拆解与敏感性矩阵共同表明：`IRS` 已在多信号交易日改变 `final_rank`，并且已经进入 `Top-N / MAX_POSITIONS / BUY 数量` 约束。
4. `MSS` 当前已进入执行层，但它的价值应解释为市场级控仓位，而不是横截面排序增益。
5. 长窗口证据进一步表明：`IRS` 已具备“影响执行”的能力，但收益改善并不稳定。
6. 当前推荐结论为 `NO-GO`：`v0.01-plus` 继续作为主开发线推进，但默认运行路径暂不切到 `DTT`。
7. `EG5` 仍未完成，因为七维评审 `S0/S1=0` 尚未正式收口；当前阻塞点已经从 OOM 转成收益稳定性与回退条件解释。
8. `preselect_score` 已被证明带有交易后果：同一 `DTT` 变体下，不同 `candidate_top_n / preselect_score_mode` 会显著改变 `trade_count / EV / PF / MDD`；因此它不能再被简单视为“纯工程参数”。
9. 当前短窗最优候选集中在 `volume_ratio_only`，其中：
   - `top150 + volume_ratio_only` 的 `EV` 最好（`0.013677`）
   - `top100 + volume_ratio_only` 的 `PF` 最好（`1.6781`）
10. 更长窗口 `2025-09-01` 至 `2026-02-24` 的复核已经完成：当前主线 `v0_01_dtt_bof_plus_irs_score` 下，`top100 + amount_plus_volume_ratio` 仍然是最高 `EV` 默认组合；`volume_ratio_only` 虽然能抬高 `PF`，但会同步恶化 `EV / MDD / reject_rate`。
11. 当前默认初选参数不切换：继续保持 `CANDIDATE_TOP_N=100` 与 `PRESELECT_SCORE_MODE=amount_plus_volume_ratio`；`volume_ratio_only` 仅保留为后续专项实验候选。
12. 当前 `IRS-lite` 仍然过薄；因此 `IRS-upgrade` 不再后推为纯 `v0.02` 话题，而是提前在 `v0.01-plus` 内推进一轮可实现升级。
