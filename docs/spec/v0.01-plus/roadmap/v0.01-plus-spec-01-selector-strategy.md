## 1. 目标

在保留 `v0.01 Frozen` 历史基线的前提下，把 `v0.01-plus` 落为当前主开发线：用 `DTT` 替代 legacy top-down，并把主线执行链路统一收口为：

`Selector 初选 -> BOF 触发 -> IRS 排序 -> MSS 控仓位 -> Broker 执行`

本实现卡只回答当前主线如何落地，不再把历史漏斗和未来版本目标混写在一起。

## 2. 非目标

1. 不把本次链路替代工作改名为 `v0.02`。
2. 不在本阶段引入 `BPB / TST / PB / CPB`。
3. 不在第一阶段就强制改写正式 `l3_signals` schema；优先采用 sidecar 真相源。
4. 不删除 legacy 路径；legacy 仍保留用于 compare / rollback。

## 3. 实现边界

### 3.1 Selector

1. `legacy_bof_baseline`：沿用 `v0.01` 历史链路，即 `基础过滤 + MSS gate + IRS filter`，仅作对照组。
2. `DTT` 组：Selector 只做基础过滤、规模控制和候选准备，不再把 `MSS/IRS` 当硬过滤漏斗。
3. `preselect_score` 的职责仅是算力调度，不具有交易优先级语义。
4. `candidate_top_n` 仅用于控制 BOF 扫描规模，不允许被 `MSS/IRS` 动态改写。
5. 当前证据表明：`preselect_score_mode` 虽然不携带显式交易语义，但它会通过候选覆盖直接改变交易结果，因此必须纳入证据矩阵，而不能再当作“只影响算力、不影响收益”的隐式参数。

### 3.2 Strategy

1. 先对候选池跑 `BOF` 触发，生成最小正式 `Signal`。
2. `IRS` 是唯一进入横截面排序的后置增强因子。
3. `final_score` 当前定义为 `bof_strength + irs_score` 的排序结果，不再把 `MSS` 作为个股横截面加权项。
4. `Strategy` 负责输出排序信号，不负责最终下单数量、仓位控制和市场级风险折扣。

### 3.3 Broker / Risk

1. `Broker` 负责按排序信号执行下单截断。
2. `RiskManager` 读取 `l3_mss_daily`，动态调整：
   - `max_positions`
   - `risk_per_trade_pct`
   - `max_position_pct`
3. `MSS` 当前主线职责是市场级控仓位，不再承担前置硬停手，也不再承担横截面排序职责。

### 3.4 契约纪律

1. 第一阶段保持 `Signal / Order / Trade` 的下游兼容，避免主线替代与全量 schema 迁移同时发生。
2. `v0.01-plus` 的排序真相源以 `l3_signal_rank_exp` 为准，需能通过 `run_id + signal_id` 追溯到正式信号。
3. `strength` 继续保留为兼容字段；排序解释以 `bof_strength / irs_score / final_score` 为准。
4. 是否将扩展字段正式并入 `Signal`，作为第二阶段迁移决策单独处理。

## 5. 建议配置

```python
PIPELINE_MODE = "dtt"
DTT_VARIANT = "v0_01_dtt_bof_plus_irs_score"
DTT_SCORE_FILL = 50.0
DTT_TOP_N = 50
MSS_RISK_OVERLAY_VARIANT = "v0_01_dtt_bof_plus_irs_mss_score"
PRESELECT_SCORE_MODE = "amount_plus_volume_ratio"
```

约束：

1. `v0.01-plus` 当前主线默认进入 `DTT` 模式；legacy 仅用于 compare / rollback。
2. 任何切换前都必须保留 `legacy_bof_baseline` 可重跑入口。
3. 缺失 `IRS / MSS` 分数时，只允许 `FILL=50.0`，不得静默跳过或向前看。
4. `v0_01_dtt_bof_plus_irs_mss_score` 的含义已收口为：`IRS 排序 + MSS 控仓位`。
5. `PRESELECT_SCORE_MODE` 的长窗口复核已经完成；当前默认继续保持 `amount_plus_volume_ratio`，不切到 `volume_ratio_only`。
6. 原因是：`volume_ratio_only` 在更长窗口里虽能提高部分场景的 `PF`，但会同步恶化 `EV / MDD / reject_rate`，不满足主线默认参数的稳定性要求。
