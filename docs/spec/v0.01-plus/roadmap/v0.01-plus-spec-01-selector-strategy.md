# v0.01-plus Spec-01：Selector / Strategy 主线替代实现卡

- **版本**: `v0.01-plus Spec-01`
- **状态**: `Active`
- **日期**: `2026-03-07`
- **基线**: `docs/design-v2/01-system/system-baseline.md`, `docs/design-v2/02-modules/selector-design.md`, `docs/design-v2/03-algorithms/core-algorithms/down-to-top-integration.md`, `docs/spec/v0.01-plus/governance/v0.01-plus-run-artifact-rules.md`
- **治理状态**: `docs/spec/common/records/development-status.md`

## 1. 目标

在保留 `v0.01 Frozen` 历史基线的前提下，把 `v0.01-plus` 落为当前主开发线：用 `DTT` 替代 legacy top-down，完成 `Selector / Strategy` 的职责切换、对照验证与默认运行路径准备。

## 2. 非目标

1. 不把本次链路替代工作改名为 `v0.02`。
2. 不在本阶段引入 `BPB / TST / PB / CPB`。
3. 不在第一阶段就强制改写全部下游契约；优先采用兼容迁移，降低切换面。
4. 不删除 legacy 路径；legacy 仍保留用于 compare / rollback。

## 3. 实现边界

### 3.1 Selector

1. `legacy_bof_baseline`：沿用 `v0.01` 历史链路，即 `基础过滤 + MSS gate + IRS filter`，仅作对照组。
2. `DTT` 组：Selector 只做基础过滤、规模控制和候选准备，不再把 `MSS/IRS` 当硬过滤漏斗。
3. 当前主线目标是让 `DTT` 组成为默认实现路径；legacy 不再作为主交付目标。

### 3.2 Strategy

1. 先对候选池跑 `BOF` 触发，生成最小正式 `Signal`。
2. 评分环节放在 `BOF` 之后，根据 variant 叠加 `bof_strength / irs_score / mss_score`。
3. 当前迁移期采用兼容优先策略：排序结果与分数明细写入 `l3_signal_rank_exp`，下游先维持 `v0.01` 兼容消费方式。

### 3.3 契约纪律

1. 第一阶段保持 `Signal / Order / Trade` 的下游兼容，避免主线替代与全量 schema 迁移同时发生。
2. `v0.01-plus` 的排序真相源以 `l3_signal_rank_exp` 为准，需能通过 `run_id + signal_id` 追溯到正式信号。
3. 是否将 `bof_strength / irs_score / mss_score / final_score` 正式并入 `Signal`，作为第二阶段迁移决策单独处理。

## 4. 建议交付物

1. `src/selector/selector.py`：补齐 `legacy` 与 `dtt` 两类入口，并明确当前主线路径。
2. `src/strategy/strategy.py`：补齐 variant 路由与主线默认逻辑。
3. `src/strategy/ranker.py`：主线评分器。
4. `src/config.py`：新增主线切换与 variant 配置。
5. `scripts/backtest/run_v001_plus_dtt_matrix.py`：统一跑对照矩阵。
6. `tests/unit/selector/`、`tests/unit/strategy/`、`tests/integration/backtest/`：补主线路径回归。

## 5. 建议配置

```python
ENABLE_DTT_MODE = True
DTT_VARIANT = "v0_01_dtt_bof_plus_irs_score"
DTT_SCORE_FILL = 50.0
DTT_TOP_N = 50
```

约束：

1. `v0.01-plus` 当前主线默认进入 `DTT` 模式；legacy 仅用于 compare / rollback。
2. 任何切换前都必须保留 `legacy_bof_baseline` 可重跑入口。
3. 缺失 `MSS / IRS` 分数时，只允许 `FILL=50.0`，不得静默跳过或向前看。
