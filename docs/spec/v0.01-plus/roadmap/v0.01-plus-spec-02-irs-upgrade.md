# v0.01-plus Spec-02：IRS 升级设计

- **版本**: `v0.01-plus Spec-02`
- **状态**: `Active`
- **日期**: `2026-03-08`
- **基线**: `blueprint/01-full-design/05-irs-lite-contract-supplement-20260308.md`, `blueprint/02-implementation-spec/01-current-mainline-implementation-spec-20260308.md`
- **治理状态**: `docs/spec/common/records/development-status.md`

## 1. 定位

这份文档不是“以后再说”的方向草案，而是 `v0.01-plus` 当前主开发线里对 `IRS-upgrade` 的可实现规格。

当前决策已经明确：

1. `IRS-lite` 太薄，排序层信息量不够。
2. `IRS` 的下一批关键因子不再全部后推到 `v0.02`。
3. 至少一轮可实现的 `IRS-upgrade` 要提前纳入 `v0.01-plus`，与当前 `DTT` 主线一起完成验证。
4. 当前版本完成时，`IRS` 不得仍然只是“两因子行业日评分器”，而必须达到最小可交易的行业轮动排序强度。

当前主线边界保持不变：

`Selector 初选 -> BOF 触发 -> IRS 排序 -> MSS 控仓位 -> Broker 执行`

---

## 2. 当前问题

当前 `IRS-lite` 只有两层：

1. `RS`：行业相对涨跌幅
2. `CF`：简化资金流向

它存在三个现实问题：

1. 只能回答“行业今天强不强”，不能回答“行业正在进入哪个轮动阶段”。
2. 只能看到行业表面强弱，不能看到行业内部是否已长出可交易强股结构。
3. 成交量口径过粗，缺少“相对自身均值 / 相对全市场活跃度 / 相对行业内部强股”的相对化表达。

因此当前 `IRS-lite` 只够做最小排序增强，不够支撑你要的“行业轮动 + 牛股基因”排序层。

---

## 3. 当前决策

### 3.1 提前进入 v0.01-plus 的内容

以下能力提前进入 `v0.01-plus`：

1. **多周期相对强度层**
2. **相对量能层**
3. **轮动状态层**
4. **行业内部扩散层**
5. **轻量牛股基因层**

### 3.2 暂不进入 v0.01-plus 的内容

以下能力继续留在 `v0.02+`：

1. 政策 / 事件语义层
2. 行业生命周期完整状态机
3. 跨主题映射与风格漂移校正
4. 更复杂的自适应权重学习

也就是说，`v0.01-plus` 不再满足于 `IRS-lite`，但也不追求一步到位做成“最终完整版”。

---

## 4. 目标算法结构

### 4.1 目标分层

`IRS-upgrade` 在 `v0.01-plus` 内收成五层：

1. **RS：相对强度层**
2. **RV：相对量能层**
3. **RT：轮动状态层**
4. **BD：扩散度层**
5. **GN：牛股基因轻量层**

### 4.2 目标公式

建议起步版：

```text
irs_total_score =
    0.30 * rs_score
  + 0.25 * rv_score
  + 0.15 * rt_score
  + 0.15 * bd_score
  + 0.15 * gn_score
```

说明：

1. `RS + RV` 仍是主体。
2. `RT / BD / GN` 先作为增强层，不抢主体权重。
3. 这是 `v0.01-plus` 的起步权重，不是最终固定权重。

---

## 5. 五层因子定义

### 5.1 RS：多周期相对强度层

目标：

回答“这个行业当前相对市场到底强不强，而且是短强还是中强”。

建议子因子：

1. `rs_1d = industry_pct_chg - benchmark_pct_chg`
2. `rs_5d = industry_return_5d - benchmark_return_5d`
3. `rs_20d = industry_return_20d - benchmark_return_20d`
4. `rank_stability`：行业近 5 日名次稳定性

建议聚合：

```text
rs_score = 0.35 * rs_1d + 0.35 * rs_5d + 0.20 * rs_20d + 0.10 * rank_stability
```

### 5.2 RV：相对量能层

目标：

把当前过粗的 `CF` 升级成真正可解释的“量能相对化”。

建议子因子：

1. `amount_vs_self_20d = industry_amount / industry_amount_ma20`
2. `volume_vs_self_20d = industry_volume / industry_volume_ma20`
3. `activity_vs_market = industry_turnover_proxy / market_turnover_proxy`
4. `strong_stock_amount_share = industry_strong_stock_amount / industry_amount`

建议聚合：

```text
rv_score = 0.30 * amount_vs_self_20d
         + 0.30 * volume_vs_self_20d
         + 0.20 * activity_vs_market
         + 0.20 * strong_stock_amount_share
```

### 5.3 RT：轮动状态层

目标：

回答“行业当前处于哪种轮动状态，是刚启动、正在延续、走向衰竭，还是已经回落”。

建议子因子：

1. `top_rank_streak_5d`：近 5 日进入前列的持续天数
2. `score_slope_5d`：近 5 日行业得分斜率
3. `momentum_consistency`：近 5 日强度方向一致性
4. `rotation_status`：`START / CONTINUE / EXHAUST / FALLBACK`

说明：

- `rotation_status` 是标签字段
- 真正进入总分的是数值化连续性特征

### 5.4 BD：扩散度层

目标：

回答“行业强是不是只靠一两只龙头硬顶，还是已经形成板块共振”。

建议子因子：

1. `up_ratio`：行业内上涨家数占比
2. `strong_up_ratio`：行业内强上涨家数占比
3. `new_high_ratio`：行业内创新高家数占比
4. `bof_hit_density_5d`：近 5 日 BOF 命中密度

### 5.5 GN：牛股基因轻量层

目标：

把你说的“行业牛股基因”先做出第一版，不再只停留在复盘概念。

建议子因子：

1. `leader_strength`：行业头部强股强度中位数
2. `leader_count`：行业内强股数量
3. `leader_follow_through`：强股次日延续率
4. `strong_stock_ratio`：强股占行业股票数比例

当前约束：

- `GN` 在 `v0.01-plus` 先做轻量版
- 只用当前主线已经可得到的行业内个股结构派生，不引入事件叙事层

---

## 5A. 当前版本最低完成标准

`v0.01-plus` 内的 `IRS-upgrade`，最低必须做到：

1. `RS` 从单日相对强度升级到多周期相对强度。
2. `RV` 不再只看绝对成交额占比，而必须引入相对行业均值的量能口径。
3. `RT` 必须产出可回放的 `rotation_status`。
4. `GN` 必须至少给出一层轻量牛股基因信号，而不是继续完全空缺。

若以上四点有任一缺失，则 `IRS` 仍然只是 `IRS-lite`，当前版本不算达到最小可交易强度。

## 8. 与主线边界的硬约束

1. 不允许把 `IRS` 拉回 `Selector` 前置过滤。
2. 不允许重新出现“行业 Top-N 才能进 BOF”的硬门控。
3. `IRS` 只负责排序增强，不负责执行仓位。
4. `MSS` 仍只负责市场级控仓位，不得重新混入 `irs_total_score`。

## 10. 当前结论

当前结论已经明确：

1. `IRS-lite` 不够，必须升级。
2. 这次升级不再全部后推到 `v0.02`。
3. 第一批真正有信息量的 `IRS` 因子，应该提前纳入 `v0.01-plus`。
4. 最值得先做的不是“更多故事”，而是：
   - 多周期相对强度
   - 相对量能
   - 连续性
   - 扩散度
   - 牛股基因轻量层

这五层足够把当前 `IRS` 从“太监版”推进到“可玩的主线排序器”。
