## 1. 目标

本文只做一件事：

`把四个战场当前准备进入统一默认系统候选的对象，冻结成 promote-now / shadow-only / remain-research 三层。`

## 2. 当前统一默认系统候选的核心口径

当前 Phase 6 要验证的统一默认系统候选，不是“把所有研究结论一次性打开”，而是：

`BOF baseline + control hierarchy + historical context sidecar + MSS/IRS retirement boundary`

换句话说：

1. entry 仍以 `BOF baseline` 为核心
2. capital control 仍以 `FIXED_NOTIONAL_CONTROL + FULL_EXIT_CONTROL` 为核心
3. 第四战场当前优先以 `context sidecar / report / dashboard` 身份进入候选系统
4. 旧 `IRS-lite / MSS-lite` 继续保持退役状态，不回到默认运行层

## 4. 当前明确排除在 Phase 6 初始 promotion package 外的内容

1. 任何旧 `IRS / MSS` 默认运行回归
2. 任何 `Normandy` watch / retained branch 的自动升格
3. 任何 `Positioning` retained partial-exit family 的默认 promotion
4. 任何 `Gene` score / mirror / conditioning 对 runtime 的硬 gate
5. 任何口头把 report-side context 说成 runtime filter 的表述

## 5. 对 Phase 6B 的直接要求

`Phase 6B` 只允许验证下面这件事：

`当前冻结后的 promotion candidate，是否能在不破坏 baseline runtime stability 的前提下，形成更清楚、更统一、更可治理的默认系统。`

这意味着验证必须同时检查：

1. 端到端运行稳定性
2. trace / report / sidecar 完整性
3. baseline 对照一致性
4. shadow-only 对象没有偷变成 runtime hard gate

## 6. 一句话结论

当前 freeze 的不是“新赢家参数”，而是“统一默认系统候选的合法组成边界”：`BOF baseline + control hierarchy + context sidecar + retired old IRS/MSS`。
