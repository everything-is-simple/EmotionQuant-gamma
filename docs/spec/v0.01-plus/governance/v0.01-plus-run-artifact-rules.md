# v0.01-plus 运行命名与产物规则

- **版本**: `v0.01-plus 运行规则`
- **状态**: `Active`
- **日期**: `2026-03-15`
- **关联文档**: `docs/spec/v0.01-plus/roadmap/v0.01-plus-roadmap.md`, `docs/spec/v0.01-plus/roadmap/v0.01-plus-spec-01-selector-strategy.md`, `docs/spec/v0.01-plus/governance/v0.01-plus-data-contract-table.md`, `docs/spec/v0.01-plus/governance/v0.01-plus-mainline-no-fake-rules.md`

## 1. 固定规则

1. 当前主线治理线是 `v0.01-plus`，但默认运行路径仍是 `legacy_bof_baseline`。
2. `PIPELINE_MODE=dtt` 与 `DTT_VARIANT=*` 只用于 `v0.01-plus` compare / migration run，不得被口头升格成“当前默认运行口径”。
3. 主线 compare / migration matrix 固定为 4 组，不允许运行时临时发明新 variant：
   - `legacy_bof_baseline`
   - `v0_01_dtt_bof_only`
   - `v0_01_dtt_bof_plus_irs_score`
   - `v0_01_dtt_bof_plus_irs_mss_score`
4. `legacy_bof_baseline` 当前同时承担 `default runtime + compare + rollback target` 三重身份，直到新的 formal Gate record 改写它。

## 1.1 No-fake 运行口径

当前运行命名和 artifact 规则必须同时遵守：

1. 有 `dtt` 前缀的 run，不等于主线默认运行路径已经切换。
2. 研究线 runner 的 evidence，不等于主线默认参数已经 promotion。
3. `TRAIL_SCALE_OUT_25_75`、`CD10`、`single_lot floor`、`R4 + R5 + R6 + R7 + R10` 这些对象若出现在 artifact 中，都必须保留其治理身份，不得在命名或摘要中冒充 canonical default。

## 2. run_id 规则

### 2.1 命名模板

- 日常运行：`daily_<mode>_<variant>_dYYYYMMDD_tHHMMSS`
- 烟雾验证：`smoke_<mode>_<variant>_dYYYYMMDD_tHHMMSS`
- 回测窗口：`backtest_<mode>_<variant>_wYYYYMMDD_YYYYMMDD_tHHMMSS`
- 矩阵运行：`matrix_<mode>_<variant>_wYYYYMMDD_YYYYMMDD_tHHMMSS`

### 2.2 含义

1. `<mode>` 只允许 `legacy` 或 `dtt`。
2. `<variant>` 必须是固定矩阵中的合法值。
3. `dYYYYMMDD` 表示单日信号日；`wYYYYMMDD_YYYYMMDD` 表示回测窗口。
4. `tHHMMSS` 是 UTC 时间戳尾段，用于避免同日重复 run 冲突。

## 3. artifact 命名规则

### 3.1 模板

`<run_id>__<artifact_kind>.<ext>`

示例：

- `smoke_dtt_v0_01_dtt_bof_plus_irs_score_d20260307_t140501__selector_strategy_smoke.json`
- `matrix_dtt_v0_01_dtt_bof_plus_irs_score_w20240101_20241231_t140501__dtt_matrix.json`

### 3.2 artifact_kind 固定词

- `selector_strategy_smoke`
- `dtt_matrix`
- `distribution_audit`
- `daily_run_summary`

## 4. 落点规则

1. **正式文档证据**：落在 `docs/spec/v0.01-plus/evidence/`
2. **工作副本 DuckDB / pytest / ruff / mypy / 运行时缓存**：落在 `TEMP_PATH`
3. **正式执行库**：落在 `DATA_PATH`
4. `G:\EmotionQuant-gamma` 只允许保留代码与文档，不再作为临时产物默认落点

## 5. `_meta_runs` 最低要求

每次运行必须写入：

- `run_id`
- `mode`
- `variant`
- `config_hash`
- `data_snapshot`
- `git_commit`
- `artifact_root`

## 6. sidecar 追溯规则

1. `l3_signal_rank_exp` 以 `(run_id, signal_id)` 为幂等键。
2. 同一次运行内，排序真相以 `l3_signal_rank_exp` 为准。
3. `l3_signals` 只保留当前正式输出信号；未入选 `Top-N` 的 BOF 命中只保留在 sidecar。
