## 2.2 EG3 当前状态

1. `2026-02-10 / 02-11 / 02-13` 的 raw/execution 覆盖异常已修复，`31` 行截断不再存在。
2. `IRS` 已证明会进入 `Top-N / MAX_POSITIONS / BUY 数量` 约束，排序差异不是“只停留在 final_rank 上”。
3. `BOF` 经过“批量预取历史 + 分批计算 + 临时表落地”优化后，更长窗口五个敏感性场景已全部在 `4GB` 会话内存限制下跑通。
4. `EG3` 仍暂不勾选，因为收益改善没有稳定成立，且 fallback/失败模式说明仍未彻底收口。

## 2.3 EG5 / EG6 当前判断

### `EG5`

暂不勾选。

原因：

1. 七维评审 `S0/S1=0` 尚未正式完成。
2. 更长窗口全场景虽然已经跑通，但收益来源、失败模式与回退条件还没有完成正式七维收口。

### `EG6`

当前结论：`NO-GO`

原因：

1. `IRS` 已经进入执行层，但当前证据只支持“会改成交身份/数量”，还不支持“稳定改善收益”。
2. `top1_pos2` 的长窗口结果是 `EV/MDD` 改善但 `PF` 明显变差，收益结构不稳。
3. `top50_pos10` 的长窗口结果几乎不改聚合收益，只带来很小的回撤改善。
4. 尽管更长窗口五场景已经能在个人 PC 上跑通，但当前证据链仍不足以把 `DTT` 默认切成运行主路径。

## 3. 判定规则

1. 任一前置 Gate 未通过：`NO-GO`，只允许修 Gate，不允许直接推进主线切换。
2. 任一切换后 Gate 未通过：默认运行路径暂不切到 `DTT`，但修正工作仍留在 `v0.01-plus` 主线内完成。
3. 只有 `PG1-PG7` 与 `EG1-EG6` 同时通过，才允许把 `v0.01-plus` 作为实际默认运行路径。
