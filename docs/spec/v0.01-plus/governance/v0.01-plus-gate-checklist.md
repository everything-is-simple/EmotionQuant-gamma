# v0.01-plus Gate Checklist（主线切换）

- **版本**: `v0.01-plus Gate`
- **状态**: `Active`
- **日期**: `2026-03-07`
- **关联文档**: `docs/spec/v0.01-plus/roadmap/v0.01-plus-roadmap.md`, `docs/spec/v0.01-plus/roadmap/v0.01-plus-spec-01-selector-strategy.md`, `docs/spec/v0.01-plus/governance/v0.01-plus-data-contract-table.md`, `docs/spec/v0.01-plus/governance/v0.01-plus-run-artifact-rules.md`

## 1. 开工前 Gate

- [ ] `PG1` 版本边界已切开：`v0.01` 保持 Frozen，`v0.01-plus` 升为当前主开发线。
- [ ] `PG2` 主线切换语义已写死：`DTT` 是当前默认目标，legacy 仅作 compare / rollback。
- [ ] `PG3` 最小切换矩阵固定为 4 组，不得边做边加 variant。
- [ ] `PG4` 兼容优先迁移策略已确定：下游契约暂保持兼容，排序明细统一写入 `l3_signal_rank_exp`。
- [ ] `PG5` run 命名、artifact 落点、evidence 路径与脚本入口已固定，确保可重跑、可回放、可归档。
- [ ] `PG6` `check_docs.ps1` 与 `preflight.ps1 -Profile full` 已通过，避免在脏治理基线上开工。
- [ ] `PG7` legacy 回退路径仍可运行，且回退条件写明。

## 2. 切换后 Gate

- [ ] `EG1` 四组对照均完成，并保留同口径输出。
- [ ] `EG2` 每组结果均记录 `trade_count / win_rate / profit_factor / expected_value / max_drawdown / 环境分桶中位数路径`。
- [ ] `EG3` 新主链的改善不是依赖样本异常收缩、过滤失真或不可解释的 fallback 堆叠。
- [ ] `EG4` 幂等重跑通过：同一数据快照、同一参数快照下结果一致。
- [ ] `EG5` 七维评审 `S0/S1=0`，且能解释新主链的收益来源、失败模式与回退条件。
- [ ] `EG6` 给出明确结论：`GO`（继续作为当前主开发线推进代码落地）或 `NO-GO`（保留 legacy 为默认路径并在 `v0.01-plus` 线内修正）。

## 3. 判定规则

1. 任一前置 Gate 未通过：`NO-GO`，只允许修 Gate，不允许直接推进主线切换。
2. 任一切换后 Gate 未通过：默认运行路径暂不切到 `DTT`，但修正工作仍留在 `v0.01-plus` 主线内完成。
3. 只有 `PG1-PG7` 与 `EG1-EG6` 同时通过，才允许把 `v0.01-plus` 作为实际默认运行路径。
