## 3. 聚合结论

### 3.1 Slot scarcity 的结果不是“延迟入场”，而是“直接丢机会”

`slot_path_summary` 给出了当前最关键的结论：

1. `slot_reject_count = 7`
2. `baseline_only_trade_count = 7`
3. `delayed_entry_count = 0`
4. `permanent_miss_count = 7`
5. `baseline_only_entry_pnl_total = +3087.2014`

约束性结论：

1. `max_positions` 删掉的这 `7` 笔交易，一笔都没有在后续窗口里以“延迟入场”补回来
2. 这不是“先错过，后面再接回”的路径
3. 当前 BOF 默认路径下，`slot scarcity` 的直接后果就是：
   - `trade_count` 变少
   - 而且被删掉的交易集合整体还是正收益

### 3.2 当前焦点不在“同日新赢家抢走槽位”，而在“已有持仓把槽位占满”

五个 slot focus dates 分别是：

1. `2026-01-30`
2. `2026-02-02`
3. `2026-02-06`
4. `2026-02-09`
5. `2026-02-24`

共同事实：

1. `only_max_positions` 在这五天的同日 `accepted` 新买单都是 `0`
2. 被拒的票全部是 `MAX_POSITIONS_REACHED`
3. 因此当前问题不是“同日更高 rank 的新票把它们挤掉”
4. 而是“开盘前就已经没有可用 slot 了”

这把主因进一步从“同日 signal competition”收口到了：

`position carryover + shrinking max_positions`

### 3.3 动态缩容和持仓 carryover 在两天上已经出现“过载状态”

最关键的两天是：

1. `2026-02-06`
2. `2026-02-09`

这两天都出现：

1. `risk_regime = RISK_OFF`
2. `effective_max_positions = 4`
3. `holdings_before = 5`

也就是：

`已有持仓数 > 当天有效 max_positions`

约束性结论：

1. 当前系统允许把更早形成的持仓带入后续更保守的 `RISK_OFF` 日
2. 一旦 `effective_max_positions` 下调，系统不会先消化到新上限以下
3. 结果就是：当天还没接任何新票，就已经处在“超载”状态
4. 这会让 rank=1 的新信号直接被挡掉

## 4. 逐日归因

当日状态：

1. `only_max_positions` 开盘前持仓：`002594 / 300502 / 600010 / 600340`
2. `holdings_before = 4`
3. `effective_max_positions = 4`

被拒信号：

1. `600436`，`rank=1`
2. `601668`，`rank=2`
3. `300755`，`rank=3`

左侧原本成交且后续表现：

结论：

`这是典型的“满槽即拒”。当天没有任何新接受单，三笔机会直接永久丢失。`

### 4.2 2026-02-02：1 笔 rank=1 机会继续被满槽挡掉

当日状态：

1. `only_max_positions` 开盘前持仓仍为 `4`
2. `effective_max_positions = 4`

   - `300755`
   - `600436`
   - `601668`

1. `300014`，`rank=1`

`它证明 slot scarcity 不一定总删掉坏交易，但关键点仍然是：它不是“延迟”，而是“直接删除”。`

### 4.3 2026-02-06：第一次出现 over-cap carryover

`这是当前最重要的路径证据：系统因为前序持仓 carryover，在 RISK_OFF 日进入 over-cap，连 rank=1 的正收益机会都直接挡掉。`

### 4.4 2026-02-09：第二次 over-cap carryover，继续删掉正收益机会

`over-cap carryover 不是偶发点，而是在当前窗口里至少重复出现了两次。`

### 4.5 2026-02-24：仍然是满槽即拒

## 6. 当前根因链条

到这里，当前 BOF 默认路径下的主因链条可以收口为：

1. `RISK_OFF` 将 `effective_max_positions` 压低到 `4`
2. 系统允许更早形成的持仓带入新的更低上限
3. 在 `2026-02-06 / 2026-02-09` 已经出现 `holdings_before > effective_max_positions`
4. 因此新信号在进入同日竞争前就已经没有 slot
5. 被挡掉的 `7` 笔交易没有任何一笔在后续窗口里延迟补回
6. 而这 `7` 笔交易合计原本还是正收益 `+3087.2014`
7. sizing 只能在剩余成交里减一点美元损失，不能修复被删掉的机会集合
