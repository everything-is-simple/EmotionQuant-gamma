## 2. 两个必答问题

### 2.1 行业排名是否真的变化

结论：变化明显，而且是全窗口级别的变化。

1. `IRS_RSRV` 相对 `IRS_LITE`：
   - `industry_rank_diff_days_vs_lite = 30`
   - top 行业变化记录数 = `272`
2. `IRS_RSRVRTBDGN` 相对 `IRS_LITE`：
   - `industry_rank_diff_days_vs_lite = 30`
   - top 行业变化记录数 = `272`
3. 这说明 `RV / RT / BD / GN` 不是“写了列但没生效”，而是真的改写了行业横截面排序。
4. 代表性变化样本包括：
   - `2026-01-05 / 国防军工 / rank=1`
   - `2026-01-05 / 有色金属 / rank=2`
   - `2026-01-06 / 非银金融 / rank=2`
   - `2026-01-07 / 煤炭 / rank=1`

### 2.2 执行结果是否因此变化

结论：这轮窗口里，执行层没有被改动。

1. 三个场景的：
   - `trade_count`
   - `expected_value`
   - `profit_factor`
   - `exposure_rate`
   - `participation_rate`
   基本一致。
2. `IRS_RSRV` 与 `IRS_RSRVRTBDGN` 都出现了新增选中 signal：
   - `IRS_RSRV`: `4`
   - `IRS_RSRVRTBDGN`: `5`
3. 但 `execution_diff_days_vs_lite = 0`，说明这些变化还没有穿透到最终买入执行层。
4. 当前这段窗口的实际含义是：
   - `IRS` 排序层已经真实变化
   - 但在 `BOF-only + 当前仓位容量 + 当前窗口信号密度` 下，排序变化还不足以改写成交集合

## 3. 这轮 evidence 说明了什么

### 3.1 可以确认的结论

1. `Phase 2 / IRS` 已不再停留在 `IRS-lite`。
2. `RS / RV / RT / BD / GN` 五层输入已经进入正式排序链，而不是只写 trace。
3. 新 `IRS` 层目前没有破坏执行稳定性；至少在这轮短窗口里，没有带来额外执行回归。
4. 这份 evidence 可以支持一句保守结论：
   - `Phase 2` 已达到“最小可交易排序层已接通”的状态。

### 3.2 还不能写死的结论

1. 现在还不能说 `IRS_RSRVRTBDGN` 已经证明优于 `IRS_LITE`。
2. 当前窗口里，排序变化没有转化成执行变化，所以还不能给出 alpha 增益结论。
3. 这也说明 `Phase 2` 还差最后一类工作：
   - 参数治理
   - 更长窗口 / 更高信号密度下的二轮 evidence
