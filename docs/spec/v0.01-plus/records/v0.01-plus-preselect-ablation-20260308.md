## 1. 消融矩阵

## 2. 关键结论

### 2.1 初选不是“纯工程参数”

同一 `DTT` 变体下，不同 `candidate_top_n / preselect_score_mode` 会直接改变：

1. `trade_count`
2. `expected_value`
3. `profit_factor`
4. `max_drawdown`
5. `skip_maxpos_count / participation_rate`

因此，`preselect_score` 虽然不应该携带显式交易语义，但它已经不是“只影响算力、不影响交易结果”的纯工程旋钮。

### 2.2 短窗内最强组合集中在 `volume_ratio_only`

两条主线在相同初选组合下结果几乎一致，说明这轮差异主要来自初选本身，而不是 `IRS`。

| 组合 | 结论 | 指标 |
|---|---|---|
| `top150 + volume_ratio_only` | 短窗 `EV` 最佳 | `EV=0.013677`, `PF=1.0951`, `MDD=0.007185`（`BOF+IRS`） |
| `top100 + volume_ratio_only` | 短窗 `PF` 最佳 | `EV=0.009082`, `PF=1.6781`, `MDD=0.010310`（`BOF+IRS`） |
| `top50 + amount_only` | 样本很少，结果偏偶然 | `trade_count=4`, `PF=1.1505` |
| `top100 + amount_plus_volume_ratio` | 当前默认，但短窗表现一般 | `EV=-0.003532`, `PF=0.4968`, `MDD=0.017567`（`BOF+IRS`） |
| `top150 + amount_plus_volume_ratio` | 当前矩阵中最差组之一 | `EV=-0.008089`, `PF=0.5475`, `MDD=0.023088`（`BOF+IRS`） |

### 2.3 `candidate_top_n` 提大，不一定自动更好

1. `top150` 会带来更多机会，但也明显增加 `skip_maxpos_count`
2. `top150 + amount_plus_volume_ratio` 在这轮短窗里反而恶化
3. `top150 + volume_ratio_only` 虽然最优，但当前仍不足以直接证明“大池子 + 量比优先”就是长期最优

## 3. 对当前主线的解释

### 3.1 为什么 `volume_ratio_only` 会更强

当前短窗里，`BOF` 更像偏事件/活跃度驱动的触发器。

在这种条件下：

1. `amount_only` 倾向大票和高成交额稳态样本
2. `amount_plus_volume_ratio` 在短窗内仍然被成交额成分明显牵引
3. `volume_ratio_only` 更容易把“当下活跃、量能异动”的标的排进 BOF 扫描范围

这能解释为什么它在短窗里同时改善了 `EV` 和 `PF`。

### 3.2 为什么现在不直接翻默认

原因不是怀疑结果，而是证据还不够长：

1. 当前消融只覆盖一个短窗
2. `volume_ratio_only` 可能对事件密集窗口天然更友好
3. 一旦换成长窗口、换环境切片，它也可能带来更高噪音与更差回撤

所以现在最稳的动作不是立刻翻默认，而是：

1. 把 `volume_ratio_only` 记为候选默认值
2. 继续跑更长窗口复核
3. 再决定是否从 `amount_plus_volume_ratio` 切到新默认

## 4. 当前建议

### 4.1 可以立即写死的结论

1. `Selector` 的初选参数必须纳入证据矩阵
2. `preselect_score` 不能再被视为“纯工程近似”
3. `candidate_top_n` 和 `preselect_score_mode` 都会改变交易结果

### 4.2 暂时不写死的结论

1. 暂不把 `volume_ratio_only` 直接写成唯一默认值
2. 暂不把 `candidate_top_n=150` 直接写成唯一默认值

### 4.3 当前推荐动作

1. 继续保留当前默认：
   - `CANDIDATE_TOP_N=100`
   - `PRESELECT_SCORE_MODE=amount_plus_volume_ratio`
2. 把下面两组列为下一轮长窗口优先候选：
   - `top100 + volume_ratio_only`
   - `top150 + volume_ratio_only`
3. 在更长窗口复核前，不对主线默认参数做翻转式调整

## 5. 当前结论

一句话结论：

`preselect` 已经被证明会影响交易结果，但当前最优解仍然只在短窗成立；它值得继续追，但还不值得立刻写死为新默认。`

### 6.2 长窗口裁决

1. 短窗里表现最强的 `volume_ratio_only`，在更长窗口上**不能支持默认切换**。
2. 对当前主线 `v0_01_dtt_bof_plus_irs_score` 来说：
   - `top100 + amount_plus_volume_ratio` 的 `EV` 最好
   - `top150 + amount_plus_volume_ratio` 的 `MDD` 最好
   - `volume_ratio_only` 虽然抬高了 `PF`，但同步恶化了 `EV / MDD / reject_rate`
3. `top100 + volume_ratio_only` 在两条 DTT 线里都把 `EV` 直接打成负值，这已经足够否决它作为新的默认初选。
4. `top150 + volume_ratio_only` 在 `BOF-only` 上仍有吸引力，但一旦接回当前主线 `BOF+IRS`，它只保住了略高的 `PF`，却带来更差的 `MDD` 和更高的拒单率。

### 6.3 默认值决策

当前决定：

1. **不把默认初选从 `amount_plus_volume_ratio` 切到 `volume_ratio_only`**
2. 当前默认继续保持：
   - `CANDIDATE_TOP_N=100`
   - `PRESELECT_SCORE_MODE=amount_plus_volume_ratio`

理由：

1. 这是当前主线 `v0_01_dtt_bof_plus_irs_score` 的长窗口最高 `EV` 组合
2. 它同时保有最低的拒单率与最高的参与率
3. `volume_ratio_only` 的优势主要体现在 `PF`，但代价是更差的 `EV / MDD / reject_rate`
4. 这不符合当前主线要优先追求“稳定可解释、可回退”的默认配置标准

### 6.4 仍值得保留的候选

虽然默认不切，但下列组合仍值得保留为后续专项实验：

1. `top150 + volume_ratio_only`
   - 适合研究“更激进的活跃度导向初选”是否适用于特定市场环境
2. `top150 + amount_plus_volume_ratio`
   - 适合研究“大候选池 + 保守粗排”是否能在不明显伤害回撤的前提下提升覆盖
