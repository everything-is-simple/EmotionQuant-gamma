## 4. 建议开工顺序

### 第一优先级

1. `IRS-A1`
2. `IRS-A3`
3. `MSS-A1`

原因：

- 这三项能最快把 `IRS-lite / MSS-lite` 从文档升级到第一轮代码落地

### 第二优先级

1. `IRS-A2`
2. `MSS-A2`
3. `MSS-A3`

原因：

- 这是从“有字段”走向“有结构语义”的关键层

### 第三优先级

1. `IRS-A5`
2. `MSS-A4`

原因：

- 没证据，升级后的字段还是停留在设计层

---

## 5. 当前结论

当前最值得先进入代码层的，不是再写更多设计文档，而是：

1. 先把 `IRS` 的多周期强度和相对量能落地
2. 先把 `MSS` 的 `phase / phase_trend / phase_days` 落地
3. 再让证据脚本把这些升级项真正拉进回测与敏感性矩阵

这样 `v0.01-plus` 才会从“口径已经正确”进入“实现开始跟上”。
