## 4. Re-entry 裁决

### 4.1 为什么允许重新进入 Gate replay

本轮给出 `re-entry eligible` 的理由是：

1. 相对当前 `hard_cap`，候选已稳定改善：
   - `trade_count`
   - `expected_value`
   - `profit_factor`
   - `skip_maxpos_count`
2. 改善来源已可解释：
   - `trade attribution` 能逐日回连到具体票和具体 slot swap
3. 候选已通过幂等
4. rollback path 仍完整保留：
   - `legacy_bof_baseline`
   - `current_full_overlay_hard_cap`

因此当前正式判断是：

`carryover_buffer(1)` 可以被带回 Phase 4，作为单一整改候选重做 Gate replay。`

### 4.2 为什么现在还不是 GO

当前仍然不能宣布 `GO`，因为：

1. 相对当前 `hard_cap`，它的 `max_drawdown` 仍更差：
   - `0.007427 > 0.006035`
2. 相对 `legacy_bof_baseline`，它仍未追平：
   - `EV`
   - `PF`
   - `MDD`
3. 更长窗口 `top50_pos10` 里，候选虽然改善 `EV / PF / trade_count`，但 `MDD` 继续走坏

约束性结论：

`P4.1-D` 的正确结果是“允许回到 Gate 再审”，不是“整改已经完成、系统可以升级默认”。`
