## 3. 候选定义

### 3.1 当前 rollback 语义保持为 `hard_cap`

当前正式 `NO-GO` 路径不改写：

1. `risk_regime` 下调后，`effective_max_positions = floor(base_max_positions * mult)`
2. 若 `holdings_before >= effective_max_positions`，当天新买单继续直接命中 `MAX_POSITIONS_REACHED`
3. 这条路径继续作为 rollback / Gate 对照基线

### 3.2 新冻结候选：`carryover_buffer(1)`

候选只改一件事：

1. 新配置项：
   - `MSS_MAX_POSITIONS_MODE=carryover_buffer`
   - `MSS_MAX_POSITIONS_BUFFER_SLOTS=1`
2. 触发条件：
   - 当天 `overlay_enabled=true`
   - `target_max_positions < base_max_positions`
   - 且 `batch_start_holdings_count >= target_max_positions`
3. 行为：
   - 当天允许在 `batch_start_holdings_count` 之上，额外保留 `1` 个 fresh slot
   - `effective_max_positions = min(base_max_positions, batch_start_holdings_count + 1)`
4. 不变项：
   - 不改 `risk_per_trade_pct`
   - 不改 `max_position_pct`
   - 不改 `PAS / IRS / final_score`

约束性结论：

`P4.1-C` 冻结的不是“放松所有风控”，而是“在 shrink 已经撞上 carryover 时，给当天保留极小的 fresh-slot 缓冲”。`

## 5. 核心结论

### 5.1 候选已经证明：主因确实可以沿 `max_positions shrink / carryover` 方向缓解

相对当前 `hard_cap`：

1. `trade_count`: `9 -> 13`
2. `expected_value`: `-0.008988 -> -0.000116`
3. `profit_factor`: `0.4613 -> 0.6213`
4. `reject_rate`: `0.2800 -> 0.1034`
5. `buy_reject_maxpos_count`: `7 -> 3`

对应 trade/reject 集合变化：

1. `relieved_maxpos_reject_count = 5`
2. `trade_set_changed_count = 6`
3. `reject_set_changed_count = 6`
4. `quantity_changed_count = 0`

约束性结论：

`候选的改善来自 slot path 本身，而不是 sizing path。`

### 5.2 候选还不能直接宣称“Gate ready”

相对当前 `hard_cap`，候选虽然改善了 `EV / PF / trade_count`，但：

1. `max_drawdown`: `0.006035 -> 0.007427`
2. 它不是纯粹“补回 5 笔被拒单”，还挤掉了 `1` 笔原本会成交的交易：
   - `2026-02-04`
   - `002517`
   - `entry_pnl = +371.5763`

同时新增的 `5` 笔候选独有成交聚合为：

1. `entry_pnl_total = +2965.0290`
2. `positive_count = 4`
3. `negative_count = 1`

净效果仍为正：

1. `candidate_only_entry_pnl_total - current_only_entry_pnl_total = +2593.4526`

但约束性结论仍然是：

`carryover_buffer(1)` 已经是有效候选，不等于已经通过 Gate。下一步仍必须进入 P4.1-D，重跑 matrix / idempotency / attribution / windowed sensitivity。`

### 5.3 候选释放的是“有限 fresh slot”，不是“恢复无 overlay”

最关键的 `relieved` 样本共 `5` 笔：

1. `2026-01-30`:

约束性结论：

`候选没有把 MSS overlay 退回“无风控”；它只是把 carryover 撞满后的 fresh-slot 语义从 0 改成了 1。`
